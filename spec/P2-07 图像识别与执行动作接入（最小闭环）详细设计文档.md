# 1. 文档目标

P2-05 已经实现：

```text
ExecutorWorker
ExecutorTaskRunner
StopToken
后台异步执行
协作式停止
completed / stopped / failed 信号
```

P2-06 已经实现：

```text
执行日志
执行进度
当前步骤
结果反馈
日志最大条数限制
```

但目前 `ExecutorWorker` 的执行内容仍然只是：

```text
占位循环
sleep
progress
```

还没有真正的自动化闭环。

P2-07 的目标是：

```text
将 ExecutorWorker 从占位循环升级为最小 Observe-Decide-Act 执行闭环。
```

即：

```text
Observe：截图
Decide：识别目标
Act：执行动作 / dry-run 动作
```

最终形成：

```text
截图 → 识别 → 坐标映射 → 动作执行 → 日志反馈
```

本阶段仍然不实现复杂游戏逻辑，不实现真实模板资源管理，不默认真实点击。

P2-07 最终采用以下方案：

# 6. 识别器设计

## 6.1 `vision/recognizers.py`

```python
from __future__ import annotations

from typing import Protocol

from PIL import Image

from nzfz_executor.core.vision.models import (
    ImagePoint,
    RecognitionResult,
    TargetCandidate,
)


class ImageRecognizer(Protocol):
    def recognize(self, image: Image.Image) -> RecognitionResult:
        ...


class CenterPointRecognizer:
    def recognize(self, image: Image.Image) -> RecognitionResult:
        width, height = image.size

        if width <= 0 or height <= 0:
            return RecognitionResult(
                found=False,
                candidates=[],
                message="截图尺寸无效",
            )

        candidate = TargetCandidate(
            name="center",
            point=ImagePoint(
                x=width // 2,
                y=height // 2,
            ),
            confidence=1.0,
        )

        return RecognitionResult(
            found=True,
            candidates=[candidate],
            message="识别到截图中心点",
        )
```

---

## 6.2 为什么使用 CenterPointRecognizer

P2-07 的目标是打通架构，不追求复杂识别准确率。

`CenterPointRecognizer` 的作用：

```text
1. 不依赖模板资源
2. 不依赖 OpenCV
3. 结果稳定
4. 便于验证截图、识别、坐标映射、动作链路
5. 后续可以替换为 TemplateMatcher
```

# 8. MouseController 设计

## 8.1 `actions/mouse_controller.py`

```python
from __future__ import annotations

from nzfz_executor.core.actions.models import (
    ActionResult,
    ClickAction,
)


class MouseController:
    def __init__(self, dry_run: bool = True) -> None:
        self._dry_run = dry_run

    @property
    def dry_run(self) -> bool:
        return self._dry_run

    def click(self, action: ClickAction) -> ActionResult:
        if self._dry_run:
            point = action.point
            return ActionResult(
                success=True,
                message=(
                    "dry-run：跳过真实点击 "
                    f"screen=({point.x},{point.y})"
                ),
            )

        return ActionResult(
            success=False,
            message="真实点击尚未实现",
        )
```

# 9. 坐标映射设计

## 9.1 新增文件

```text
nzfz_executor/core/executor/coordinate_mapper.py
```

---

## 9.2 设计目标

识别器输出：

```text
截图图像坐标 ImagePoint
```

动作执行器需要：

```text
屏幕坐标 ScreenPoint
```

因此需要：

```text
CoordinateMapper
```

将：

```text
image=(x,y)
```

映射为：

```text
screen=(x,y)
```

---

## 9.3 初版映射策略

P2-07 采用最小实现：

```text
screen_x = window_rect.left + image_x
screen_y = window_rect.top + image_y
```

该实现假设：

```text
截图图像左上角对应窗口矩形左上角。
```

如果当前截图实际是客户区截图，后续应替换为：

```text
ClientToScreen(hwnd, image_x, image_y)
```

因此 P2-07 必须把映射逻辑封装在 `CoordinateMapper` 中，避免散落在 Worker 中。

---

## 9.4 推荐代码

```python
from __future__ import annotations

from nzfz_executor.core.actions.models import ScreenPoint
from nzfz_executor.core.models import ConnectedWindowContext
from nzfz_executor.core.vision.models import ImagePoint


class CoordinateMapper:
    def image_to_screen(
        self,
        context: ConnectedWindowContext,
        point: ImagePoint,
    ) -> ScreenPoint:
        rect = context.window_rect

        return ScreenPoint(
            x=rect.left + point.x,
            y=rect.top + point.y,
        )
```

---

## 9.5 注意事项

如果 `ConnectedWindowContext` 中字段不是：

```python
context.window_rect.left
context.window_rect.top
```

而是其他结构，例如：

```python
context.rect.x
context.rect.y
```

实现时应按当前项目实际模型调整。

# 12. Observe-Decide-Act 执行流程

## 12.1 总体流程

```text
run()
    ↓
校验 runtime_context
    ↓
for iteration in max_iterations:
    ↓
    check stop
    ↓
    Observe: capture screenshot
    ↓
    check stop
    ↓
    Decide: recognize target
    ↓
    check stop
    ↓
    if found:
        coordinate map
        action click / dry-run
    else:
        skip action
    ↓
    check stop
    ↓
    sleep interval if not last iteration
    ↓
completed
```

---

## 12.2 推荐 Worker 代码骨架

```python
from __future__ import annotations

import time

from PySide6.QtCore import QObject, Signal, Slot

from nzfz_executor.core.actions.models import ClickAction
from nzfz_executor.core.executor.runtime_context import ExecutorRuntimeContext
from ui.workers.stop_token import StopToken


class ExecutorWorker(QObject):
    completed = Signal(int)
    stopped = Signal(int)
    failed = Signal(int, str)
    log = Signal(int, str)
    progress = Signal(int, int, str)

    def __init__(
        self,
        execution_id: int,
        runtime_context: ExecutorRuntimeContext | None,
        stop_token: StopToken,
    ) -> None:
        super().__init__()
        self._execution_id = execution_id
        self._runtime_context = runtime_context
        self._stop_token = stop_token

    @Slot()
    def run(self) -> None:
        try:
            if self._runtime_context is None:
                self.failed.emit(
                    self._execution_id,
                    "执行上下文为空，无法执行任务",
                )
                return

            ctx = self._runtime_context

            max_iterations = max(1, ctx.max_iterations)
            loop_interval_ms = max(0, ctx.loop_interval_ms)

            self._emit_log("执行任务已启动")
            self._emit_progress(0, "执行任务已启动")

            for index in range(max_iterations):
                if self._check_stop_and_emit():
                    return

                iteration = index + 1
                base_percent = int(index / max_iterations * 100)

                self._emit_log(f"开始第 {iteration} 轮执行")
                self._emit_progress(
                    base_percent,
                    f"开始第 {iteration} 轮执行",
                )

                self._run_one_iteration(
                    iteration=iteration,
                    total=max_iterations,
                )

                if self._check_stop_and_emit():
                    return

                self._emit_log(f"第 {iteration} 轮执行完成")

                if index < max_iterations - 1:
                    time.sleep(loop_interval_ms / 1000)

            self._emit_progress(100, "任务执行完成")
            self._emit_log("执行任务已完成")
            self.completed.emit(self._execution_id)

        except Exception as exc:
            self.failed.emit(self._execution_id, str(exc))
```

---

## 12.3 `_run_one_iteration()` 推荐代码

```python
def _run_one_iteration(
    self,
    iteration: int,
    total: int,
) -> None:
    ctx = self._runtime_context
    if ctx is None:
        raise RuntimeError("执行上下文为空")

    # 1. Observe
    self._emit_progress(
        self._iteration_percent(iteration, total, 10),
        "正在截图...",
    )
    self._emit_log("正在截图...")

    screenshot = ctx.screenshot_manager.capture(
        ctx.connected_context,
    )

    if screenshot is None:
        raise RuntimeError("截图失败：返回结果为空")

    image = screenshot.image

    self._emit_log(
        f"截图成功：{image.width}x{image.height}",
    )

    if self._check_stop_and_emit():
        return

    # 2. Decide
    self._emit_progress(
        self._iteration_percent(iteration, total, 40),
        "正在识别目标...",
    )
    self._emit_log("正在识别目标...")

    recognition = ctx.recognizer.recognize(image)

    if not recognition.found or not recognition.candidates:
        message = recognition.message or "未识别到目标，本轮跳过动作"
        self._emit_log(message)
        self._emit_progress(
            self._iteration_percent(iteration, total, 80),
            "未识别到目标，跳过动作",
        )
        return

    candidate = recognition.candidates[0]

    self._emit_log(
        "识别到目标："
        f"{candidate.name}, "
        f"image=({candidate.point.x},{candidate.point.y}), "
        f"confidence={candidate.confidence:.2f}",
    )

    if self._check_stop_and_emit():
        return

    # 3. Coordinate Mapping
    screen_point = ctx.coordinate_mapper.image_to_screen(
        ctx.connected_context,
        candidate.point,
    )

    self._emit_log(
        "坐标映射："
        f"image=({candidate.point.x},{candidate.point.y}) "
        f"-> screen=({screen_point.x},{screen_point.y})",
    )

    if self._check_stop_and_emit():
        return

    # 4. Act
    self._emit_progress(
        self._iteration_percent(iteration, total, 70),
        "正在执行动作...",
    )

    action = ClickAction(point=screen_point)
    result = ctx.mouse_controller.click(action)

    if not result.success:
        raise RuntimeError(result.message or "动作执行失败")

    self._emit_log(result.message or "动作执行成功")

    self._emit_progress(
        self._iteration_percent(iteration, total, 95),
        f"第 {iteration} 轮动作完成",
    )
```

## 12.4 停止检查方法

```python
def _check_stop_and_emit(self) -> bool:
    if self._stop_token.is_stop_requested():
        self._emit_log("任务收到停止请求")
        self.stopped.emit(self._execution_id)

```

## 12.5 进度计算方法

```python
def _iteration_percent(
    self,
    iteration: int,
    total: int,
    inner_percent: int,
) -> int:
    total = max(1, total)
    iteration = max(1, iteration)
    inner_percent = max(0, min(100, inner_percent))

    return int(((iteration - 1) + inner_percent / 100) / total * 100)
```


但如果当前项目实际接口为：

```python

```


推荐抽象目标是：

```text
Worker 拿到 PIL Image 即可。
```

如果截图失败，应统一抛出异常或转换为 failed：

```python
raise RuntimeError(f"截图失败：{error_message}")
```

# 18. 失败策略

## 18.1 任务失败

以下情况视为失败：

```text
1. runtime_context 为空
2. 截图失败
3. 截图返回空图像
4. 识别器抛异常
5. 坐标映射抛异常
6. 动作执行返回 success=False
7. 其他未捕获异常
```

Worker 应发送：

```python
failed.emit(execution_id, message)
```

---

## 18.2 不视为失败

以下情况不视为失败：

```text
1. 未识别到目标
2. CenterPointRecognizer 正常返回
3. dry-run 跳过真实点击
```

未识别到目标只记录日志，并跳过动作。

# 25. 最终结论

P2-07 的核心价值是：

```text
首次将执行器从“占位循环”升级为真实自动化闭环骨架。
```

它建立了：

```text
截图 Observe
识别 Decide
坐标映射
动作 Act
日志反馈
停止检查
失败收敛
```

但为了安全和控制范围，P2-07 明确：

```text
默认 dry-run，不真实点击。
```

最终完成后，执行器将能够在后台线程中执行：

```text
1. 截取已连接窗口
2. 识别截图中心点
3. 将图像坐标映射为屏幕坐标
4. 生成点击动作
5. 以 dry-run 方式记录动作
6. 输出完整执行日志
7. 正常 completed / stopped / failed
```

这为后续需求奠定基础：

```text
P2-08 Windows 输入动作后端
    将 dry-run 升级为可选真实点击

P2-09 模板匹配与资源管理
    将 CenterPointRecognizer 替换为真实模板识别

P2-10 自动化任务配置与流程编排
    将单轮闭环升级为可配置任务流程
```
