P2-04 已经完成执行器运行状态机设计，定义了：

```text

```

但 P2-04 仍然只是状态骨架，没有真实的异步执行任务。

当前点击开始执行只是：

```text

```

```text

```

缺少：

```text
后台 Worker

协作式停止
执行完成信号
执行失败信号
执行停止信号

```

P2-05 的目标是：

```text
为执行器引入异步执行 Worker 和 TaskRunner，使执行任务在后台线程运行，不阻塞 UI，并支持用户请求停止、Worker 安全退出、状态机正确收敛。
```

本阶段仍然不实现真实游戏自动化业务，只实现可运行、可停止、可收敛的执行任务框架。

# 2. 设计结论

根据讨论，P2-05 最终采用以下方案：

| 项目 | 结论 |
|---|---|
| 需求名称 | `P2-05 执行任务异步化与停止机制` |
| 异步模型 | `QThread + QObject Worker` |
| Worker | `ExecutorWorker` |
| Runner | `ExecutorTaskRunner` |
| 停止机制 | 协作式停止 |
| 是否强杀线程 | 否，禁止 `thread.terminate()` |
| StopToken | 是，基于 `threading.Event` |
| 同时执行任务数 | 1 个 |

| 是否预留进度信号 | 是 |
| 旧结果防污染 | `execution_id + connection_generation` |
| 停止超时 | 是 |

| 停止超时后状态 | `FAILED` |

# 5. StopToken 设计

## 5.1 文件位置

新增：

```text
ui/workers/stop_token.py
```

---

## 5.2 设计原则

停止机制采用：

```text
协作式停止
```

即：

```text
UI 请求停止
    ↓
StopToken 标记 stop_requested
    ↓
Worker 在安全点检查 stop_requested
    ↓
Worker 自行退出
```

禁止：

```python
thread.terminate()
```

原因：

```text
1. 强杀线程可能导致资源泄漏
2. 截图后端可能无法释放
3. WGC / Direct3D / WinRT 对象可能处于异常状态
4. UI 状态可能错乱
5. 后续真实执行动作无法安全回滚
```

---

## 5.3 推荐实现

```python
from __future__ import annotations

import threading


class StopToken:
    def __init__(self) -> None:
        self._event = threading.Event()

    def request_stop(self) -> None:
        self._event.set()

    def is_stop_requested(self) -> bool:
        return self._event.is_set()

    def reset(self) -> None:
        self._event.clear()
```

## 6.5 占位执行循环

P2-05 不做真实业务，使用占位循环：

```text
共执行 20 个 step
每个 step sleep 0.2 秒
每个 step 检查 stop_token
完成后发 completed
```

总时长约：

```text
4 秒
```

该循环用于验证：

```text
1. UI 不阻塞
2. 状态进入 RUNNING
3. 停止可以被响应
4. completed/stopped/failed 可以驱动 UI
5. 线程能正常释放
```

---

## 6.6 推荐代码骨架

```python
from __future__ import annotations

import time

from PySide6.QtCore import QObject, Signal, Slot

from nzfz_executor.core.models import ConnectedWindowContext
from ui.workers.stop_token import StopToken


class ExecutorWorker(QObject):
    completed = Signal(int)
    stopped = Signal(int)
    failed = Signal(int, str)
    log = Signal(int, str)
    progress = Signal(int, int, str)

    def __init__(
        self,
        execution_id: int,
        context: ConnectedWindowContext | None,
        stop_token: StopToken,
    ) -> None:
        super().__init__()
        self._execution_id = execution_id
        self._context = context
        self._stop_token = stop_token

    @Slot()
    def run(self) -> None:
        try:
            if self._context is None:
                self.failed.emit(
                    self._execution_id,
                    "当前未连接游戏窗口，无法执行任务",
                )
                return

            self.log.emit(self._execution_id, "执行任务已启动")

            total_steps = 20

            for step in range(total_steps):
                if self._stop_token.is_stop_requested():
                    self.log.emit(self._execution_id, "任务收到停止请求")
                    self.stopped.emit(self._execution_id)
                    return

                percent = int((step + 1) / total_steps * 100)
                self.progress.emit(
                    self._execution_id,
                    percent,
                    f"正在执行占位步骤 {step + 1}/{total_steps}",
                )

                time.sleep(0.2)

            self.log.emit(self._execution_id, "执行任务已完成")
            self.completed.emit(self._execution_id)

        except Exception as exc:
            self.failed.emit(self._execution_id, str(exc))
```


## 7.6 推荐代码骨架

```python
from __future__ import annotations

from PySide6.QtCore import QObject, QThread, Signal

from nzfz_executor.core.models import ConnectedWindowContext
from ui.workers.executor_workers import ExecutorWorker
from ui.workers.stop_token import StopToken


class ExecutorTaskRunner(QObject):
    completed = Signal(int)
    stopped = Signal(int)
    failed = Signal(int, str)
    log = Signal(int, str)
    progress = Signal(int, int, str)
    start_rejected = Signal(int, str)

    def __init__(self, parent: QObject | None = None) -> None:
        super().__init__(parent)
        self._thread: QThread | None = None
        self._worker: ExecutorWorker | None = None
        self._stop_token: StopToken | None = None

    def start(
        self,
        execution_id: int,
        context: ConnectedWindowContext | None,
    ) -> bool:
        if self.is_running():
            self.start_rejected.emit(
                execution_id,
                "当前已有任务正在运行",
            )
            return False

        self._stop_token = StopToken()
        self._thread = QThread(self)
        self._worker = ExecutorWorker(
            execution_id=execution_id,
            context=context,
            stop_token=self._stop_token,
        )

        self._worker.moveToThread(self._thread)

        self._thread.started.connect(self._worker.run)

        self._worker.completed.connect(self.completed)
        self._worker.stopped.connect(self.stopped)
        self._worker.failed.connect(self.failed)
        self._worker.log.connect(self.log)
        self._worker.progress.connect(self.progress)

        self._worker.completed.connect(self._thread.quit)
        self._worker.stopped.connect(self._thread.quit)
        self._worker.failed.connect(self._thread.quit)

        self._worker.completed.connect(self._worker.deleteLater)
        self._worker.stopped.connect(self._worker.deleteLater)
        self._worker.failed.connect(self._worker.deleteLater)

        self._thread.finished.connect(self._thread.deleteLater)
        self._thread.finished.connect(self._clear_current_task)

        self._thread.start()
        return True

    def request_stop(self) -> bool:
        if self._stop_token is None:
            return False

        self._stop_token.request_stop()
        return True

    def is_running(self) -> bool:
        return self._thread is not None and self._thread.isRunning()

    def _clear_current_task(self) -> None:
        self._thread = None
        self._worker = None
        self._stop_token = None
```

# 9. execution_id 与 generation

## 9.1 为什么需要 execution_id

执行任务是异步的，可能出现：

```text
任务 A 启动
用户请求停止
状态变化
旧任务信号稍后返回
```

为了避免旧信号污染 UI，需要使用：

```text
execution_id
```

---

## 9.2 为什么需要 connection_generation

执行任务依赖当前连接窗口。

可能出现：

```text
窗口 A 上启动执行
连接切换为窗口 B
A 的任务信号返回
```

此时不能让 A 的结果覆盖当前 UI。

因此执行任务启动时记录：

```text
当前 connection_generation
```

---

## 9.3 字段设计

```python
self._execution_id: int = 0
self._active_execution_id: int | None = None
self._execution_generations: dict[int, int] = {}
```

---

## 9.4 生成 execution_id

```python
def _next_execution_id(self) -> int:
    self._execution_id += 1
    return self._execution_id
```

---

## 9.5 记录 generation

```python
execution_id = self._next_execution_id()
self._active_execution_id = execution_id
self._execution_generations[execution_id] = self._connection_generation
```

---

## 9.6 结果有效性判断

```python
def _is_active_execution_result(self, execution_id: int) -> bool:
    if execution_id != self._active_execution_id:
        logger.debug(
            "Discard expired execution result: execution_id=%s, active=%s",
            execution_id,
            self._active_execution_id,
        )
        return False

    request_generation = self._execution_generations.get(execution_id)

    if request_generation != self._connection_generation:
        logger.debug(
            "Discard execution result due to generation mismatch: "
            "execution_id=%s, request_generation=%s, current_generation=%s",
            execution_id,
            request_generation,
            self._connection_generation,
        )
        return False

    return True
```

# 10. 开始执行流程

## 10.1 流程

```text
用户点击开始执行
    ↓
检查 executor_state == READY
    ↓
检查 _is_executor_ready()
    ↓
生成 execution_id
    ↓
记录 connection_generation
    ↓
获取 connected_context
    ↓
设置 active_execution_id
    ↓
状态切换 RUNNING
    ↓
调用 ExecutorTaskRunner.start()
    ↓
如果 start 失败，状态切换 FAILED 或刷新 READY
```

---

## 10.2 推荐代码骨架

```python
def _on_start_executor_clicked(self) -> None:
    if self._executor_state != ExecutorRunState.READY:
        self._show_executor_feedback(
            FeedbackCode.EXECUTOR_START_BLOCKED,
        )
        return

    if not self._is_executor_ready():
        self._show_executor_feedback(
            FeedbackCode.EXECUTOR_START_BLOCKED,
        )
        self._refresh_executor_ready_state()
        return

    context = self._window_manager.get_connected_context()
    if context is None:
        self._show_executor_feedback(
            FeedbackCode.EXECUTOR_START_BLOCKED,
        )
        self._refresh_executor_ready_state()
        return

    execution_id = self._next_execution_id()
    self._active_execution_id = execution_id
    self._execution_generations[execution_id] = self._connection_generation

    self._set_executor_state(ExecutorRunState.RUNNING)

    started = self._executor_task_runner.start(
        execution_id=execution_id,
        context=context,
    )

    if not started:
        self._active_execution_id = None
        self._set_executor_state(
            ExecutorRunState.FAILED,
            "当前已有任务正在运行",
        )
        self._refresh_executor_ready_state()
```

# 11. 停止执行流程

## 11.1 流程

```text

    ↓

    ↓

    ↓

    ↓

    ↓

```

## 11.2 推荐代码骨架

```python
def _on_stop_executor_clicked(self) -> None:
    if self._executor_state != ExecutorRunState.RUNNING:
        return

    self._set_executor_state(ExecutorRunState.STOPPING)

    self._executor_stop_timeout_timer.start(
        EXECUTOR_STOP_TIMEOUT_MS,
    )

    accepted = self._executor_task_runner.request_stop()
    if not accepted:
        self._executor_stop_timeout_timer.stop()
        self._active_execution_id = None
        self._set_executor_state(
            ExecutorRunState.FAILED,
            "当前没有正在运行的任务",
        )
        self._refresh_executor_ready_state()
```

## 12.3 failed

```python
def _on_executor_failed(
    self,
    execution_id: int,
    message: str,
) -> None:
    if not self._is_active_execution_result(execution_id):
        return

    self._executor_stop_timeout_timer.stop()
    self._active_execution_id = None

    self._set_executor_state(
        ExecutorRunState.FAILED,
        message or "任务执行失败",
    )
    self._refresh_executor_ready_state()
```

# 13. 停止超时处理

## 13.1 设计

停止超时用于防止 UI 永久停留在：

```text
STOPPING
```

默认：

```text
10 秒
```

如果超时：

```text
1. 状态切换 FAILED
2. 显示任务停止超时
3. active_execution_id 清空
4. 刷新按钮状态
5. 不强杀线程
```

注意：

```text
即使停止超时，也不调用 thread.terminate()
```

---

## 13.2 推荐代码

```python
def _on_executor_stop_timeout(self) -> None:
    execution_id = self._active_execution_id

    logger.warning(
        "Executor stop timeout: execution_id=%s",
        execution_id,
    )

    self._active_execution_id = None

    self._set_executor_state(
        ExecutorRunState.FAILED,
        "任务停止超时，请检查后台任务状态",
    )
    self._refresh_executor_ready_state()
```

---

## 13.3 超时后旧信号处理

超时后：

```python
self._active_execution_id = None
```

因此 Worker 后续返回：

```text
stopped / completed / failed
```

都会被：

```python
_is_active_execution_result()
```

丢弃。

# 22. 验收标准

## 22.1 Worker/Runner 验收

P2-05 完成后应满足：

```text
1. 存在 StopToken
2. 存在 ExecutorWorker
3. 存在 ExecutorTaskRunner
4. 使用 QThread + QObject Worker
5. 执行任务不阻塞 UI
6. Worker 可正常 completed
7. Worker 可响应 request_stop 并 stopped
8. Worker 异常时 failed
9. Runner 管理线程生命周期
10. 同一时间只允许一个执行任务
11. 不使用 thread.terminate()
```

---

## 22.2 UI 状态验收

```text
1. READY 点击开始后进入 RUNNING
2. RUNNING 时后台任务实际启动
3. RUNNING 点击停止后进入 STOPPING
4. STOPPING 等待 Worker stopped
5. Worker stopped 后进入 STOPPED
6. Worker completed 后进入 COMPLETED
7. Worker failed 后进入 FAILED
8. 结束后根据连接状态恢复 READY 或 NOT_READY
```

---

## 22.3 停止机制验收

```text
1. 停止采用 StopToken
2. Worker 在循环中检查 StopToken
3. request_stop 后 Worker 能安全退出
4. 停止按钮在 STOPPING 中禁用
5. STOPPING 超过 10 秒进入 FAILED
6. 停止超时后不强杀线程
```

---

## 22.4 防污染验收

```text
1. Worker 信号携带 execution_id
2. UI 校验 active_execution_id
3. UI 校验 connection_generation
4. 旧任务 completed 不覆盖 UI
5. 旧任务 failed 不覆盖 UI
6. 停止超时后的旧 stopped 不覆盖 UI
```

---

## 22.5 操作互斥验收

```text
1. 执行中不能再次开始执行
2. 执行中不能断开/切换连接
3. 执行中不能手动刷新截图
4. 手动截图中不能开始执行
5. Runner 正在运行时再次 start 会被拒绝
```

奠定可扩展的执行框架基础。
