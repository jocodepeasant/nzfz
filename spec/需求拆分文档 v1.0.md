# 逆战塔防自动化通关系统 · 需求拆分文档 v1.0

## 1. 文档说明

本文档按照需求优先级对「逆战塔防自动化通关系统」进行拆分，用于指导开发排期、里程碑规划和验收管理。

优先级分为：

| 优先级 | 含义 |
|---|---|
| P0 | 必须实现，决定最小闭环是否成立 |
| P1 | 核心增强，决定系统是否具备实际可用性 |
| P2 | 体验优化，提升易用性、稳定性、适配性 |
| P3 | 高级能力，属于后续增强或预研方向 |

---

## 2. 总体优先级策略

当前项目最小闭环是：

```text
共享 Schema
  ↓
示例脚本
  ↓
执行器加载脚本
  ↓
执行器连接窗口
  ↓
执行基础动作
  ↓
配置器导出脚本
```

因此，优先级应围绕以下目标排序：

1. 先固化协议
2. 再实现执行器窗口连接
3. 再实现脚本加载
4. 再实现基础动作执行
5. 再实现配置器最小可用
6. 再实现塔防动作抽象
7. 最后做截图识别、后台输入和高级恢复

## 4. P0 详细拆分

---

### P0-005：游戏窗口搜索

#### 目标

搜索真实 Windows 顶层窗口。

#### 搜索方式

- 按窗口标题
- 按进程名
- 按 PID

#### 验收标准

- 输入标题能找到窗口
- 输入进程名能找到窗口
- 输入 PID 能找到窗口
- 搜索结果包含 hwnd
- 搜索结果按匹配度排序

---

### P0-006：游戏窗口连接

#### 目标

连接指定窗口并保存连接上下文。

#### 必须保存

- hwnd
- pid
- title
- process_name
- window_rect
- client_rect
- connected_at

#### 验收标准

- 可以连接选中窗口
- 连接前重新校验窗口状态
- 最小化窗口禁止连接
- 连接成功后窗口被激活
- 激活失败时提示原因

---

### P0-007：断开连接

#### 目标

清空连接状态，但不影响游戏窗口。

#### 验收标准

- 断开后状态为未连接
- 不关闭游戏
- 不最小化游戏
- 不改变游戏前台状态
- 健康检测停止

---

### P0-008：健康检测

#### 目标

周期性维护连接状态。

#### 检测内容

- hwnd 是否有效
- PID 是否存活
- PID 是否一致
- 窗口是否可见
- 窗口是否最小化
- 窗口尺寸是否有效
- 刷新窗口矩形和客户区矩形

#### 验收标准

- 连接成功后每 1 秒检测一次
- 窗口正常时保持已连接
- 窗口最小化后显示异常
- 窗口恢复后自动回到已连接
- 窗口关闭后显示异常

---

### P0-009：脚本加载

#### 目标

执行器可以选择并读取 script.json。

#### 验收标准

- 可以选择文件
- 可以读取 JSON
- JSON 格式错误时提示
- 加载成功后显示脚本名称、地图名称、版本

---

### P0-010：Schema 校验

#### 目标

执行器加载脚本时校验 Schema。

#### 验收标准

- 合法脚本通过
- 非法脚本拒绝
- 显示错误字段路径
- 校验失败时禁止执行

---

### P0-011：基础动作模型

#### 目标

建立动作执行统一接口。

#### 交付物

```text
nzfz_executor/actions/base.py
ActionHandler
ActionResult
```

#### 验收标准

- 动作处理器可以注册
- 未知动作类型能提示错误
- 动作执行结果统一返回

---

### P0-012：wait 动作

#### 目标

支持等待动作。

#### 验收标准

- 能等待指定毫秒数
- 用户停止时可尽快中断
- 日志显示开始和结束

---

### P0-013：click 动作

#### 目标

支持点击坐标或槽位。

#### 验收标准

- 能点击指定客户区坐标
- 能根据 slotId 点击槽位
- 坐标转换正确
- 点击失败有错误提示

---

### P0-014：key_press 动作

#### 目标

支持基础键盘按键。

#### 验收标准

- 能发送普通键
- 能发送功能键
- 支持按下时长
- 执行结果有日志

---

### P0-015：前台输入通道

#### 目标

实现基础前台鼠标键盘输入。

#### 验收标准

- 能移动鼠标
- 能点击
- 能按键
- 执行前能激活窗口
- 窗口异常时拒绝执行

---

### P0-016：基础执行引擎

#### 目标

顺序执行脚本中的 wave 和 action。

#### 验收标准

- 能按波次顺序执行
- 能按动作顺序执行
- 能记录当前波次
- 能记录当前动作
- 能执行完成

---

### P0-017：基础运行日志

#### 目标

展示系统运行过程。

#### 验收标准

- 显示脚本加载日志
- 显示窗口连接日志
- 显示动作执行日志
- 显示错误日志

---

### P0-018：配置器最小导出能力

#### 目标

配置器至少可以生成一个合法脚本。

#### 最小功能

- 编辑脚本名称
- 编辑地图名称
- 添加陷阱
- 添加槽位
- 添加波次
- 添加基础动作
- 导出 JSON

#### 验收标准

- 导出的 JSON 通过 Schema 校验
- 执行器可以加载该 JSON

## 6. P1 详细拆分

### P1-001：地图编辑器基础画布

验收标准：

- 可以加载背景图
- 可以显示鼠标坐标
- 可以点击生成槽位

### P1-002：槽位可视化编辑

验收标准：

- 槽位显示在画布上
- 槽位可拖动
- 槽位可删除
- 槽位坐标可精确编辑

### P1-003：陷阱管理

验收标准：

- 可以新增陷阱
- 可以编辑热键
- 可以删除未引用陷阱
- 被引用陷阱删除时提示

### P1-004：波次编辑器

验收标准：

- 可以添加波次
- 可以添加动作
- 可以调整动作顺序
- 可以删除动作
- 参数缺失时提示

### P1-005：引用完整性校验

验收标准：

- 不存在的 slotId 能被发现
- 不存在的 trapId 能被发现
- 配置器导出前提示错误
- 执行器加载时二次校验

### P1-006：坐标换算模块

验收标准：

- 支持基准分辨率换算
- 支持窗口大小变化
- 支持窗口移动
- 客户区坐标准确

### P1-007：place_trap 动作

执行流程：

```text
按陷阱热键
等待
点击槽位
等待
返回结果
```

验收标准：

- 能根据 trapId 找到热键
- 能根据 slotId 找到坐标
- 能完成基础放置流程

### P1-008：upgrade_trap 动作

验收标准：

- 能点击槽位
- 能执行升级按键或点击
- 有日志输出

### P1-009：remove_trap 动作

验收标准：

- 能点击槽位
- 能执行拆除操作
- 可配置是否需要确认

### P1-010：暂停 / 继续 / 停止

验收标准：

- 运行中可以暂停
- 暂停后不继续发送输入
- 可以继续
- 可以停止
- 停止后状态正确

### P1-011：执行状态展示

验收标准：

- 显示当前波次
- 显示当前动作
- 显示总进度
- 显示执行状态

### P1-012：错误处理策略

验收标准：

- stopOnError=true 时失败停止
- stopOnError=false 时失败继续
- 错误被记录到日志

### P1-013：配置器导入脚本

验收标准：

- 能打开合法脚本
- 能显示地图、槽位、波次
- 非法脚本提示错误

## 11. 关键结论

P0 目标不是完整自动通关，而是打通最小工程闭环。

P1 目标是让系统具备塔防实际动作表达能力。

P2 目标是提升长期使用体验和适配能力。

P3 目标是进入截图识别、后台控制、自动恢复等高级阶段。
