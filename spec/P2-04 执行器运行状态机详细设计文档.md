# 1. 文档目标

在 P2-01 到 P2-03 中，系统已经逐步具备：

```text
连接游戏窗口
获取已连接窗口上下文
手动刷新截图
显示截图预览
接入或预留高级截图后端
```

但目前仍缺少一个执行器主流程的状态模型。

一旦开始实现自动化执行逻辑，如果没有统一的运行状态机，UI 中会出现大量分散状态字段，例如：

```python
self.is_running = True
self.is_stopping = False
self.start_button.setEnabled(...)
self.stop_button.setEnabled(...)
self.status_label.setText(...)
```

这会导致：

```text
1. 按钮启用规则分散
2. 执行状态难以维护
3. 连接、截图、执行之间的互斥关系不清晰
4. 后续 Worker 接入困难
5. 异常和停止流程容易混乱
```

因此，P2-04 的目标是：

```text
为执行器主流程建立独立的运行状态机，统一管理执行器的未就绪、就绪、运行中、停止中、已停止、已完成、失败等状态，并规范 UI 控件启用、连接操作限制、截图操作限制和用户反馈。
```

---

# 2. 设计结论

根据讨论，P2-04 最终采用以下方案：

| 项目 | 结论 |
|---|---|
| 需求名称 | `P2-04 执行器运行状态机` |
| 是否新增执行器状态枚举 | 是 |
| 状态枚举名称 | `ExecutorRunState` |
| 状态模块位置 | `ui/states/executor_state.py` |
| 是否引入暂停/恢复 | 否 |
| 是否引入 `ExecutorController` | 否 |
| 状态维护位置 | 暂由 `game_connect.py` 维护 |
| 是否新增执行控制区 | 是 |
| 执行控制区位置 | 暂放在 `game_connect.py` |
| 是否实现真实执行 Worker | 否 |
| 开始执行行为 | `READY → RUNNING` |
| 停止执行行为 | `RUNNING → STOPPING → STOPPED` |
| 是否立即停止完成 | 是，P2-04 暂时立即转为 `STOPPED` |
| 执行中是否禁止断开/切换连接 | 是 |
| 执行中是否禁用手动刷新截图 | 是 |
| 手动截图中是否允许开始执行 | 否 |
| 开始执行是否要求已有截图 | 否 |
| 是否扩展 feedback 文案 | 是 |

# 4. 与截图功能的关系

## 4.1 职责区别

截图功能负责：

```text
从当前连接窗口获取一帧画面。
```

执行器状态机负责：

```text
管理自动化任务生命周期，以及执行期间哪些 UI 操作允许、哪些禁止。
```

二者不是同一层。

---

## 4.2 依赖关系

执行器未来运行时通常会反复执行：

```text
截图
识别
决策
点击/按键
等待
再次截图
```

因此：

```text
截图是执行器的核心输入能力之一。
```

---

## 4.3 手动截图与执行中截图

P2-02 的手动截图用于：

```text
用户预览
调试连接
验证截图后端
```

未来 P2-05 执行中的截图用于：

```text
自动化识别
执行决策
```

二者使用同一个 `ScreenshotManager` 能力，但调用方不同。

---

## 4.4 执行中禁用手动截图

P2-04 规定：

```text
执行器 RUNNING / STOPPING 时，禁用手动刷新截图。
```

原因：

```text
1. 避免 UI 手动截图与执行 Worker 截图竞争资源
2. 避免 WGC 捕获会话冲突
3. 避免截图任务堆积
4. 避免执行器获取延迟帧
5. 保持状态简单
```

---

## 4.5 手动截图中禁止开始执行

P2-04 规定：

```text
_is_capturing=True 时，开始执行按钮禁用。
```

原因：

```text
1. 避免手动截图尚未结束时执行器启动
2. 避免执行器第一步截图和手动截图冲突
3. 保证执行入口状态清晰
```

## 5.2 ExecutorRunState

```python
from __future__ import annotations

from enum import Enum


class ExecutorRunState(Enum):
    NOT_READY = "not_ready"
    READY = "ready"
    RUNNING = "running"
    STOPPING = "stopping"
    STOPPED = "stopped"
    COMPLETED = "completed"
    FAILED = "failed"
```

---

## 5.3 状态说明

| 状态 | 含义 | 是否可开始 | 是否可停止 |
|---|---|---:|---:|
| `NOT_READY` | 执行器未就绪 | 否 | 否 |
| `READY` | 执行器就绪 | 是 | 否 |
| `RUNNING` | 正在执行任务 | 否 | 是 |
| `STOPPING` | 正在停止任务 | 否 | 否 |
| `STOPPED` | 任务已停止 | 否，刷新后回 READY | 否 |
| `COMPLETED` | 任务执行完成 | 否，刷新后回 READY | 否 |
| `FAILED` | 任务执行失败 | 否，刷新后回 READY/NOT_READY | 否 |

# 6. 状态转换规则

## 6.1 基础转换

```text
NOT_READY → READY
READY → NOT_READY
READY → RUNNING
RUNNING → STOPPING
STOPPING → STOPPED
RUNNING → COMPLETED
RUNNING → FAILED
STOPPED → READY / NOT_READY
COMPLETED → READY / NOT_READY
FAILED → READY / NOT_READY
```

---

## 6.2 事件驱动转换表

| 当前状态 | 事件 | 下一个状态 |
|---|---|---|
| `NOT_READY` | 连接就绪且满足执行条件 | `READY` |
| `READY` | 连接失效或条件不满足 | `NOT_READY` |
| `READY` | 点击开始执行 | `RUNNING` |
| `RUNNING` | 点击停止执行 | `STOPPING` |
| `STOPPING` | 停止完成 | `STOPPED` |
| `RUNNING` | 执行正常完成 | `COMPLETED` |
| `RUNNING` | 执行异常 | `FAILED` |
| `RUNNING` | 连接异常 | `FAILED` |
| `STOPPED` | 状态刷新且条件满足 | `READY` |
| `STOPPED` | 状态刷新且条件不满足 | `NOT_READY` |
| `COMPLETED` | 状态刷新且条件满足 | `READY` |
| `COMPLETED` | 状态刷新且条件不满足 | `NOT_READY` |
| `FAILED` | 状态刷新且条件满足 | `READY` |
| `FAILED` | 状态刷新且条件不满足 | `NOT_READY` |

---

## 6.3 P2-04 特殊规则

P2-04 不接入真实 Worker，因此：

```text
点击开始执行：
READY → RUNNING
```

```text
点击停止执行：
RUNNING → STOPPING → STOPPED
```

其中：

```text
STOPPING → STOPPED
```

在 P2-04 中可以立即完成。

P2-05 接入真实异步 Worker 后，`STOPPING` 应等待 Worker 确认停止完成后再进入 `STOPPED`。

# 7. 执行器就绪条件

## 7.1 基础就绪条件

P2-04 中执行器就绪条件为：

```text
1. 当前连接状态为 CONNECTED_READY
2. 当前存在 connected_context
3. 当前没有正在手动截图
4. 当前执行器不在 RUNNING / STOPPING
```

推荐方法：

```python
def _is_executor_ready(self) -> bool:
    context = self._window_manager.get_connected_context()

    return (
        self._connection_state == ConnectionUiState.CONNECTED_READY
        and context is not None
        and not self._is_capturing
        and self._executor_state
        not in {
            ExecutorRunState.RUNNING,
            ExecutorRunState.STOPPING,
        }
    )
```

实际字段名以 P1-04 实现为准。

---

## 7.2 是否要求已有截图

P2-04 规定：

```text
开始执行不要求用户已有一次手动截图成功。
```

原因：

```text
1. 手动截图是 UI 预览能力
2. 执行器未来会自行截图
3. 不应强制用户多点击一次“刷新截图”
```

---

## 7.3 是否要求 WGC 可用

P2-04 不强制要求 WGC 可用。

原因：

```text
1. P2-03 有 SCREEN 回退
2. 执行器的截图后端选择由 ScreenshotManager 负责
3. 后续可根据任务类型再细化要求
```

# 10. 状态设置方法

## 10.1 统一入口

所有执行器状态变化必须通过：

```python
_set_executor_state()
```

禁止散落写法：

```python
self._executor_state = ...
self.executor_status_label.setText(...)
self.start_executor_button.setEnabled(...)
```

---

## 10.2 推荐实现

```python
def _set_executor_state(
    self,
    state: ExecutorRunState,
    message: str | None = None,
) -> None:
    if self._executor_state == state and message is None:
        self._update_executor_controls()
        return

    self._executor_state = state

    if message is not None:
        self.executor_status_label.setText(message)
    else:
        self._show_executor_feedback_by_state(state)

    self._update_executor_controls()
    self._update_screenshot_button_state()
    self._update_connection_controls_state()
```

# 13. 刷新执行就绪状态

## 13.1 方法定义

```python
def _refresh_executor_ready_state(self) -> None:
    if self._executor_state in {
        ExecutorRunState.RUNNING,
        ExecutorRunState.STOPPING,
    }:
        return

    if self._is_executor_ready():
        self._set_executor_state(ExecutorRunState.READY)
    else:
        self._set_executor_state(ExecutorRunState.NOT_READY)
```

---

## 13.2 调用时机

以下场景应调用：

7. 手动截图超时后
8. 停止完成后

# 14. 开始执行流程

## 14.1 流程

```text
用户点击开始执行
    ↓
检查 executor_state 是否 READY
    ↓
检查 _is_executor_ready()
    ↓
如果不满足，提示当前状态不满足执行条件
    ↓
如果满足，设置 RUNNING
    ↓
禁用开始按钮
    ↓
启用停止按钮
    ↓
禁用连接/断开/切换
    ↓
禁用手动刷新截图
```

---

## 14.2 推荐代码骨架

```python
def _on_start_executor_clicked(self) -> None:
    if self._executor_state != ExecutorRunState.READY:
        self._show_executor_feedback(
            FeedbackCode.EXECUTOR_START_BLOCKED,
        )
        return

    if not self._is_executor_ready():
        self._show_executor_feedback(
            FeedbackCode.EXECUTOR_START_BLOCKED,
        )
        self._refresh_executor_ready_state()
        return

    self._set_executor_state(ExecutorRunState.RUNNING)
```

# 15. 停止执行流程

## 15.1 P2-04 流程

```text
用户点击停止执行
    ↓
检查 executor_state 是否 RUNNING
    ↓
设置 STOPPING
    ↓
禁用停止按钮
    ↓
显示正在停止任务
    ↓
立即设置 STOPPED
    ↓
刷新是否可重新 READY
```

---

def _on_connection_state_changed(
    self,
    state: ConnectionUiState,

                ExecutorRunState.FAILED,
