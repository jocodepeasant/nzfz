## 1. 文档目标

本文档用于定义 `WindowManager` 在完成真实窗口连接能力后的**健康检测完善能力**。

P0-01 到 P0-07 已经完成窗口连接 Core 层闭环：

```text
数据模型
    ↓
WindowManager 基础类
    ↓
真实窗口搜索
    ↓
连接前校验
    ↓
构建 ConnectedWindow
    ↓
激活目标窗口
    ↓
保存 _connected_window
    ↓
状态查询 / 断开连接
```

P1-01 的目标是在此基础上补齐：

```text
连接后的窗口状态检测能力
```

使执行器能够稳定判断当前连接窗口是否仍然有效，并为 UI 和后续自动化执行流程提供可靠状态依据。

本阶段重点解决以下问题：

- 当前是否已经连接窗口；
- 连接窗口句柄是否仍然有效；
- 目标进程是否仍然存活；
- 目标窗口是否仍然可见；
- 目标窗口是否已最小化；
- 目标窗口矩形是否仍然有效；
- 目标窗口客户区矩形是否仍然有效；
- 目标窗口是否为当前前台窗口；
- 当前连接是否健康；
- 当前连接是否执行就绪；
- 健康检测是否应刷新连接上下文中的动态窗口信息；
- 健康检测失败时是否自动断开连接。

## 2. 设计结论

根据当前 P0 设计和 P1 关键设计讨论，P1-01 最终采用以下方案：

| 项目 | 最终选择 |
|---|---|
| 实现位置 | `nzfz_executor/core/window_manager.py` |
| 模型扩展位置 | `nzfz_executor/core/models.py` |
| 对外接口 | `WindowManager.check_health()` |
| 返回类型 | `HealthCheckResult` |
| Core 层是否依赖 Qt | 否 |
| UI 层是否负责周期触发 | 是 |
| 健康检测是否自动断开 | 否 |
| 健康检测是否自动清空 `_connected_window` | 否 |
| 健康检测是否自动激活窗口 | 否 |
| 健康检测是否自动恢复最小化窗口 | 否 |
| 健康检测是否自动重连 | 否 |
| 健康检测是否允许刷新连接动态信息 | 是，仅健康时刷新 |
| 窗口不在前台是否算不健康 | 否 |
| 窗口不在前台语义 | 连接健康，但执行未就绪 |
| 执行就绪定义 | 连接健康且窗口在前台 |
| 最小化窗口健康状态 | 不健康 |
| 最小化窗口提示文案 | `窗口已最小化，请恢复窗口后重试` |
| hwnd 无效处理 | 返回不健康，不自动断开 |
| 进程退出处理 | 返回不健康，不自动断开 |
| 非 Windows 平台处理 | 返回错误结果，不抛出到 UI |
| 缺少依赖处理 | 返回错误结果，不抛出到 UI |
| `_last_error` 语义 | 仅记录异常/错误场景；健康时清空 |
| 周期检测间隔 | UI 层默认 2000ms |
| 上一次检测未完成时 | UI 层跳过本轮，避免堆积 |

## 3. 模块边界

### 3.1 `models.py` 职责

`models.py` 仍然是纯数据模型层。

P1-01 允许在其中对 `HealthCheckResult` 做小幅扩展，用于表达：

```text
连接是否健康
连接是否存在
窗口是否前台
是否执行就绪
```

允许包含：

- `dataclass`
- `Enum`
- 类型注解
- 简单属性计算
- 简单便捷方法

不允许包含：

- Windows API 调用；
- `win32gui`；
- `win32process`；
- `psutil`；
- PySide6 / Qt 代码；
- UI 状态逻辑；
- 线程逻辑；
- 健康检测业务逻辑。

---

### 3.2 `window_manager.py` 职责

`window_manager.py` 中的 `WindowManager.check_health()` 负责执行真实健康检测。

负责：

- 检查当前是否已连接；
- 检查平台和依赖；
- 检查 `hwnd` 是否有效；
- 检查进程是否存在并存活；
- 检查窗口是否可见；
- 检查窗口是否最小化；
- 获取窗口矩形；
- 获取客户区矩形；
- 判断是否当前前台窗口；
- 构造 `HealthCheckResult`；
- 在健康时刷新 `_connected_window` 的动态字段；
- 记录 `_last_error`；
- 记录基础日志。

不负责：

- 启动定时器；
- 管理 `QTimer`；
- 创建 `QThread`；
- 更新 UI；
- 禁用/启用按钮；
- 自动断开；
- 自动激活窗口；
- 自动恢复最小化窗口；
- 自动重连；
- 截图；
- 自动点击；
- 后台输入；
- 遮挡检测。

---

### 3.3 UI 层职责

UI 层不在本文档中详细实现，但 P1-01 需要为其预留语义。

UI 层负责：

- 连接成功后启动周期健康检测；
- 默认每 `2000ms` 检测一次；
- 使用 Worker 异步调用 `WindowManager.check_health()`；
- 上一轮健康检测未完成时跳过本轮；
- 根据 `HealthCheckResult` 更新连接状态展示；
- 根据 `HealthCheckResult.is_ready` 判断是否允许后续执行；
- 断开连接后停止健康检测。

## 4. 核心概念定义

### 4.1 连接存在

连接存在指：

```python
self._connected_window is not None
```

连接存在只表示 Core 层当前保存了一个连接上下文，不代表该窗口当前仍然可用。

例如以下情况中，连接对象可能仍然存在：

- 游戏窗口已关闭；
- 游戏进程已退出；
- 窗口句柄已失效；
- 窗口已最小化；
- 窗口不可见。

P1-01 明确：

```text
连接存在 != 连接健康
```

---

### 4.2 连接健康

连接健康指当前连接窗口满足以下条件：

```text
已连接
hwnd 有效
目标进程存在且存活
目标窗口可见
目标窗口未最小化
窗口矩形有效
客户区矩形有效
```

只要其中任意一项不满足，则连接不健康。

---

### 4.3 执行就绪

执行就绪指：

```text
连接健康
并且
目标窗口是当前前台窗口
```

即：

```text
执行就绪 = 连接健康 + 窗口在前台
```

执行就绪用于后续需要前台窗口的自动化能力，例如：

- 前台键盘输入；
- 前台鼠标点击；
- 依赖当前活动窗口的截图；
- 需要游戏窗口获得焦点的操作。

---

### 4.4 窗口不在前台

窗口不在前台时：

```text
连接仍可视为健康
但执行未就绪
```

即：

| 状态 | 结果 |
|---|---|
| hwnd 有效 | 是 |
| 进程存活 | 是 |
| 窗口可见 | 是 |
| 窗口未最小化 | 是 |
| 矩形有效 | 是 |
| 客户区有效 | 是 |
| 是否前台 | 否 |
| 连接健康 | 是 |
| 执行就绪 | 否 |

返回信息建议为：

```text
窗口连接正常，但当前不在前台
```

---

### 4.5 最小化窗口

最小化窗口在搜索阶段允许返回，但连接阶段禁止连接。

在健康检测阶段，如果已连接窗口变为最小化，则视为：

```text
连接不健康
执行未就绪
```

统一提示文案：

```text
窗口已最小化，请恢复窗口后重试
```

P1-01 不自动恢复最小化窗口。

## 5. 数据模型设计

### 5.1 `HealthCheckResult` 扩展目标

P0-01 已允许为 `HealthCheckResult` 增加便捷属性：

```text
is_healthy
is_connected
```

P1-01 建议在兼容原有模型基础上继续小幅扩展：

```text
is_foreground
is_ready
```

目的：

- UI 可以直接判断连接是否健康；
- UI 可以直接判断后续执行是否就绪；
- 不需要通过解析 `message` 判断状态；
- 后续执行器主流程可以复用该结果。

---

### 5.2 推荐模型定义

如现有 `HealthCheckResult` 已存在，应在不破坏原字段的前提下补充字段或属性。

推荐定义：

```python
from dataclasses import dataclass
from datetime import datetime

@dataclass(frozen=True)
class HealthCheckResult:
    status: HealthStatus
    message: str
    connected_window: ConnectedWindow | None = None
    is_foreground: bool = False
    checked_at: datetime | None = None

    @property
    def is_connected(self) -> bool:
        return self.connected_window is not None

    @property
    def is_healthy(self) -> bool:
        return self.status == HealthStatus.HEALTHY

    @property
    def is_ready(self) -> bool:
        return self.is_healthy and self.is_foreground
```

---

### 5.3 字段说明

| 字段 | 类型 | 说明 |
|---|---|---|
| `status` | `HealthStatus` | 健康检测状态 |
| `message` | `str` | 面向 UI 的状态说明 |
| `connected_window` | `ConnectedWindow \| None` | 当前连接窗口快照 |
| `is_foreground` | `bool` | 目标窗口是否是当前前台窗口 |
| `checked_at` | `datetime \| None` | 本次检测时间 |

---

### 5.4 `is_connected` 语义

`is_connected` 表示：

```text
Core 当前是否保存连接对象
```

推荐实现：

```python
@property
def is_connected(self) -> bool:
    return self.connected_window is not None
```

注意：

```text
is_connected=True 不代表窗口健康。
```

例如：

| 场景 | `is_connected` | `is_healthy` |
|---|---:|---:|
| 未连接 | False | False |
| 已连接且正常 | True | True |
| 已连接但窗口最小化 | True | False |
| 已连接但进程退出 | True | False |
| 已连接但 hwnd 失效 | True | False |

---

### 5.5 `is_healthy` 语义

`is_healthy` 表示：

```text
目标窗口当前满足连接健康条件
```

推荐实现：

```python
@property
def is_healthy(self) -> bool:
    return self.status == HealthStatus.HEALTHY
```

如果现有 `HealthStatus` 使用中文枚举值或混合枚举值，判断应以枚举成员为准，不应以字符串为准。

---

### 5.6 `is_ready` 语义

`is_ready` 表示：

```text
目标窗口当前可以执行需要前台窗口的自动化动作
```

推荐实现：

```python
@property
def is_ready(self) -> bool:
    return self.is_healthy and self.is_foreground
```

---

### 5.7 是否扩展 `ConnectedWindow`

P1-01 不强制扩展 `ConnectedWindow`。

如果现有模型已经包含以下字段，则健康检测成功时可以刷新：

- `title`
- `process_name`
- `window_rect`
- `client_rect`
- `is_visible`
- `is_minimized`
- `last_check_time`
- `last_active_time`

如果没有对应字段，不应为了 P1-01 大幅重构 `ConnectedWindow`。

推荐原则：

```text
最小必要扩展 HealthCheckResult；
谨慎扩展 ConnectedWindow。
```

## 6. `HealthStatus` 映射规则

### 6.1 总体原则

P1-01 不要求大规模扩展 `HealthStatus` 枚举。

如果现有枚举已包含足够状态，应直接复用。

如果现有枚举较少，则可以采用：

```text
HealthStatus 表达大类
message 表达细节
```

例如：

```text
HealthStatus.UNHEALTHY + "窗口已最小化，请恢复窗口后重试"
HealthStatus.UNHEALTHY + "目标窗口不可见"
HealthStatus.ERROR + "健康检测异常：xxx"
```

---

### 6.2 推荐状态映射表

| 场景 | 推荐 `HealthStatus` | `is_healthy` | `is_foreground` | `is_ready` | `message` |
|---|---|---:|---:|---:|---|
| 未连接 | `NOT_CONNECTED` | False | False | False | `未连接游戏窗口` |
| 非 Windows | `ERROR` | False | False | False | `当前功能仅支持 Windows 10 / Windows 11` |
| 缺少依赖 | `ERROR` | False | False | False | `缺少 pywin32 或 psutil 依赖，请先安装依赖后重试` |
| hwnd 无效 | `UNHEALTHY` / `DISCONNECTED` | False | False | False | `目标窗口句柄已失效，请重新搜索窗口` |
| 进程退出 | `UNHEALTHY` / `DISCONNECTED` | False | False | False | `目标窗口进程已退出，请重新搜索窗口` |
| 窗口不可见 | `UNHEALTHY` | False | False | False | `目标窗口不可见` |
| 窗口最小化 | `UNHEALTHY` | False | False | False | `窗口已最小化，请恢复窗口后重试` |
| 窗口矩形异常 | `UNHEALTHY` | False | False | False | `目标窗口矩形异常` |
| 客户区矩形异常 | `UNHEALTHY` | False | False | False | `目标窗口客户区矩形异常` |
| 健康但非前台 | `HEALTHY` | True | False | False | `窗口连接正常，但当前不在前台` |
| 健康且前台 | `HEALTHY` | True | True | True | `窗口连接正常` |
| API 异常 | `ERROR` | False | False | False | `健康检测异常：{error}` |

---

### 6.3 `DISCONNECTED` 状态说明

如果现有 `HealthStatus` 中存在 `DISCONNECTED`，可以用于：

- hwnd 已失效；
- 目标进程已退出；
- 连接对象对应的窗口已不可恢复。

但 P1-01 明确：

```text
即使返回 DISCONNECTED，也不自动清空 _connected_window。
```

是否真正清空连接状态仍由：

```python
disconnect_window()
```

显式执行。

## 7. 对外接口设计

### 7.1 接口签名

```python
class WindowManager:
    def check_health(self) -> HealthCheckResult:
        ...
```

---

### 7.2 输入参数

无输入参数。

健康检测始终针对：

```python
self._connected_window
```

当前保存的连接窗口执行。

---

### 7.3 返回值

返回：

```python
HealthCheckResult
```

任何情况下不应向 UI 直接抛出未捕获异常。

---

### 7.4 异常处理原则

`check_health()` 内部应捕获异常并返回：

```python
HealthCheckResult(
    status=HealthStatus.ERROR,
    message=f"健康检测异常：{error}",
    connected_window=self._connected_window,
    is_foreground=False,
)
```

除非是严重编程错误，否则不应让异常穿透到 UI 线程。

## 8. 健康检测流程设计

### 8.1 总体流程

```text
1. 获取当前 _connected_window
2. 若未连接，返回 NOT_CONNECTED
3. 清理/准备检测上下文
4. 检查平台是否支持
5. 检查 Windows 依赖是否可用
6. 读取 hwnd / pid 等连接信息
7. 检查 hwnd 是否有效
8. 检查目标进程是否存在并存活
9. 检查窗口是否可见
10. 检查窗口是否最小化
11. 获取窗口矩形
12. 获取客户区矩形
13. 校验窗口矩形和客户区矩形
14. 判断目标窗口是否当前前台窗口
15. 构建新的 ConnectedWindow 快照或刷新动态字段
16. 写入 _connected_window
17. 清空 _last_error
18. 返回 HealthCheckResult
```

---

### 8.2 伪代码

```python
def check_health(self) -> HealthCheckResult:
    connected = self._connected_window

    if connected is None:
        self._last_error = None
        return HealthCheckResult(
            status=HealthStatus.NOT_CONNECTED,
            message="未连接游戏窗口",
            connected_window=None,
            is_foreground=False,
            checked_at=datetime.now(),
        )

    if not self._is_windows:
        self._last_error = "当前功能仅支持 Windows 10 / Windows 11"
        return HealthCheckResult(
            status=HealthStatus.ERROR,
            message=self._last_error,
            connected_window=connected,
            is_foreground=False,
            checked_at=datetime.now(),
        )

    if not self._is_dependency_available:
        self._last_error = "缺少 pywin32 或 psutil 依赖，请先安装依赖后重试"
        return HealthCheckResult(
            status=HealthStatus.ERROR,
            message=self._last_error,
            connected_window=connected,
            is_foreground=False,
            checked_at=datetime.now(),
        )

    try:
        hwnd = connected.hwnd
        pid = connected.pid

        if not win32gui.IsWindow(hwnd):
            self._last_error = "目标窗口句柄已失效，请重新搜索窗口"
            return HealthCheckResult(
                status=HealthStatus.UNHEALTHY,
                message=self._last_error,
                connected_window=connected,
                is_foreground=False,
                checked_at=datetime.now(),
            )

        if not self._is_process_alive(pid):
            self._last_error = "目标窗口进程已退出，请重新搜索窗口"
            return HealthCheckResult(
                status=HealthStatus.UNHEALTHY,
                message=self._last_error,
                connected_window=connected,
                is_foreground=False,
                checked_at=datetime.now(),
            )

        if not win32gui.IsWindowVisible(hwnd):
            self._last_error = "目标窗口不可见"
            return HealthCheckResult(
                status=HealthStatus.UNHEALTHY,
                message=self._last_error,
                connected_window=connected,
                is_foreground=False,
                checked_at=datetime.now(),
            )

        if win32gui.IsIconic(hwnd):
            self._last_error = "窗口已最小化，请恢复窗口后重试"
            return HealthCheckResult(
                status=HealthStatus.UNHEALTHY,
                message=self._last_error,
                connected_window=connected,
                is_foreground=False,
                checked_at=datetime.now(),
            )

        window_rect = self._get_window_rect(hwnd)
        if not self._is_valid_rect(window_rect):
            self._last_error = "目标窗口矩形异常"
            return HealthCheckResult(
                status=HealthStatus.UNHEALTHY,
                message=self._last_error,
                connected_window=connected,
                is_foreground=False,
                checked_at=datetime.now(),
            )

        client_rect = self._get_client_rect(hwnd)
        if not self._is_valid_rect(client_rect):
            self._last_error = "目标窗口客户区矩形异常"
            return HealthCheckResult(
                status=HealthStatus.UNHEALTHY,
                message=self._last_error,
                connected_window=connected,
                is_foreground=False,
                checked_at=datetime.now(),
            )

        foreground_hwnd = win32gui.GetForegroundWindow()
        is_foreground = foreground_hwnd == hwnd

        refreshed = self._refresh_connected_window(
            connected=connected,
            window_rect=window_rect,
            client_rect=client_rect,
            is_foreground=is_foreground,
        )

        self._connected_window = refreshed
        self._last_error = None

        message = (
            "窗口连接正常"
            if is_foreground
            else "窗口连接正常，但当前不在前台"
        )

        return HealthCheckResult(
            status=HealthStatus.HEALTHY,
            message=message,
            connected_window=refreshed,
            is_foreground=is_foreground,
            checked_at=datetime.now(),
        )

    except Exception as error:
        self._last_error = f"健康检测异常：{error}"
        return HealthCheckResult(
            status=HealthStatus.ERROR,
            message=self._last_error,
            connected_window=connected,
            is_foreground=False,
            checked_at=datetime.now(),
        )
```

## 10. `_connected_window` 刷新规则

### 10.1 总体原则

P1-01 允许健康检测在成功时刷新 `_connected_window` 的动态字段。

健康检测失败时：

```text
不刷新 _connected_window
不清空 _connected_window
```

---

### 10.2 允许刷新的场景

只有当以下条件全部满足时，允许刷新：

```text
hwnd 有效
进程存活
窗口可见
窗口未最小化
窗口矩形有效
客户区矩形有效
```

也就是：

```text
连接健康时才刷新
```

---

### 10.3 可刷新字段

根据现有 `ConnectedWindow` 模型，允许刷新：

| 字段 | 是否建议刷新 | 说明 |
|---|---:|---|
| `title` | 是 | 窗口标题可能变化 |
| `process_name` | 可选 | 通常不变，但可重新读取 |
| `window_rect` | 是 | 窗口可能移动或缩放 |
| `client_rect` | 是 | 客户区可能变化 |
| `is_visible` | 是 | 如果模型中存在 |
| `is_minimized` | 是 | 如果模型中存在 |
| `connected_at` | 否 | 连接建立时间不应改变 |
| `control_mode` | 否 | 控制模式不应由健康检测改变 |
| `hwnd` | 否 | 目标句柄不应改变 |
| `pid` | 否 | 目标进程 ID 不应改变 |

---

### 10.4 刷新方式

由于 `ConnectedWindow` 在 P0 设计中是普通 `dataclass`，允许后续刷新。

可以采用两种方式。

#### 方案 A：原地更新

```python
connected.window_rect = window_rect
connected.client_rect = client_rect
connected.title = title
self._connected_window = connected
```

优点：

- 简单；
- 与普通 dataclass 设计一致。

缺点：

- 如果 UI 持有旧对象引用，可能发生隐式变化。

---

#### 方案 B：创建新对象替换

```python
refreshed = ConnectedWindow(
    hwnd=connected.hwnd,
    title=new_title,
    process_name=new_process_name,
    pid=connected.pid,
    window_rect=window_rect,
    client_rect=client_rect,
    ...
)
self._connected_window = refreshed
```

优点：

- 快照语义更清晰；
- 减少隐式副作用；
- 更利于 UI 比较。

缺点：

- 代码稍长。

---

### 10.5 推荐选择

P1-01 推荐：

```text
创建新 ConnectedWindow 对象替换 _connected_window。
```

原因：

1. 与 `WindowRect` / `WindowInfo` 的不可变值对象理念更一致；
2. UI 层更容易把健康检测结果视为一次快照；
3. 避免多个模块共享同一个可变对象时产生隐式副作用；
4. 后续多线程场景更容易维护。

如果现有 `ConnectedWindow` 字段较多，可使用：

```python
from dataclasses import replace

refreshed = replace(
    connected,
    title=new_title,
    process_name=new_process_name,
    window_rect=window_rect,
    client_rect=client_rect,
)
```

## 13. 辅助方法设计

P1-01 建议在 `WindowManager` 中补充以下私有辅助方法，便于复用和测试。

### 13.1 `_is_process_alive`

```python
def _is_process_alive(self, pid: int) -> bool:
    ...
```

职责：

- 检查 PID 是否存在；
- 检查进程是否运行；
- 捕获 `psutil.NoSuchProcess`；
- 捕获 `psutil.AccessDenied`；
- 捕获 `psutil.Error`。

建议：

```python
def _is_process_alive(self, pid: int) -> bool:
    try:
        if not psutil.pid_exists(pid):
            return False
        process = psutil.Process(pid)
        return process.is_running()
    except psutil.NoSuchProcess:
        return False
    except psutil.AccessDenied:
        return True
    except psutil.Error:
        return False
```

说明：

```text
AccessDenied 不代表进程不存在，因此可暂时视为存活。
```

---

### 13.2 `_get_window_rect`

```python
def _get_window_rect(self, hwnd: int) -> WindowRect:
    ...
```

职责：

- 调用 `win32gui.GetWindowRect(hwnd)`；
- 转换为 `WindowRect`。

---

### 13.3 `_get_client_rect`

```python
def _get_client_rect(self, hwnd: int) -> WindowRect:
    ...
```

职责：

- 调用 `win32gui.GetClientRect(hwnd)`；
- 使用 `ClientToScreen` 转换为屏幕坐标；
- 返回 `WindowRect`。

---

### 13.4 `_is_valid_rect`

```python
def _is_valid_rect(self, rect: WindowRect) -> bool:
    return rect.width > 0 and rect.height > 0
```

如果 `WindowRect` 已有 `is_valid` 属性，应优先复用：

```python
return rect.is_valid
```

### 13.5 `_refresh_connected_window`

```python
def _refresh_connected_window(
    self,
    connected: ConnectedWindow,
    window_rect: WindowRect,
    client_rect: WindowRect,
    is_foreground: bool,

    ...
```

职责：

- 获取最新窗口标题；
- 获取最新进程名；
- 使用 `dataclasses.replace` 创建新 `ConnectedWindow`；
- 保留连接建立时间、控制模式等不应变化的字段。

## 18. 验收标准

P1-01 完成后，应满足以下验收标准。

### 18.1 功能验收

- 已连接窗口正常存在时，`check_health()` 返回健康；
- 已连接窗口是前台时，`is_ready=True`；
- 已连接窗口不是前台时，`is_healthy=True` 且 `is_ready=False`；
- 已连接窗口最小化时，返回不健康；
- 已连接窗口被关闭时，返回不健康；
- 已连接窗口进程退出时，返回不健康；
- 已连接窗口不可见时，返回不健康；
- 窗口矩形异常时，返回不健康；
- 客户区矩形异常时，返回不健康；
- 未连接时，返回未连接；
- 非 Windows 平台返回明确错误；
- 缺少依赖时返回明确错误。

---

### 18.2 生命周期验收

- 健康检测失败时不自动断开；
- 健康检测失败时不清空 `_connected_window`；
- 健康检测失败时不自动激活窗口；
- 健康检测失败时不自动恢复最小化窗口；
- 健康检测失败时不自动重连；
- 只有 `disconnect_window()` 才负责显式断开当前连接。

---

### 18.3 状态语义验收

- `is_connected` 表示是否存在连接对象；
- `is_healthy` 表示连接窗口是否健康；
- `is_ready` 表示是否可以执行需要前台的动作；
- “窗口不在前台”不算错误；
- “窗口不在前台”不写入 `_last_error`；
- “窗口已最小化”必须使用统一文案：

```text
窗口已最小化，请恢复窗口后重试
```

---

### 18.4 UI 支撑验收

`HealthCheckResult` 必须足够支撑 UI 判断：

```text
未连接
已连接且就绪
已连接但未在前台
已连接但不健康
检测异常
```

UI 不应通过解析 `message` 判断核心状态，而应优先使用：

```python
result.is_connected
result.is_healthy
result.is_foreground
result.is_ready
result.status
```

当前不做：

## 21. 最终结论

P1-01 的核心设计结论是：

```text
健康检测只负责报告当前连接窗口状态，不负责改变连接生命周期。
```

本阶段通过 `WindowManager.check_health()` 建立稳定的连接状态判断能力，并明确区分：

```text
连接存在
连接健康
执行就绪
```

其中：

```text
连接存在 = Core 保存了 ConnectedWindow
连接健康 = 窗口句柄、进程、可见性、最小化状态、矩形均正常
执行就绪 = 连接健康且目标窗口在前台
```

健康检测失败时：

```text
不自动断开
不自动清空连接
不自动恢复最小化
不自动激活窗口
不自动重连
```

健康检测成功时：

```text
允许刷新 _connected_window 的动态窗口信息
```

该设计为后续 P1-02 异步 Worker、P1-04 UI 状态机以及 P2 执行器主流程集成提供稳定基础。
