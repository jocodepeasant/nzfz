# 1. 文档目标

P2-05 已经为执行器引入了异步执行框架：

```text
ExecutorWorker
ExecutorTaskRunner
StopToken
completed / stopped / failed 信号
log / progress 预留信号
协作式停止
停止超时
execution_id + connection_generation 防旧结果污染
```

但目前这些信号主要用于：

```text
驱动状态机
写入 logger
临时更新状态标签
```

用户无法清楚看到执行过程。

如果后续直接接入图像识别和自动点击，用户只能看到：

```text
正在执行任务...
任务执行完成
```

或：

```text
正在执行任务...
任务执行失败
```

但无法理解中间发生了什么：

```text
执行到了哪一步？
当前进度是多少？
是否收到停止请求？
为什么失败？
任务是否真的停止？
是否有可恢复警告？
```

因此，P2-06 的目标是：

```text
为执行器提供用户可见的执行日志、进度反馈、当前步骤展示和最终结果反馈，使用户能够理解任务执行过程、停止过程、失败原因和最终状态。
```

本阶段仍然不实现真实游戏自动化业务，只将 P2-05 的执行信号正式接入 UI 展示。

---

# 3. 配置化设计原则

从 P2-06 开始，确立以下原则：

```text
凡是影响用户体验、性能、保留数量、超时、刷新频率、后端策略的参数，都应提供默认值，并预留配置化能力。
```

例如：

| 配置项 | 当前默认值 | 是否配置化 |
|---|---:|---:|
| 执行日志最大条数 | `1000` | 是 |
| 截图超时时间 | `5000 ms` | 是 |
| 执行停止超时时间 | `10000 ms` | 是 |
| 截图后端策略 | `AUTO` | 是 |
| 是否优先 WGC | `True` | 是 |
| 日志时间格式 | `%H:%M:%S` | 是 |
| 执行中是否记录进度日志 | `True` | 是 |
| 执行占位任务 step 数 | `20` | 可配置 / 测试配置 |
| 执行占位任务 step 间隔 | `0.2s` | 可配置 / 测试配置 |

## 6.2 ExecutorLogLevel

```python
from __future__ import annotations

from enum import Enum


class ExecutorLogLevel(Enum):
    DEBUG = "debug"
    INFO = "info"
    SUCCESS = "success"
    WARNING = "warning"
    ERROR = "error"
```

---

## 6.3 日志等级说明

| 等级 | 用途 |
|---|---|
| `DEBUG` | 开发调试信息，默认尽量少显示 |
| `INFO` | 普通执行步骤 |
| `SUCCESS` | 成功节点 |
| `WARNING` | 可恢复异常、用户停止、提醒 |
| `ERROR` | 执行失败、停止超时、不可恢复错误 |

---

## 6.4 ExecutorLogEntry

```python
from __future__ import annotations

from dataclasses import dataclass
from datetime import datetime

from ui.models.executor_log import ExecutorLogLevel


@dataclass
class ExecutorLogEntry:
    timestamp: datetime
    level: ExecutorLogLevel
    message: str
    execution_id: int | None = None
    step: str | None = None
```

---

## 6.5 注意事项

如果 `ExecutorLogLevel` 和 `ExecutorLogEntry` 放在同一个文件，应避免自引用导入。

推荐完整文件写法：

```python
from __future__ import annotations

from dataclasses import dataclass
from datetime import datetime
from enum import Enum


class ExecutorLogLevel(Enum):
    DEBUG = "debug"
    INFO = "info"
    SUCCESS = "success"
    WARNING = "warning"
    ERROR = "error"


@dataclass
class ExecutorLogEntry:
    timestamp: datetime
    level: ExecutorLogLevel
    message: str
    execution_id: int | None = None
    step: str | None = None
```

# 11. 日志追加方法

## 11.1 统一入口

所有用户可见执行日志必须通过：

```python
_append_executor_log()
```

禁止在各处直接：

```python
self.executor_log_text_edit.appendPlainText(...)
```

---

## 11.2 推荐签名

```python
def _append_executor_log(
    self,
    level: ExecutorLogLevel,
    message: str,
    execution_id: int | None = None,
    step: str | None = None,
) -> None:
    ...
```

---

## 11.3 推荐实现

```python
from datetime import datetime

from ui.config.defaults import DEFAULT_EXECUTOR_LOG_TIME_FORMAT
from ui.models.executor_log import ExecutorLogEntry, ExecutorLogLevel


def _append_executor_log(
    self,
    level: ExecutorLogLevel,
    message: str,
    execution_id: int | None = None,
    step: str | None = None,
) -> None:
    entry = ExecutorLogEntry(
        timestamp=datetime.now(),
        level=level,
        message=message,
        execution_id=execution_id,
        step=step,
    )

    self._executor_log_entries.append(entry)

    while len(self._executor_log_entries) > self._max_executor_log_lines:
        self._executor_log_entries.pop(0)

    self._render_executor_logs()
    self._scroll_executor_log_to_bottom()
```

# 12. 日志渲染方法

## 12.1 格式化单条日志

```python
def _format_executor_log_entry(
    self,
    entry: ExecutorLogEntry,
) -> str:
    timestamp = entry.timestamp.strftime(
        DEFAULT_EXECUTOR_LOG_TIME_FORMAT,
    )
    level = entry.level.value.upper()

    return f"[{timestamp}] [{level}] {entry.message}"
```

---

## 12.2 渲染全部日志

```python
def _render_executor_logs(self) -> None:
    lines = [
        self._format_executor_log_entry(entry)
        for entry in self._executor_log_entries
    ]

    self.executor_log_text_edit.setPlainText("\n".join(lines))
```

---

## 12.3 自动滚动到底部

```python
def _scroll_executor_log_to_bottom(self) -> None:
    scroll_bar = self.executor_log_text_edit.verticalScrollBar()
    scroll_bar.setValue(scroll_bar.maximum())
```

# 13. 日志清空

## 13.1 清空方法

```python
def _clear_executor_logs(self) -> None:

    self.executor_log_text_edit.clear()
```

---

## 13.2 执行中是否允许清空

P2-06 规定：

```text
执行中允许清空日志。
```

原因：

```text
1. 清空只影响 UI 展示
2. 不影响 Worker
3. 不影响状态机
4. 不影响 logger 文件或控制台日志
```

---

## 13.3 清空后是否写一条日志

可以选择不写。

否则用户点击清空后又立刻出现：

```text

```

体验可能不一致。

P2-06 建议：

```text
清空后不自动追加“日志已清空”。
```

# 15. Worker 信号接入

## 15.1 log 信号

def _on_executor_log(
    self,
    execution_id: int,
    message: str,
) -> None:
    if not self._is_active_execution_result(execution_id):
        logger.debug(
            "Discard expired executor log: execution_id=%s",
            execution_id,
        )
        return

    logger.info("Executor log[%s]: %s", execution_id, message)

    self._append_executor_log(
        ExecutorLogLevel.INFO,

def _on_executor_progress(
    self,
    execution_id: int,
    percent: int,
    message: str,
) -> None:
    if not self._is_active_execution_result(execution_id):
        logger.debug(
            "Discard expired executor progress: execution_id=%s",
            execution_id,
        )
        return

    self._update_executor_progress(percent, message)

```python
def _on_start_executor_clicked(self) -> None:
    if self._executor_state != ExecutorRunState.READY:
        self._show_executor_feedback(
            FeedbackCode.EXECUTOR_START_BLOCKED,
        )
        return

    if not self._is_executor_ready():
        self._show_executor_feedback(
            FeedbackCode.EXECUTOR_START_BLOCKED,
        )
        self._refresh_executor_ready_state()
        return

    context = self._window_manager.get_connected_context()
    if context is None:
        self._show_executor_feedback(
            FeedbackCode.EXECUTOR_START_BLOCKED,
        )
        self._refresh_executor_ready_state()
        return

    execution_id = self._next_execution_id()
    self._active_execution_id = execution_id
    self._execution_generations[execution_id] = self._connection_generation

    self.executor_progress_bar.setValue(0)
    self._set_executor_step("准备启动执行任务")

    self._append_executor_log(
        ExecutorLogLevel.INFO,
        "准备启动执行任务",
        execution_id=execution_id,
    )

    self._set_executor_state(ExecutorRunState.RUNNING)

    started = self._executor_task_runner.start(
        execution_id=execution_id,
        context=context,
    )

    if not started:
        self._append_executor_log(
            ExecutorLogLevel.ERROR,
            "执行任务启动失败：当前已有任务正在运行",
            execution_id=execution_id,
        )

        self._active_execution_id = None
        self._set_executor_state(
            ExecutorRunState.FAILED,
            "当前已有任务正在运行",
        )
        self._set_executor_step("执行任务启动失败")
        self._refresh_executor_ready_state()
        return

    self._append_executor_log(
        ExecutorLogLevel.SUCCESS,
        "执行任务已启动",
        execution_id=execution_id,
    )
```

def _on_stop_executor_clicked(self) -> None:
    if self._executor_state != ExecutorRunState.RUNNING:
        return

    execution_id = self._active_execution_id

    self._append_executor_log(
        ExecutorLogLevel.WARNING,
        "用户请求停止任务",
        execution_id=execution_id,
    )

    self._set_executor_state(ExecutorRunState.STOPPING)
    self._set_executor_step("正在停止任务...")

    self._executor_stop_timeout_timer.start(
        EXECUTOR_STOP_TIMEOUT_MS,

    if not accepted:

        self._append_executor_log(

            "停止任务失败：当前没有正在运行的任务",
            execution_id=execution_id,
        )

        self._active_execution_id = None
        self._set_executor_state(
            ExecutorRunState.FAILED,
            "当前没有正在运行的任务",
        )
        self._set_executor_step("停止任务失败")
        self._refresh_executor_ready_state()
        return

    self._append_executor_log(
        ExecutorLogLevel.INFO,
        "停止请求已发送，等待任务安全退出",

def _on_executor_failed(
    self,
    execution_id: int,
    message: str,
) -> None:
    if not self._is_active_execution_result(execution_id):
        return

    self._executor_stop_timeout_timer.stop()
    self._active_execution_id = None

    display_message = message or "任务执行失败"

    self._set_executor_step(f"任务执行失败：{display_message}")

# 21. 与 logger 的关系

P2-06 的执行日志区是：

```text
用户可见日志。
```

Python logger 是：

```text
开发/运维调试日志。
```

二者关系：

```text
1. 用户关心的执行步骤写入 UI 日志
2. 内部调试、旧信号丢弃、线程清理写入 logger
3. 关键错误可以两边都写
```

例如：

```python
logger.warning("Executor failed: %s", message)
self._append_executor_log(ExecutorLogLevel.ERROR, f"任务执行失败：{message}")
```

# 22. 性能考虑

## 22.1 日志条数限制

默认最大条数：

```python
DEFAULT_MAX_EXECUTOR_LOG_LINES = 1000
```

当超过限制：

```text
删除最旧日志，保留最新 1000 条。
```

---

## 22.2 渲染策略

P2-06 采用：

```text
维护 list + setPlainText 全量刷新
```

在 1000 条以内可接受。

原因：

```text
1. 实现简单
2. 易于控制最大条数
3. QPlainTextEdit 处理 1000 行压力很小
```

如果后续日志量增大，可优化为：

```text
appendPlainText + QTextDocument block count
自定义 model/view
日志分页
```

---

## 22.3 进度日志频率

P2-05 占位任务每 0.2 秒一条 progress，默认 20 条，不构成压力。

后续真实执行中，如果 progress 频率过高，应增加节流策略，例如：

```text
每 N% 记录一次
每 500ms 记录一次
只记录阶段变化
```

该优化不属于 P2-06。

DEFAULT_MAX_EXECUTOR_LOG_LINES = 1000

P2-07 图像识别与执行动作接入
