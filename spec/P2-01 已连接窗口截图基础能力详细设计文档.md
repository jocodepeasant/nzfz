## 1. 文档目标

本文档用于定义已连接游戏窗口的截图基础能力。

P1 阶段已经完成：


P2 阶段开始进入执行器主流程集成，而截图是后续所有自动化能力的基础，例如：


P2-01 的目标是：

```text
在 Core 层新增截图基础架构，通过 ScreenshotManager 对已连接窗口进行截图，并为后续支持被遮挡窗口截图的 Windows Graphics Capture 后端预留扩展能力。
```

## 4. 核心概念

### 4.1 窗口区域 Window Rect

窗口区域是整个窗口在屏幕上的矩形区域，包含：

```text
标题栏
边框
阴影或系统装饰
客户区内容
```

示意：

```text
┌──────────────────────────────┐
│ 标题栏                         │
├──────────────────────────────┤
│                              │
│        游戏画面客户区           │
│                              │
└──────────────────────────────┘
```

---

### 4.2 客户区 Client Rect

客户区是应用实际绘制内容的区域。

对游戏窗口来说，客户区通常就是：

```text
游戏画面区域
```

不包含：

```text
标题栏
窗口边框
系统菜单
阴影
```

P2-01 默认截图客户区。

原因：

```text
后续图像识别、坐标定位和自动点击都应基于游戏画面本身。
```

---

### 4.3 客户区坐标原点

P2-01 明确：

```text
截图 image 的坐标原点为客户区左上角。
```

即：

```text
image pixel (0, 0) = 目标窗口客户区左上角
```

后续坐标映射：

```python
screen_x = client_rect.left + image_x
screen_y = client_rect.top + image_y
```

---

### 4.4 被遮挡截图

被遮挡截图指：

```text
目标窗口未最小化、未隐藏，但被其他窗口覆盖时，仍然能够获取目标窗口自身画面。
```

普通屏幕截图无法做到这一点。

P2-01 的基线后端：

```text
ScreenCaptureBackend
```

不支持被遮挡截图。

后续目标后端：

```text
WindowsGraphicsCaptureBackend
```

用于支持该能力。

---

## 5. 截图技术路线

### 5.1 P2-01 基线后端：ScreenCaptureBackend

技术方式：

```text
按客户区屏幕坐标进行屏幕区域截图。
```

可使用：

```text
PIL.ImageGrab.grab(bbox=...)
```

或等价屏幕截图实现。

---

### 5.2 ScreenCaptureBackend 优点

- 实现简单；
- 依赖少；
- 便于验证截图链路；
- 对可见无遮挡窗口可靠；
- 适合作为第一版基线能力。

---

### 5.3 ScreenCaptureBackend 限制

不支持：

```text
被遮挡窗口截图
最小化窗口截图
隐藏窗口截图
后台窗口真实内容截图
```

如果目标窗口被其他窗口遮挡，截到的可能是：

```text
遮挡窗口内容
```

而不是目标游戏窗口内容。

---

### 5.4 后续后端：WindowsGraphicsCaptureBackend

目标：

```text
支持被遮挡窗口截图
更好支持硬件加速窗口
更适合游戏窗口捕获
```

但 P2-01 不实现，只预留接口。

原因：

```text
Windows Graphics Capture 实现复杂，涉及 WinRT、Direct3D、帧池、GPU 纹理读取等内容，应独立成后续需求。
```

| 后端 | 可见无遮挡 | 被遮挡 | 不在前台 | 最小化 | 游戏兼容性 | P2-01 状态 |

## 7. 新增模型设计

### 7.1 `CaptureRegion`

用于描述截图区域。

```python
class CaptureRegion(Enum):
    CLIENT = "client"
    WINDOW = "window"
```

| 值 | 含义 |
|---|---|
| `CLIENT` | 截取客户区 |
| `WINDOW` | 截取整个窗口区域 |

P2-01 默认：

```python
CaptureRegion.CLIENT
```

---

### 7.2 `CaptureBackendType`

用于选择截图后端。

```python
class CaptureBackendType(Enum):
    AUTO = "auto"
    SCREEN = "screen"
    PRINT_WINDOW = "print_window"
    WINDOWS_GRAPHICS_CAPTURE = "windows_graphics_capture"
```

| 值 | 含义 |
|---|---|
| `AUTO` | 自动选择后端 |
| `SCREEN` | 屏幕区域截图 |
| `PRINT_WINDOW` | Windows `PrintWindow` 截图，P2-01 预留 |
| `WINDOWS_GRAPHICS_CAPTURE` | Windows Graphics Capture，P2-01 预留 |

P2-01 中：

```text
AUTO 暂时选择 SCREEN。
```

---

### 7.3 `CaptureOptions`

用于描述截图请求选项。

```python
@dataclass
class CaptureOptions:
    region: CaptureRegion = CaptureRegion.CLIENT
    backend: CaptureBackendType = CaptureBackendType.AUTO
    require_foreground: bool = False
    allow_occluded: bool = True
```

字段含义：

| 字段 | 类型 | 默认值 | 含义 |
|---|---|---|---|
| `region` | `CaptureRegion` | `CLIENT` | 截图区域 |
| `backend` | `CaptureBackendType` | `AUTO` | 截图后端 |
| `require_foreground` | `bool` | `False` | 是否要求目标窗口前台 |
| `allow_occluded` | `bool` | `True` | 是否期望支持被遮挡截图 |

说明：

```text
allow_occluded=True 表示调用方希望后端能够支持遮挡截图。
如果当前后端不支持，应在 ScreenshotResult 中通过 supports_occluded=False 和 message 说明能力限制。
```

---

### 7.4 `ScreenshotResult`

用于返回截图结果。

```python
@dataclass
class ScreenshotResult:
    success: bool
    image: Image.Image | None = None
    width: int = 0
    height: int = 0
    captured_at: datetime | None = None

    hwnd: int | None = None
    window_title: str = ""

    region: CaptureRegion = CaptureRegion.CLIENT
    backend: CaptureBackendType = CaptureBackendType.SCREEN

    window_rect: Rect | None = None
    client_rect: Rect | None = None

    supports_occluded: bool = False
    message: str = ""
```

字段含义：

| 字段 | 含义 |
|---|---|
| `success` | 是否截图成功 |
| `image` | 截图图片，成功时为 `PIL.Image.Image` |
| `width` | 图片宽度 |
| `height` | 图片高度 |
| `captured_at` | 截图时间 |
| `hwnd` | 目标窗口句柄 |
| `window_title` | 目标窗口标题 |
| `region` | 实际截图区域 |
| `backend` | 实际使用后端 |
| `window_rect` | 窗口屏幕区域 |
| `client_rect` | 客户区屏幕区域 |
| `supports_occluded` | 当前结果所用后端是否支持遮挡截图 |
| `message` | 失败原因或能力提示 |

```python
    @abstractmethod

        self,
        context: ConnectedWindowContext,
        options: CaptureOptions,

        ...
```

---

### 11.2 后端实现



### 11.3 后端选择规则

```python
def _select_backend(
    self,
    options: CaptureOptions,
) -> ScreenshotBackend:
    if options.backend == CaptureBackendType.AUTO:
        return self._screen_backend

    if options.backend == CaptureBackendType.SCREEN:
        return self._screen_backend

    if options.backend == CaptureBackendType.PRINT_WINDOW:
        return self._unsupported_backend(options.backend)

    if options.backend == CaptureBackendType.WINDOWS_GRAPHICS_CAPTURE:
        return self._unsupported_backend(options.backend)

    return self._screen_backend
```

P2-01 中：

```text

```

后续 P2-03 可改为：

```text
AUTO 优先 WGC，失败后 SCREEN。
```

## 12. `ScreenshotManager.capture()` 流程

### 12.1 总体流程

```text
输入 context/options
    ↓
检查 context 是否存在
    ↓
合并默认 options
    ↓
选择 backend
    ↓
调用 backend.capture()
    ↓
返回 ScreenshotResult
```

---

### 12.2 伪代码

```python
def capture(
    self,
    context: ConnectedWindowContext | None,
    options: CaptureOptions | None = None,
) -> ScreenshotResult:
    options = options or CaptureOptions()

    if context is None:
        return ScreenshotResult(
            success=False,
            region=options.region,
            backend=self._resolve_backend_type(options),
            message="当前未连接游戏窗口",
        )

    backend = self._select_backend(options)
    return backend.capture(context, options)
```

## 13. `ScreenCaptureBackend` 设计

### 13.1 职责

`ScreenCaptureBackend` 负责：

```text
基于窗口客户区或窗口区域的屏幕坐标进行截图。
```

---

### 13.2 技术方式

推荐第一版使用：

```python
PIL.ImageGrab.grab(bbox=(left, top, right, bottom))
```

---

### 13.3 能力声明

```python
backend_type = CaptureBackendType.SCREEN
supports_occluded = False
```

---

### 13.4 截图前校验

必须校验：

```text
1. hwnd 是否有效
2. 窗口是否可见
3. 窗口是否未最小化
4. 如果 require_foreground=True，则检查是否前台
5. window_rect 是否有效
6. client_rect 是否有效
7. 截图区域宽高是否大于 0
```

---

### 13.5 遮挡提示

如果：

```python
options.allow_occluded is True
```

但当前后端：

```python
supports_occluded is False
```

截图仍可执行，但结果应携带提示：

```text
当前截图后端不支持被遮挡窗口截图，若窗口被遮挡，截图内容可能不准确
```

该提示不一定表示失败。

---

### 13.6 为什么不直接失败

因为目标窗口可能并没有被遮挡。

`ScreenCaptureBackend` 无法可靠判断窗口是否被遮挡，所以：

```text
allow_occluded=True 但 backend 不支持遮挡
```

不应直接失败，只应在结果中说明能力限制。

## 15. `ScreenCaptureBackend.capture()` 推荐代码骨架

```python
from __future__ import annotations

from datetime import datetime

from PIL import ImageGrab

from nzfz_executor.core.models import (
    CaptureBackendType,
    CaptureOptions,
    CaptureRegion,
    ConnectedWindowContext,
    Rect,
    ScreenshotResult,
)


class ScreenCaptureBackend(ScreenshotBackend):
    backend_type = CaptureBackendType.SCREEN
    supports_occluded = False

    def capture(
        self,
        context: ConnectedWindowContext,
        options: CaptureOptions,
    ) -> ScreenshotResult:
        hwnd = context.hwnd

        window_title = getattr(context, "title", "")

        if not is_window(hwnd):
            return self._failure(
                context=context,
                options=options,
                message="窗口已失效，请重新连接",
            )

        if not is_window_visible(hwnd):
            return self._failure(
                context=context,
                options=options,
                message="窗口不可见，无法截图",
            )

        if is_window_minimized(hwnd):
            return self._failure(
                context=context,
                options=options,
                message="窗口已最小化，无法截图",
            )

        if options.require_foreground and not is_foreground_window(hwnd):
            return self._failure(
                context=context,
                options=options,
                message="窗口未在前台，无法截图",
            )

        window_rect = get_window_rect(hwnd)
        client_rect = get_client_rect_on_screen(hwnd)

        target_rect = (
            client_rect
            if options.region == CaptureRegion.CLIENT
            else window_rect
        )

        if target_rect.width <= 0 or target_rect.height <= 0:
            return self._failure(
                context=context,
                options=options,
                window_rect=window_rect,
                client_rect=client_rect,
                message="截图区域尺寸无效",
            )

        image = ImageGrab.grab(
            bbox=(
                target_rect.left,
                target_rect.top,
                target_rect.right,
                target_rect.bottom,
            )
        )

        message = ""
        if options.allow_occluded and not self.supports_occluded:
            message = (
                "当前截图后端不支持被遮挡窗口截图，"
                "若窗口被遮挡，截图内容可能不准确"
            )

        return ScreenshotResult(
            success=True,
            image=image,
            width=image.width,
            height=image.height,
            captured_at=datetime.now(),
            hwnd=hwnd,
            window_title=window_title,
            region=options.region,
            backend=self.backend_type,
            window_rect=window_rect,
            client_rect=client_rect,
            supports_occluded=self.supports_occluded,
            message=message,
        )
```

## 16. `Rect` 模型要求

如果项目已有 `Rect` 模型，应确保至少具备：

```python
@dataclass
class Rect:
    left: int
    top: int
    right: int
    bottom: int

    @property
    def width(self) -> int:
        return self.right - self.left

    @property
    def height(self) -> int:
        return self.bottom - self.top
```

如果已有字段名不同，P2-01 不强制修改，但截图模块需要适配。

## 18. 最小化与隐藏窗口规则

### 18.1 最小化窗口

P2-01 不支持最小化窗口截图。

返回：

```python
ScreenshotResult(
    success=False,
    message="窗口已最小化，无法截图",
)
```

原因：

- 游戏最小化后通常暂停渲染；
- 后台截图可能黑屏或旧帧；
- 不适合作为执行器可靠输入。

---

### 18.2 隐藏窗口

P2-01 不支持隐藏或不可见窗口截图。

返回：

```python
ScreenshotResult(
    success=False,
    message="窗口不可见，无法截图",
)
```

---

### 18.3 被遮挡窗口

P2-01 基线后端不支持可靠被遮挡截图。

如果目标窗口被遮挡：


后续由：


增强。

## 26. 最终结论

P2-01 的核心设计结论是：

```text
截图能力采用独立 ScreenshotManager，并通过截图后端抽象支持未来扩展。
```

本阶段实现：

```text
ScreenCaptureBackend
```

作为基线截图能力，支持：

```text
已连接窗口客户区截图
已连接窗口窗口区截图
可见无遮挡窗口截图
```

同时明确其限制：

```text
不支持被遮挡窗口可靠截图
不支持最小化窗口截图
不支持隐藏窗口截图
```

为满足后续“被遮挡时也能截图”的目标，P2-01 预留：

```text
CaptureBackendType.WINDOWS_GRAPHICS_CAPTURE
WindowsGraphicsCaptureBackend
supports_occluded
allow_occluded
```

后续可在 P2-03 中实现 Windows Graphics Capture 后端。

P2-01 完成后，Core 层将具备稳定的截图基础架构，为 P2-02 UI 截图预览和后续执行器主流程提供基础。
