# 9. TemplateLoader 设计

## 9.1 新增文件

```text
nzfz_executor/core/vision/template_loader.py
```

---

## 9.2 设计目标

`TemplateLoader` 负责：

```text
1. 扫描模板目录
2. 过滤支持的图片扩展名
3. 读取模板图片
4. 转换为灰度或彩色 OpenCV 图像
5. 跳过读取失败或尺寸无效的模板
6. 返回 TemplateResource 列表
```

---

## 9.3 错误处理策略

| 场景 | 策略 |
|---|---|
| 模板目录不存在 | 返回空列表 |
| 模板目录为空 | 返回空列表 |
| 文件扩展名不支持 | 跳过 |
| 单个模板读取失败 | 跳过 |
| 模板尺寸无效 | 跳过 |
| 所有模板失败 | 返回空列表 |

P2-09 不在 loader 中抛出业务异常。

---

## 9.4 推荐代码

```python
from __future__ import annotations

from pathlib import Path

import cv2

from nzfz_executor.core.vision.templates import TemplateResource


class TemplateLoader:
    def __init__(
        self,
        template_dir: str | Path,
        threshold: float,
        grayscale: bool = True,
        extensions: tuple[str, ...] = (
            ".png",
            ".jpg",
            ".jpeg",
            ".bmp",
        ),
    ) -> None:
        self._template_dir = Path(template_dir)
        self._threshold = threshold
        self._grayscale = grayscale
        self._extensions = tuple(
            extension.lower()
            for extension in extensions
        )

    @property
    def template_dir(self) -> Path:
        return self._template_dir

    def load_all(self) -> list[TemplateResource]:
        if not self._template_dir.exists():
            return []

        if not self._template_dir.is_dir():
            return []

        templates: list[TemplateResource] = []

        for path in sorted(self._template_dir.iterdir()):
            if not path.is_file():
                continue

            if path.suffix.lower() not in self._extensions:
                continue

            template = self._load_one(path)
            if template is not None:
                templates.append(template)

        return templates

    def _load_one(
        self,
        path: Path,
    ) -> TemplateResource | None:
        flag = cv2.IMREAD_GRAYSCALE if self._grayscale else cv2.IMREAD_COLOR
        image = cv2.imread(str(path), flag)

        if image is None:
            return None

        height, width = image.shape[:2]
        if width <= 0 or height <= 0:
            return None

        return TemplateResource(
            name=path.stem,
            path=path,
            image=image,
            width=width,
            height=height,
            threshold=self._threshold,
        )
```

# 10. TemplateMatcherRecognizer 设计

## 10.1 新增文件

```text
nzfz_executor/core/vision/template_matcher.py
```

---

## 10.2 设计目标

`TemplateMatcherRecognizer` 实现：

```python
ImageRecognizer
```

输入：

```text
PIL Image
```

输出：

```text
RecognitionResult
```

内部流程：

```text
1. 检查模板列表
2. 将 PIL Image 转为 OpenCV 图像
3. 灰度转换
4. 遍历模板
5. 对每个模板执行 cv2.matchTemplate
6. 取 max confidence 和 max location
7. 高于阈值则生成 TargetCandidate
8. 候选按 confidence 降序
9. 返回 RecognitionResult
```

---

## 10.3 PIL 转 OpenCV 图像

PIL 图像通常为 RGB。

OpenCV 使用 BGR 或灰度。

P2-09 默认灰度匹配：

```python
np_image = np.array(image.convert("RGB"))
gray = cv2.cvtColor(np_image, cv2.COLOR_RGB2GRAY)
```

如果不使用灰度匹配：

```python
bgr = cv2.cvtColor(np_image, cv2.COLOR_RGB2BGR)
```

---

## 10.4 推荐代码

```python
from __future__ import annotations

import cv2
import numpy as np
from PIL import Image

from nzfz_executor.core.vision.models import (
    ImagePoint,
    RecognitionResult,
    TargetCandidate,
)
from nzfz_executor.core.vision.recognizers import ImageRecognizer
from nzfz_executor.core.vision.templates import TemplateResource


class TemplateMatcherRecognizer:
    def __init__(
        self,
        templates: list[TemplateResource],
        grayscale: bool = True,
    ) -> None:
        self._templates = templates
        self._grayscale = grayscale

    @property
    def templates(self) -> list[TemplateResource]:
        return self._templates

    def recognize(self, image: Image.Image) -> RecognitionResult:
        if not self._templates:
            return RecognitionResult(
                found=False,
                candidates=[],
                message="未加载任何模板，跳过模板匹配",
            )

        source = self._pil_to_cv_image(image)

        source_height, source_width = source.shape[:2]

        candidates: list[TargetCandidate] = []
        best_name = ""
        best_confidence = 0.0

        for template in self._templates:
            if template.width > source_width or template.height > source_height:
                continue

            result = cv2.matchTemplate(
                source,
                template.image,
                cv2.TM_CCOEFF_NORMED,
            )

            _, max_val, _, max_loc = cv2.minMaxLoc(result)

            confidence = float(max_val)
            if confidence > best_confidence:
                best_confidence = confidence
                best_name = template.name

            if confidence < template.threshold:
                continue

            match_x, match_y = max_loc
            center_x = int(match_x + template.width // 2)
            center_y = int(match_y + template.height // 2)

            candidates.append(
                TargetCandidate(
                    name=template.name,
                    point=ImagePoint(
                        x=center_x,
                        y=center_y,
                    ),
                    confidence=confidence,
                ),
            )

        candidates.sort(
            key=lambda candidate: candidate.confidence,
            reverse=True,
        )

        if not candidates:
            return RecognitionResult(
                found=False,
                candidates=[],
                message=(
                    "未匹配到模板："
                    f"templates={len(self._templates)}, "
                    f"best={best_name or '-'}, "
                    f"confidence={best_confidence:.3f}"
                ),
            )

        best = candidates[0]

        return RecognitionResult(
            found=True,
            candidates=candidates,
            message=(
                "模板匹配成功："
                f"templates={len(self._templates)}, "
                f"best={best.name}, "
                f"confidence={best.confidence:.3f}, "
                f"image=({best.point.x},{best.point.y})"
            ),
        )

    def _pil_to_cv_image(
        self,
        image: Image.Image,
    ) -> np.ndarray:
        rgb = np.array(image.convert("RGB"))

        if self._grayscale:
            return cv2.cvtColor(rgb, cv2.COLOR_RGB2GRAY)

        return cv2.cvtColor(rgb, cv2.COLOR_RGB2BGR)
```

## 10.5 重要注意事项

模板加载时使用灰度，则识别时也必须使用灰度：

```text
template.image 灰度
source 灰度
```

模板加载时使用彩色，则识别时也必须使用彩色：

```text
template.image BGR
source BGR
```

因此：

```python
TemplateLoader(grayscale=...)
TemplateMatcherRecognizer(grayscale=...)
```

必须使用同一个配置值。

## 11.1 新增文件


---

## 11.2 设计目标

避免 `game_connect.py` 中直接写复杂识别器创建逻辑。

工厂负责：

```text
1. 根据 recognizer_type 创建识别器
2. recognizer_type=center 时返回 CenterPointRecognizer
3. recognizer_type=template 时加载模板并返回 TemplateMatcherRecognizer
4. 未知类型回退到 CenterPointRecognizer
```

---

## 11.3 为什么不在 core 层直接 import ui.config.defaults

为了保持分层清晰：

```text
core 层不依赖 ui 层。
```

因此 factory 接收参数：

```python
recognizer_type
template_dir
threshold
grayscale
extensions
```

由 UI 层传入默认配置。

---

## 11.4 推荐代码

```python
from __future__ import annotations

from pathlib import Path

from nzfz_executor.core.vision.recognizers import (
    CenterPointRecognizer,
    ImageRecognizer,
)
from nzfz_executor.core.vision.template_loader import TemplateLoader
from nzfz_executor.core.vision.template_matcher import (
    TemplateMatcherRecognizer,
)


def create_recognizer(
    recognizer_type: str,
    template_dir: str | Path,
    threshold: float,
    grayscale: bool = True,
    extensions: tuple[str, ...] = (
        ".png",
        ".jpg",
        ".jpeg",
        ".bmp",
    ),
) -> ImageRecognizer:
    normalized_type = recognizer_type.strip().lower()

    if normalized_type == "template":
        loader = TemplateLoader(
            template_dir=template_dir,
            threshold=threshold,
            grayscale=grayscale,
            extensions=extensions,
        )
        templates = loader.load_all()

        return TemplateMatcherRecognizer(
            templates=templates,
            grayscale=grayscale,
        )

    return CenterPointRecognizer()
```

```python
recognition = ctx.recognizer.recognize(image)
```

# 15. 多模板匹配策略

P2-09 采用：

```text
每个模板取一个最佳匹配。
```

对于每个模板：

```text
cv2.matchTemplate → minMaxLoc → max_val / max_loc
```

如果：

```text
max_val >= template.threshold
```

则生成一个候选：

```python
TargetCandidate(...)
```

所有候选：

```text
按 confidence 降序排列。
```

Worker 默认选择：

```python
recognition.candidates[0]
```

也就是：

```text
置信度最高的候选。
```

---

# 16. 未识别策略

P2-09 保持 P2-07 约定：

```text
未识别到目标不视为任务失败。
```

包括以下情况：

```text
1. 模板目录不存在
2. 模板目录为空
3. 模板全部加载失败
4. 模板都低于阈值
5. 模板尺寸大于截图尺寸
```

这些都返回：

```python
RecognitionResult(
    found=False,
    candidates=[],
    message="..."
)
```

Worker 记录日志并跳过动作。

---

# 17. 失败策略

以下情况可视为异常失败：

```text
1. OpenCV 调用抛出异常
2. PIL 图像转换异常
3. 模板 image 与 source 通道不一致导致异常
4. 其他未捕获异常
```

这些异常会被 `ExecutorWorker.run()` 的全局 try-except 捕获，并转为：

```python
failed.emit(execution_id, str(exc))
```

P2-09 不在 matcher 内部吞掉所有异常，避免隐藏真实 bug。

# 18. 性能注意事项

P2-09 初版每次识别会：

```text
遍历所有模板并执行 matchTemplate。
```

如果模板数量较少，例如：

```text
1 ~ 20 张
```

通常可以接受。

后续如果模板增多或截图很大，可以优化：

```text
1. ROI 区域限制
2. 缩放截图
3. 模板分组
4. 分阶段识别
5. 多线程匹配
6. 缓存灰度截图
7. 多尺度匹配按需启用
```

但这些不属于 P2-09。

# 19. DPI 与坐标说明

P2-09 只输出：

```text
截图图像坐标
```

不关心屏幕坐标和 DPI。

坐标转换仍由：

```text
CoordinateMapper
```

负责。

这保持了职责隔离：

```text
TemplateMatcherRecognizer：只负责图像内位置
CoordinateMapper：负责图像坐标 → 屏幕坐标
MouseController：负责动作执行
```

# 23. 最终结论

P2-09 的核心目标是：

```text
将执行器的识别阶段从“固定中心点”升级为“真实模板匹配”。
```

P2-09 完成后，系统将具备：

```text
默认模板目录
模板资源加载
多模板扫描
OpenCV matchTemplate
阈值过滤
候选排序
模板中心点输出
RecognitionResult 标准输出
与现有执行闭环无缝衔接
```

执行器的识别链路将从：

```text
CenterPointRecognizer → 固定中心点
```

升级为：

```text
TemplateMatcherRecognizer → 根据截图内容识别目标位置
```

同时保持安全策略：

```text
未识别到目标不失败
默认 dry-run 不真实点击
```

这为后续需求奠定基础：

```text
P2-10 自动化任务配置与流程编排
    让用户配置“识别哪个模板、执行什么动作、执行几轮”

P2-xx 识别调试面板
    可视化模板匹配结果、置信度、候选框

P2-xx 模板配置文件与分组
    管理模板名称、阈值、分组、优先级

P2-xx ROI 与多目标识别
    提升识别性能与稳定性
```
