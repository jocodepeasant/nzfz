## 一、版本说明

### 1.1 文档目标

本文档基于《逆战塔防工具 · 执行器界面初版 v1.0》继续扩展，目标是将原先 mock 的窗口搜索、连接、断开、健康检测功能升级为真实 Windows 游戏窗口连接能力。

本版本重点实现：

- 真实搜索 Windows 可见窗口
- 根据窗口标题、进程名、PID 进行匹配
- 连接指定游戏窗口
- 保存窗口句柄 `hwnd`
- 保存进程信息、窗口矩形、客户区矩形
- 连接成功后激活目标窗口
- 不允许连接最小化窗口
- 周期性健康检测
- 支持断开连接
- 为后续“后台命令传递 / 输入控制 / 截图识别”预留架构

## 五、核心概念定义

### 5.1 窗口连接的定义

本工具中的“连接游戏窗口”定义为：

```text
工具识别并绑定一个目标 Windows 窗口，保存该窗口的 hwnd、PID、标题、进程名、窗口矩形、客户区矩形，并持续检测其有效性。
```

连接成功后，后续功能均基于该窗口执行，包括但不限于：

- 截图
- 坐标换算
- 图像识别
- 输入控制
- 命令分发
- 自动化操作

---

### 5.2 `hwnd` 的作用

`hwnd` 是 Windows 窗口句柄，是后续所有窗口操作的核心标识。

本版本必须在搜索结果和连接状态中保存 `hwnd`。

典型用途：

```python
win32gui.IsWindow(hwnd)
win32gui.GetWindowText(hwnd)
win32gui.GetWindowRect(hwnd)
win32gui.GetClientRect(hwnd)
win32gui.SetForegroundWindow(hwnd)
```

---

### 5.3 窗口矩形与客户区矩形

窗口矩形：

```text
包含标题栏、边框、客户区在内的整个窗口区域。
```

客户区矩形：

```text
游戏画面实际渲染区域，通常不包含标题栏和窗口边框。
```

后续截图、点击、坐标换算应优先基于客户区矩形。

    left: int

### 6.5 `ControlMode`

本版本暂不实现命令控制，但需要预留控制模式。

```python
from enum import Enum


class ControlMode(Enum):
    FOREGROUND = "foreground"
    BACKGROUND = "background"
    HYBRID = "hybrid"
```

说明：

| 模式 | 说明 |
|---|---|
| `FOREGROUND` | 前台控制，需要激活窗口，兼容性较好，但会影响用户当前操作 |
| `BACKGROUND` | 后台控制，不主动激活窗口，尝试通过窗口消息控制，游戏中不保证有效 |
| `HYBRID` | 混合控制，优先后台，失败时提示或切换前台 |

本版本默认：

```python
ControlMode.FOREGROUND
```

## 八、窗口搜索功能设计

### 8.1 接口定义

```python
def search_windows(self, keyword: str) -> list[WindowInfo]:
    """
    搜索匹配关键词的可见窗口。

    匹配范围：
    - 窗口标题
    - 进程名称
    - PID

    Args:
        keyword: 用户输入的搜索关键词

    Returns:
        匹配的 WindowInfo 列表，按 match_score 降序排列。
    """
```

---

### 8.2 搜索流程

```text
1. 清理 keyword 前后空格
2. keyword 为空时返回空列表
3. 调用 Windows API 枚举所有顶层窗口
4. 对每个窗口进行基础过滤
5. 获取窗口标题
6. 获取窗口 PID
7. 通过 psutil 获取进程名
8. 获取窗口矩形
9. 判断是否可见
10. 判断是否最小化
11. 计算匹配度
12. 匹配度大于 0 才加入结果
13. 按 match_score 降序排序
14. 返回结果列表
```

---

### 8.3 窗口基础过滤规则

枚举窗口时，默认过滤掉以下窗口：

| 条件 | 处理 |
|---|---|
| 不是有效窗口 | 跳过 |
| 不可见窗口 | 跳过 |
| 无标题窗口 | 跳过 |
| 宽度小于 `200` | 跳过 |
| 高度小于 `150` | 跳过 |
| 获取不到 PID | 跳过 |
| 获取不到进程名 | 跳过 |

伪代码：

```python
if not win32gui.IsWindow(hwnd):
    return

if not win32gui.IsWindowVisible(hwnd):
    return

title = win32gui.GetWindowText(hwnd).strip()
if not title:
    return

left, top, right, bottom = win32gui.GetWindowRect(hwnd)
width = right - left
height = bottom - top

if width < 200 or height < 150:
    return
```

---

### 8.4 匹配范围

关键词匹配以下内容：

| 匹配对象 | 匹配方式 |
|---|---|
| 窗口标题 | 忽略大小写，包含匹配 |
| 进程名 | 忽略大小写，包含匹配 |
| PID | 数字关键词时，精确匹配 |

---

### 8.5 匹配度规则

```python
def calculate_match_score(keyword: str, title: str, process_name: str, pid: int) -> float:
    keyword = keyword.lower().strip()
    title_lower = title.lower()
    process_lower = process_name.lower()

    if keyword.isdigit() and int(keyword) == pid:
        return 1.0

    if title_lower == keyword:
        return 1.0

    if process_lower == keyword:
        return 0.95

    if keyword in title_lower:
        return 0.85

    if keyword in process_lower:
        return 0.8

    return 0.0
```

---

### 8.6 搜索结果排序

搜索结果必须按：

```python
match_score
```

降序排列。

如果分数相同，可按窗口标题升序或保持枚举顺序。

---

### 8.7 搜索结果展示

UI 表格仍显示三列：

```text

```

不直接显示 `hwnd`。

### 9.3 连接校验流程

连接时必须重新校验搜索结果中的窗口。

原因：

```text
搜索结果只是某一时刻的快照。
用户点击连接时，目标窗口可能已关闭、隐藏、最小化或 PID 已变化。
```

校验顺序：

```text
1. hwnd 是否有效
2. 进程是否存活
3. hwnd 当前 PID 是否仍然等于 WindowInfo.pid
4. 窗口是否可见
5. 窗口是否最小化
6. 窗口尺寸是否有效
7. 获取窗口矩形
8. 获取客户区矩形
9. 激活目标窗口
10. 校验激活是否成功
11. 创建 ConnectedWindow
12. 保存内部连接状态
13. 返回连接成功
```

---

### 9.4 句柄有效性校验

```python
if not win32gui.IsWindow(window.hwnd):
    return ConnectResult(
        success=False,
        error_message="窗口句柄已失效"
    )
```

---

### 9.5 进程存活校验

```python
if not psutil.pid_exists(window.pid):
    return ConnectResult(
        success=False,
        error_message="进程已退出"

```

### 9.6 PID 一致性校验

通过：

```python
_, current_pid = win32process.GetWindowThreadProcessId(window.hwnd)
```

如果：

```python
current_pid != window.pid
```

则连接失败：

```text
窗口所属进程已变化
```

### 9.8 最小化校验

本版本明确要求：

```text
窗口最小化时不允许连接。
```

因此：

```python
if win32gui.IsIconic(window.hwnd):
    return ConnectResult(
        success=False,
        error_message="窗口已最小化，请恢复窗口后重试"
    )
```

即使 `ConnectOptions.restore_if_minimized=True`，本版本也暂不启用自动恢复能力。

---

### 9.9 窗口尺寸校验

```python
left, top, right, bottom = win32gui.GetWindowRect(window.hwnd)

width = right - left
height = bottom - top

if width <= 0 or height <= 0:
    return ConnectResult(
        success=False,
        error_message="窗口尺寸异常"
    )
```

建议最低尺寸限制：

```text
宽度 >= 200
高度 >= 150
```

---

### 9.10 客户区矩形获取

Windows API：

```python
client_left, client_top, client_right, client_bottom = win32gui.GetClientRect(hwnd)
```

`GetClientRect` 返回的是窗口内部相对坐标，需要转换成屏幕坐标：

```python
screen_left_top = win32gui.ClientToScreen(hwnd, (client_left, client_top))
screen_right_bottom = win32gui.ClientToScreen(hwnd, (client_right, client_bottom))
```

最终创建：

```python
client_rect = WindowRect(
    left=screen_left_top[0],
    top=screen_left_top[1],
    right=screen_right_bottom[0],
    bottom=screen_right_bottom[1],
)
```

## 十、连接后激活窗口设计

### 10.1 激活要求

本版本需求规定：

```text
连接成功前，必须尝试激活目标窗口。
```

如果激活失败，默认认为连接失败。

---

### 10.2 激活流程

```text
1. 如果窗口最小化，直接失败，不自动恢复
2. 调用 SetForegroundWindow(hwnd)
3. 短暂等待
4. 调用 GetForegroundWindow()
5. 如果前台窗口等于目标 hwnd，则激活成功
6. 否则激活失败
```

---

### 10.3 建议实现

```python
import time
import win32gui


def activate_window(hwnd: int) -> bool:
    try:
        win32gui.SetForegroundWindow(hwnd)
        time.sleep(0.1)
        return win32gui.GetForegroundWindow() == hwnd
    except Exception:
        return False
```

---

### 10.4 激活失败处理

激活失败时，返回：

```python
ConnectResult(
    success=False,
    error_message="无法激活窗口，请尝试手动点击游戏窗口或以管理员身份运行工具",
    activated=False
)
```

---

### 10.5 权限说明

如果目标游戏以管理员权限运行，而工具不是管理员权限运行，可能导致：

- 无法激活窗口
- 后续无法发送输入
- 后续截图异常
- 部分 Windows API 调用失败

本版本不实现自动提权，只提示用户：

```text
请尝试以管理员身份运行工具。
```

## 十三、健康检测功能设计

### 13.1 接口定义

```python
def check_health(self) -> HealthCheckResult:
    """
    检查当前连接窗口是否正常。

    Returns:
        HealthCheckResult
    """
```

---

### 13.2 检测周期

UI 层连接成功后启动 `QTimer`：

```text
每 1 秒执行一次
```

调用：

```python
window_manager.check_health()
```

---

### 13.3 检测流程

```text
1. 如果当前没有连接窗口，返回 NOT_CONNECTED
2. 检查 hwnd 是否有效
3. 检查 PID 是否存活
4. 检查 hwnd 当前 PID 是否一致
5. 检查窗口是否可见
6. 检查窗口是否最小化
7. 检查窗口尺寸是否有效
8. 刷新窗口标题
9. 刷新窗口矩形
10. 刷新客户区矩形
11. 更新 _connected_window
12. 返回 HEALTHY
```

---

### 13.4 状态返回规则

| 检查结果 | 返回状态 | message |
|---|---|---|
| 当前无连接 | `NOT_CONNECTED` | `未连接` |
| hwnd 无效 | `HANDLE_INVALID` | `窗口句柄已失效` |
| PID 不存在 | `PROCESS_DEAD` | `进程已退出` |
| hwnd 当前 PID 不一致 | `HANDLE_INVALID` | `窗口所属进程已变化` |
| 窗口不可见 | `WINDOW_HIDDEN` | `窗口不可见` |
| 窗口最小化 | `WINDOW_MINIMIZED` | `窗口已最小化` |
| 尺寸异常 | `WINDOW_SIZE_INVALID` | `窗口尺寸异常` |
| 未知异常 | `UNKNOWN` | 异常详情 |
| 正常 | `HEALTHY` | `连接正常` |

---

### 13.5 健康检测发现异常后的 UI 行为

| 状态 | UI 状态 |
|---|---|
| `HEALTHY` | `CONNECTED` |
| `HANDLE_INVALID` | `ABNORMAL` |
| `PROCESS_DEAD` | `ABNORMAL` |
| `WINDOW_HIDDEN` | `ABNORMAL` |
| `WINDOW_MINIMIZED` | `ABNORMAL` |
| `WINDOW_SIZE_INVALID` | `ABNORMAL` |
| `UNKNOWN` | `ABNORMAL` |
| `NOT_CONNECTED` | `DISCONNECTED` |

异常状态下状态文字显示：

```text
连接异常：{message}
```

---

### 13.6 健康恢复

如果处于 `ABNORMAL` 状态，但下一次健康检测返回 `HEALTHY`，UI 应自动恢复为：

```text
已连接
```

窗口信息栏刷新为最新窗口信息。

---

### 13.7 自动重连策略

本版本不实现复杂自动重连。

健康检测只做：

```text
检测当前 hwnd 是否仍可用，并在可恢复异常消失后恢复 CONNECTED 状态。
```

不做：

- 进程退出后的自动搜索重连
- 句柄失效后的自动匹配新窗口
- 游戏重启后的自动重连
- 多候选窗口自动选择

## 十五、连接流程 UI 设计

### 15.1 点击「连接」按钮

流程：

```text
1. 获取当前选中 WindowInfo
2. 如果没有有效选中，直接返回
3. 如果当前未连接，进入连接流程
4. 如果当前已连接或连接异常，弹出确认切换连接对话框
```

---

### 15.2 切换连接确认对话框

当当前已有连接时，点击连接新窗口弹出：

```text
标题：确认切换连接
内容：当前已连接「{窗口标题}」，确认后将断开并连接新窗口，是否继续？
按钮：确认 / 取消
```

用户选择：

| 操作 | 行为 |
|---|---|
| 取消 | 不做任何操作 |
| 确认 | 先断开当前连接，再连接新窗口 |

---

### 15.3 连接中状态

进入连接流程后：

```text
1. UI 切换到 CONNECTING
2. 状态显示：🟡 连接中…
3. 隐藏窗口信息栏
4. 禁用连接按钮
5. 禁用断开连接按钮
6. 启动 5 秒超时计时器
7. 在子线程调用 WindowManager.connect_window()
```

---

### 15.4 连接线程

连接操作必须在子线程中执行，避免 UI 卡顿。

推荐使用：

```text
QThread + QObject Worker
```

Worker 负责调用：

```python
self.window_manager.connect_window(window_info)
```

返回：

```python
ConnectResult
```

---

### 15.5 连接超时

超时时间：

```text
5 秒
```

如果 5 秒内连接线程未返回：

```text
1. UI 切换到 TIMEOUT
2. 状态显示：🔴 连接超时，请重试
3. 停止等待当前连接结果
4. 禁用断开连接按钮
5. 根据当前选中行决定是否启用连接按钮
```

---

### 15.6 连接请求 ID

为避免旧线程结果覆盖新状态，UI 层必须维护连接请求 ID。

```python
self._connect_request_id += 1
request_id = self._connect_request_id
```

连接结果返回时：

```python
if request_id != self._connect_request_id:
    return
```

场景示例：

```text
第 1 次连接开始
5 秒超时
用户再次点击连接，开始第 2 次连接
第 1 次线程晚返回
此时必须忽略第 1 次结果
```

---

### 15.7 连接成功处理

当 `ConnectResult.success == True`：

```text
1. 停止连接超时计时器
2. 保存当前连接窗口信息
3. UI 切换到 CONNECTED
4. 状态显示：🟢 已连接
5. 窗口信息栏显示：
   窗口标题：xxx  PID：xxx  分辨率：宽×高
6. 启用断开连接按钮
7. 根据选中情况刷新连接按钮
8. 启动健康检测定时器
```

---

### 15.8 连接失败处理

当 `ConnectResult.success == False`：

```text
1. 停止连接超时计时器
2. UI 切换到 DISCONNECTED
3. 状态显示：
   🔴 连接失败：{error_message}
4. 隐藏窗口信息栏
5. 禁用断开连接按钮
6. 根据选中情况刷新连接按钮
```

## 十九、后台命令能力预留设计

### 19.1 背景说明

后续版本需要支持：

```text
工具在不与用户当前操作冲突的情况下，向游戏窗口或游戏进程传递命令。
```

但需要明确：

```text
游戏窗口是否响应后台命令，取决于游戏本身的输入机制。
```

许多 DirectX 游戏使用：

- Raw Input
- DirectInput
- XInput
- 底层设备状态
- 独占输入

因此，普通后台窗口消息不一定有效。

---

### 19.2 本版本处理策略

v1.1 不实现后台命令，只预留架构。

本版本只保证：

```text
建立稳定的窗口连接会话，为后续命令系统提供 hwnd、pid、窗口矩形、客户区矩形等基础信息。
```

---

### 19.3 未来控制模式

预留三种控制模式：

```text
FOREGROUND：前台控制
BACKGROUND：后台控制
HYBRID：混合控制
```

说明：

| 模式 | 是否激活窗口 | 是否影响用户操作 | 游戏兼容性 |
|---|---:|---:|---:|
| 前台控制 | 是 | 会影响 | 较高 |
| 后台控制 | 否 | 理论上不影响 | 不保证 |
| 混合控制 | 视情况 | 尽量减少影响 | 中等 |

---

### 19.4 未来命令模型预留

后续可新增：

```python
from dataclasses import dataclass
from enum import Enum


class GameCommandType(Enum):
    KEY_PRESS = "key_press"

    KEY_UP = "key_up"

    MOUSE_MOVE = "mouse_move"
    ACTIVATE_WINDOW = "activate_window"


@dataclass
class GameCommand:
    command_type: GameCommandType
    payload: dict
    require_foreground: bool = False
    timeout_ms: int = 1000


@dataclass
class CommandResult:
    success: bool
    message: str = ""
    backend: str = ""
```

---

### 19.5 未来输入通道预留

后续可新增：

```text
core/input/base.py
core/input/foreground_input.py
core/input/background_message_input.py
```

#### 前台输入通道

```text
通过 SendInput 等方式发送真实输入。
优点：游戏兼容性较高。
缺点：会影响用户当前鼠标键盘。
```

#### 后台消息通道

```text
通过 PostMessage / SendMessage 向 hwnd 发送窗口消息。
优点：理论上不抢焦点，不移动鼠标。
缺点：很多游戏不响应。
```

---

### 19.6 产品表述限制

后续需求中不应承诺：

```text
后台命令一定可用。
```

建议表述为：

```text
支持后台控制通道，优先通过窗口消息向目标窗口发送命令；若目标游戏不响应后台消息，则提示用户切换为前台控制模式。
```

```text
健康检测返回 WINDOW_MINIMIZED。
UI 显示：连接异常：窗口已最小化。
```

用户恢复窗口后，下一次健康检测应恢复为 `CONNECTED`。

---

### 20.4 游戏窗口隐藏

健康检测返回：

```text
WINDOW_HIDDEN
```

UI 显示：

```text
连接异常：窗口不可见
```

---

### 20.5 游戏窗口尺寸变化

本版本策略：

```text
尺寸变化不视为异常。
健康检测中刷新窗口矩形和客户区矩形即可。
```

仅当尺寸无效时返回：

```text
WINDOW_SIZE_INVALID
```

---

### 20.6 DPI 缩放

本版本预留字段：

```python
dpi_scale: float = 1.0
```

暂不强制实现复杂 DPI 感知。

后续截图和点击功能实现时，需要专项处理：

- Windows DPI Awareness
- 125% / 150% 缩放
- 多显示器 DPI 不一致

- [ ] 目标窗口关闭后点击连接，提示 `窗口句柄已失效`

- [ ] 目标窗口不可见时，提示 `窗口不可见`

### 22.4 断开连接验收

- [ ] 未连接时断开按钮禁用

- [ ] 点击断开连接不会最小化游戏窗口

### 22.5 健康检测验收

- [ ] 连接成功后每 1 秒执行健康检测
- [ ] 目标窗口正常时状态保持 `已连接`
- [ ] 目标窗口关闭后状态显示 `连接异常：窗口句柄已失效` 或 `连接异常：进程已退出`

### 22.6 超时验收

- [ ] 连接超过 5 秒未返回时，状态显示 `连接超时，请重试`

## 二十四、关键设计结论

本版本最终设计结论如下：

| 设计项 | 结论 |
|---|---|
| 平台 | 仅 Windows |
| 依赖 | `pywin32` + `psutil` |
| 后台接口形态 | `WindowManager` 类 |
| 搜索结果是否包含 `hwnd` | 必须包含 |
| 窗口最小化是否允许连接 | 不允许 |
| 连接后是否激活窗口 | 必须激活 |
| 激活失败是否连接成功 | 默认失败 |
| 健康检测频率 | 每 1 秒 |
| 后台命令是否本版本实现 | 不实现，仅预留 |
| 是否承诺后台命令对游戏一定有效 | 不承诺 |
