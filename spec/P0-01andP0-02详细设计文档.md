## 2. 模块边界

### 2.1 `models.py` 职责

`models.py` 是纯数据模型层，只负责定义窗口连接相关数据结构。

允许包含：

- `dataclass`
- `Enum`
- 简单属性计算
- 简单便捷方法
- 类型注解

不允许包含：

- Windows API 调用
- `win32gui`
- `win32process`
- `psutil`
- PySide6 / Qt 代码
- UI 状态逻辑
- 线程逻辑
- 搜索窗口逻辑
- 连接窗口逻辑
- 健康检测业务逻辑

---

### 2.2 `window_manager.py` 职责

`WindowManager` 是执行器窗口连接核心管理类。

负责：

- 保存当前连接窗口状态
- 保存最近一次连接使用的搜索结果
- 保存最近错误信息
- 提供统一公开接口
- 屏蔽 Windows API 细节
- 提供非 Windows 平台安全提示
- 为后续真实窗口搜索、连接、健康检测提供扩展点

不负责：

- GUI 状态切换
- 按钮启用/禁用
- 表格渲染
- `QThread`
- `QTimer`
- 自动点击
- 截图识别
- 后台输入
- 自动重连
- 多窗口会话管理

## 4. `WindowRect` 设计

### 4.1 语义

`WindowRect` 表示一个矩形区域。

在当前窗口连接模块中主要用于：

- 窗口矩形
- 客户区矩形

坐标约定：

```text
left / top / right / bottom 均使用屏幕坐标。
width = right - left
height = bottom - top
```

---

### 4.2 设计要求

`WindowRect` 使用：

```python
@dataclass(frozen=True)
```

原因：

1. 它是典型值对象；
2. 不应被 UI 或其他模块原地修改；
3. 健康检测刷新窗口尺寸时，应创建新的 `WindowRect`；
4. 便于减少隐式副作用。

---

### 4.3 字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `left` | `int` | 左上角屏幕 X 坐标 |
| `top` | `int` | 左上角屏幕 Y 坐标 |
| `right` | `int` | 右下角屏幕 X 坐标 |
| `bottom` | `int` | 右下角屏幕 Y 坐标 |

---

### 4.4 属性与方法

| 名称 | 类型 | 说明 |
|---|---|---|
| `width` | `property` | 宽度 |
| `height` | `property` | 高度 |
| `size_text` | `property` | 例如 `1280×720` |
| `is_valid()` | `method` | 判断尺寸是否有效 |

---

## 5. `WindowInfo` 设计

### 5.1 语义

`WindowInfo` 表示搜索结果中的窗口快照。

它来自：

```python
WindowManager.search_windows(keyword)
```

注意：

```text
WindowInfo 只是搜索时某一瞬间的快照，不代表窗口当前仍然有效。
```

用户点击连接时，必须重新校验：

- `hwnd` 是否有效
- PID 是否仍一致
- 进程是否存活
- 窗口是否可见
- 窗口是否最小化
- 窗口尺寸是否有效

---

### 5.2 设计要求

`WindowInfo` 使用：

```python
@dataclass(frozen=True)
```

原因：

1. 搜索结果不应被 UI 原地修改；
2. 表格中通过 `Qt.UserRole` 保存该对象时更安全；
3. 连接线程使用时不容易被外部改动；
4. 符合“搜索快照”的语义。

---

### 5.3 字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `hwnd` | `int` | Windows 窗口句柄 |
| `title` | `str` | 窗口标题 |
| `process_name` | `str` | 进程名 |
| `pid` | `int` | 进程 ID |
| `width` | `int` | 搜索时窗口宽度 |
| `height` | `int` | 搜索时窗口高度 |
| `match_score` | `float` | 匹配分数 |
| `is_visible` | `bool` | 搜索时是否可见 |
| `is_minimized` | `bool` | 搜索时是否最小化 |

---

### 5.4 属性

| 名称 | 说明 |
|---|---|
| `size_text` | 显示尺寸，例如 `1280×720` |
| `display_name` | 显示名称，例如 `逆战 (nz.exe, PID: 1234)` |

## 6. `ConnectedWindow` 设计

### 6.1 语义

`ConnectedWindow` 表示当前已经连接成功的窗口上下文。

它是后续模块的核心依赖，包括：

- 截图
- 客户区坐标换算
- 图像识别
- 前台输入
- 后台命令预留
- 自动化执行前置检查

---

### 6.2 设计要求

`ConnectedWindow` 使用普通：

```python
@dataclass
```

不使用 `frozen=True`。

原因：

1. 健康检测中需要刷新窗口标题、窗口矩形、客户区矩形；
2. 后续实现可以选择原地更新，也可以整体替换；
3. P0/P1 阶段保持实现灵活性。

推荐健康检测时使用“整体替换”方式刷新：

```python
self._connected_window = ConnectedWindow(...)
```

但当前设计不强制。

---

### 6.3 字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `hwnd` | `int` | 当前连接窗口句柄 |
| `title` | `str` | 当前窗口标题 |
| `process_name` | `str` | 当前进程名 |
| `pid` | `int` | 当前进程 ID |
| `window_rect` | `WindowRect` | 整个窗口矩形，屏幕坐标 |
| `client_rect` | `WindowRect` | 客户区矩形，屏幕坐标 |
| `dpi_scale` | `float` | DPI 缩放比例，当前默认 `1.0` |
| `connected_at` | `float` | 连接时间戳 |

---

### 6.4 `connected_at` 默认值

采用：

```python
field(default_factory=time.time)
```

而不是固定默认值：

```python
connected_at: float = 0.0
```

原因：

1. 防止创建连接对象时忘记设置时间；
2. 语义更自然；
3. 每个连接对象都会自动记录创建时间。

## 7. `HealthStatus` 设计

### 7.1 语义

`HealthStatus` 表示当前连接窗口的健康状态。

---

### 7.2 枚举值

保持需求文档中的方案 A：

```python
class HealthStatus(Enum):
    HEALTHY = "healthy"
    NOT_CONNECTED = "未连接"
    HANDLE_INVALID = "窗口句柄已失效"
    PROCESS_DEAD = "进程已退出"
    WINDOW_HIDDEN = "窗口不可见"
    WINDOW_MINIMIZED = "窗口已最小化"
    WINDOW_SIZE_INVALID = "窗口尺寸异常"
    UNKNOWN = "未知异常"
```

---

### 7.3 使用约束

虽然枚举值中包含中文，但 UI 层不应强依赖：

```python
result.status.value
```

推荐 UI 层优先使用：

```python
result.message
```

原因：

1. `message` 更适合作为展示文案；
2. 后续如果需要统一枚举值为英文，不会影响 UI；
3. 错误详情可以通过 `message` 携带。

## 8. `ControlMode` 设计

### 8.1 语义

`ControlMode` 是后续输入控制模式预留。

当前版本只定义，不实现后台控制。

---

### 8.2 枚举值

```python
class ControlMode(Enum):
    FOREGROUND = "foreground"
    BACKGROUND = "background"
    HYBRID = "hybrid"
```

---

### 8.3 当前版本行为

当前版本默认：

```python
ControlMode.FOREGROUND
```

即：

```text
连接成功前必须尝试激活窗口。
```

即使后续调用方传入：

```python
ControlMode.BACKGROUND
```

当前版本也不应因此跳过激活逻辑。

## 10. `ConnectResult` 设计

### 10.1 语义

`ConnectResult` 是连接窗口操作的返回结果。

它用于 Core 层向 UI 层传递：

- 是否成功
- 成功后的连接窗口
- 失败原因
- 是否成功激活窗口

---

### 10.2 字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `success` | `bool` | 是否连接成功 |
| `window` | `Optional[ConnectedWindow]` | 成功连接后的窗口对象 |
| `error_message` | `str` | 失败原因 |
| `activated` | `bool` | 是否成功激活目标窗口 |

---

### 10.3 便捷工厂方法

允许增加：

```python
ConnectResult.ok()
ConnectResult.fail()
```

用于减少后续重复代码。

示例：

```python
return ConnectResult.fail("窗口句柄已失效")
```

### 11.1 语义

`HealthCheckResult` 是健康检测结果。

用于：

```python

```

---

### 11.2 字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `status` | `HealthStatus` | 健康状态 |
| `message` | `str` | 状态描述 |
| `window` | `Optional[ConnectedWindow]` | 刷新后的窗口对象 |

---

### 11.3 便捷属性

允许增加：

```python
is_healthy
is_connected
```

含义：

```python
result.is_healthy
```

等价于：

```python
result.status == HealthStatus.HEALTHY
```

`is_connected` 表示当前状态不是 `NOT_CONNECTED`。

注意：

```text
is_connected=True 不代表健康，只代表存在连接上下文或异常连接状态。
```

## 12. `models.py` 最终代码

```python
from __future__ import annotations

import time
from dataclasses import dataclass, field
from enum import Enum
from typing import Optional


@dataclass(frozen=True)
class WindowRect:
    """
    Windows 窗口矩形。

    坐标约定：
    - left/top/right/bottom 均为屏幕坐标
    - width = right - left
    - height = bottom - top
    """

    left: int
    top: int
    right: int
    bottom: int

    @property
    def width(self) -> int:
        return self.right - self.left

    @property
    def height(self) -> int:
        return self.bottom - self.top

    @property
    def size_text(self) -> str:
        return f"{self.width}×{self.height}"

    def is_valid(self, min_width: int = 1, min_height: int = 1) -> bool:
        return self.width >= min_width and self.height >= min_height


@dataclass(frozen=True)
class WindowInfo:
    """
    搜索结果中的窗口信息。

    注意：
    - 这是搜索时的窗口快照
    - 不代表窗口当前仍然有效
    - 连接前必须重新校验 hwnd、pid、可见性、最小化状态和尺寸
    """

    hwnd: int
    title: str
    process_name: str
    pid: int
    width: int
    height: int
    match_score: float
    is_visible: bool = True
    is_minimized: bool = False

    @property
    def size_text(self) -> str:
        return f"{self.width}×{self.height}"

    @property
    def display_name(self) -> str:
        return f"{self.title} ({self.process_name}, PID: {self.pid})"


@dataclass
class ConnectedWindow:
    """
    当前已连接窗口上下文。

    后续截图、坐标换算、输入控制、自动化执行都应基于该对象。
    """

    hwnd: int
    title: str
    process_name: str
    pid: int
    window_rect: WindowRect
    client_rect: WindowRect
    dpi_scale: float = 1.0
    connected_at: float = field(default_factory=time.time)

    @property
    def window_size_text(self) -> str:
        return self.window_rect.size_text

    @property
    def client_size_text(self) -> str:
        return self.client_rect.size_text

    @property
    def display_name(self) -> str:
        return f"{self.title} ({self.process_name}, PID: {self.pid})"


class HealthStatus(Enum):
    HEALTHY = "healthy"
    NOT_CONNECTED = "未连接"
    HANDLE_INVALID = "窗口句柄已失效"
    PROCESS_DEAD = "进程已退出"
    WINDOW_HIDDEN = "窗口不可见"
    WINDOW_MINIMIZED = "窗口已最小化"
    WINDOW_SIZE_INVALID = "窗口尺寸异常"
    UNKNOWN = "未知异常"


class ControlMode(Enum):
    FOREGROUND = "foreground"
    BACKGROUND = "background"
    HYBRID = "hybrid"


@dataclass(frozen=True)
class ConnectOptions:
    """
    窗口连接选项。

    当前版本固定前台连接语义：
    - 默认连接时激活窗口
    - 最小化窗口不自动恢复
    - 当前版本不因 control_mode=BACKGROUND 跳过激活逻辑
    """

    activate_on_connect: bool = True
    restore_if_minimized: bool = False
    control_mode: ControlMode = ControlMode.FOREGROUND


@dataclass
class ConnectResult:
    """
    窗口连接结果。
    """

    success: bool
    window: Optional[ConnectedWindow] = None
    error_message: str = ""
    activated: bool = False

    @classmethod
    def ok(
        cls,
        window: ConnectedWindow,
        activated: bool = True,
    ) -> "ConnectResult":
        return cls(
            success=True,
            window=window,
            error_message="",
            activated=activated,
        )

    @classmethod
    def fail(
        cls,
        message: str,
        activated: bool = False,
    ) -> "ConnectResult":
        return cls(
            success=False,
            window=None,
            error_message=message,
            activated=activated,
        )


@dataclass
class HealthCheckResult:
    """
    窗口健康检测结果。
    """

    status: HealthStatus
    message: str = ""
    window: Optional[ConnectedWindow] = None

    @property
    def is_healthy(self) -> bool:
        return self.status == HealthStatus.HEALTHY

    @property
    def is_connected(self) -> bool:
        return self.status != HealthStatus.NOT_CONNECTED
```

## 16. 非 Windows 平台安全设计

### 16.1 设计目标

即使在以下环境中运行：

```text
macOS
Linux
未安装 pywin32 的环境
未安装 psutil 的环境
```

程序也不应在导入阶段崩溃。

---

### 16.2 安全导入策略

`window_manager.py` 不直接无条件导入：

```python
import win32gui
import win32process
import psutil
```

而是使用条件导入：

```python
IS_WINDOWS = platform.system() == "Windows"

if IS_WINDOWS:
    try:
        import psutil
        import win32gui
        import win32process
    except ImportError:
        psutil = None
        win32gui = None
        win32process = None
else:
    psutil = None
    win32gui = None
    win32process = None
```

---

### 16.3 平台检查方法

提供私有方法：

```python
def _ensure_supported(self) -> Optional[str]:
    ...
```

返回规则：

| 情况 | 返回 |
|---|---|
| 非 Windows | `当前窗口连接功能仅支持 Windows 10 / Windows 11` |
| 缺少依赖 | `缺少窗口连接依赖，请安装 pywin32 和 psutil` |
| 环境正常 | `None` |

## 17. P0-02 阶段各方法行为

P0-02 是基础类阶段，真实窗口搜索、连接、健康检测完整逻辑会在后续需求中实现。

因此本阶段方法行为如下。

---

### 17.1 `search_windows(keyword)`

P0-02 阶段：

1. 检查平台和依赖；
2. 清理关键词；
3. 空关键词返回空列表；
4. 真实搜索逻辑在 P0-03 实现；
5. 当前阶段可返回空列表。

行为：

```python
error = self._ensure_supported()
if error:
    self._last_error = error
    return []

keyword = keyword.strip()
if not keyword:
    return []

return []
```

---

### 17.2 `connect_window(window, options=None)`

P0-02 阶段：

1. 检查平台和依赖；
2. 检查 `window` 是否为空；
3. 创建默认 `ConnectOptions`；
4. 真实连接逻辑在 P0-04~P0-07 实现；
5. 当前阶段返回“尚未实现”。

行为：


## 19. `window_manager.py` 最终代码

```python
from __future__ import annotations

import logging
import platform
from typing import Optional

from nzfz_executor.core.models import (
    WindowInfo,
    ConnectedWindow,
    ConnectOptions,
    ConnectResult,
    HealthCheckResult,
    HealthStatus,
)

logger = logging.getLogger(__name__)

IS_WINDOWS = platform.system() == "Windows"

if IS_WINDOWS:
    try:
        import psutil
        import win32gui
        import win32process
    except ImportError:
        psutil = None
        win32gui = None
        win32process = None
else:
    psutil = None
    win32gui = None
    win32process = None


class WindowManager:
    """
    Windows 游戏窗口连接管理器。

    职责：
    - 搜索真实 Windows 顶层窗口
    - 连接并绑定指定窗口
    - 保存当前连接窗口上下文
    - 断开连接
    - 健康检测

    不负责：
    - GUI 状态切换
    - QThread/QTimer
    - 自动点击
    - 截图识别
    - 后台命令
    - 自动重连
    - 多窗口会话管理
    """

    def __init__(self):
        self._connected_window: Optional[ConnectedWindow] = None
        self._last_window_info: Optional[WindowInfo] = None
        self._last_error: str = ""

    def search_windows(self, keyword: str) -> list[WindowInfo]:
        """
        搜索匹配关键词的可见窗口。

        匹配范围：
        - 窗口标题
        - 进程名称
        - PID

        P0-02 阶段仅保留接口和基础检查。
        真实 Windows API 搜索逻辑在 P0-03 阶段实现。
        """
        error = self._ensure_supported()
        if error:
            self._last_error = error
            logger.warning(error)
            return []

        keyword = keyword.strip()
        if not keyword:
            return []

        logger.info("开始搜索窗口，关键词：%s", keyword)

        # P0-03 阶段实现真实搜索逻辑
        logger.info("搜索窗口完成，结果数量：0")
        return []

    def connect_window(
        self,
        window: WindowInfo,
        options: Optional[ConnectOptions] = None,
    ) -> ConnectResult:
        """
        连接指定窗口。

        Args:
            window: 搜索结果中的窗口信息。
            options: 连接选项。

        Returns:
            ConnectResult

        P0-02 阶段仅保留接口和基础检查。
        真实连接逻辑在 P0-04 ~ P0-07 阶段实现。
        """
        error = self._ensure_supported()
        if error:
            self._last_error = error
            logger.warning(error)
            return ConnectResult.fail(error)

        if window is None:
            message = "未选择窗口"
            self._last_error = message
            logger.warning(message)
            return ConnectResult.fail(message)

        options = options or ConnectOptions()

        logger.info(
            "开始连接窗口：title=%s, process=%s, pid=%s, hwnd=%s, control_mode=%s",
            window.title,
            window.process_name,
            window.pid,
            window.hwnd,
            options.control_mode.value,
        )

        # P0-04 ~ P0-07 阶段实现真实连接逻辑
        message = "窗口连接功能尚未实现"
        self._last_error = message
        logger.warning(message)
        return ConnectResult.fail(message)

    def disconnect_window(self) -> None:
        """
        断开当前连接，清理内部状态。

        注意：
        - 不关闭游戏窗口
        - 不最小化游戏窗口
        - 不改变游戏前台状态
        - 不发送任何输入
        """
        if self._connected_window is not None:
            logger.info(
                "断开窗口连接：title=%s, pid=%s, hwnd=%s",
                self._connected_window.title,
                self._connected_window.pid,
                self._connected_window.hwnd,
            )
        else:
            logger.info("断开窗口连接：当前未连接")

        self._connected_window = None
        self._last_error = ""

    def check_health(self) -> HealthCheckResult:
        """
        检查当前连接窗口是否正常。

        P0-02 阶段：
        - 未连接返回 NOT_CONNECTED
        - 已连接暂时返回 HEALTHY

        真实健康检测逻辑在 P1-01 阶段实现。
        """
        if self._connected_window is None:
            return HealthCheckResult(
                status=HealthStatus.NOT_CONNECTED,
                message="未连接",
                window=None,
            )

        return HealthCheckResult(
            status=HealthStatus.HEALTHY,
            message="连接正常",
            window=self._connected_window,
        )

    def get_connected_window(self) -> Optional[ConnectedWindow]:
        """
        获取当前已连接窗口。

        Returns:
            未连接时返回 None。
        """
        return self._connected_window

    def get_last_error(self) -> str:
        """
        获取最近一次错误信息。
        """
        return self._last_error

    def get_last_window_info(self) -> Optional[WindowInfo]:
        """
        获取最近一次连接使用的搜索结果。
        """
        return self._last_window_info

    def is_supported(self) -> bool:
        """
        当前运行环境是否支持窗口连接能力。
        """
        return self._ensure_supported() is None

    def get_unsupported_reason(self) -> str:
        """
        获取当前运行环境不支持窗口连接能力的原因。
        """
        return self._ensure_supported() or ""

    def _ensure_supported(self) -> Optional[str]:
        """
        检查当前环境是否支持窗口连接能力。

        Returns:
            None 表示支持。
            str 表示不支持原因。
        """
        if not IS_WINDOWS:
            return "当前窗口连接功能仅支持 Windows 10 / Windows 11"

        if win32gui is None or win32process is None or psutil is None:
            return "缺少窗口连接依赖，请安装 pywin32 和 psutil"

        return None
```

## 24. 设计注意事项

### 24.1 UI 不应直接调用 Windows API

错误做法：

```python
win32gui.GetWindowText(hwnd)
```

出现在 UI 层。

正确做法：

```python
windows = self.window_manager.search_windows(keyword)
```

---

### 24.2 UI 不应从表格文本反推窗口对象

错误做法：

```python
title = self.result_table.item(row, 0).text()
pid = int(self.result_table.item(row, 2).text())
```

正确做法：

```python
window = item.data(Qt.UserRole)
```

---

### 24.3 断开连接不操作游戏窗口

`disconnect_window()` 只能清理执行器内部状态。

不允许：

```python
win32gui.CloseWindow(hwnd)
win32gui.ShowWindow(hwnd, ...)
win32gui.SetForegroundWindow(...)
```

---

### 24.4 Core 层不向 UI 抛业务异常

`WindowManager` 后续实现真实逻辑时，应尽量捕获异常，并转换为：

```python
ConnectResult.fail(...)
```

或：

```python
HealthCheckResult(
    status=HealthStatus.UNKNOWN,
    message=str(e),
)
```

避免 UI 层因为异常崩溃。

## 26. 最终结论

P0-01 / P0-02 的最终设计以“稳定地基”为目标：

1. **模型层纯净**：只放数据结构，不混入 Windows API 和 UI 逻辑；
2. **WindowManager 边界清晰**：Core 层统一封装窗口连接能力；
3. **非 Windows 平台安全**：不会因为导入 `pywin32` 直接崩溃；
4. **搜索快照与连接状态区分明确**：`WindowInfo` 不等于 `ConnectedWindow`；
5. **结果对象结构化**：连接与健康检测都通过结果对象向 UI 传递状态；
6. **后续扩展自然**：P0-03 到 P1-10 可以直接在当前骨架上继续实现；
7. **符合当前版本边界**：不提前实现截图、点击、后台输入、自动重连、多窗口管理。
