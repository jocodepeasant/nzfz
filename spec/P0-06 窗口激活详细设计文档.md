## 1. 版本说明

### 1.1 v1.2 修订目标

本文档用于定义 `WindowManager.connect_window()` 中的**目标窗口激活阶段**。

P0-06 的职责是：

```text
在连接前校验和连接上下文构建均通过后，尝试将目标窗口切换到前台，并确认它确实成为前台窗口。
```

只有 P0-06 激活成功后，连接流程才允许保存：

```python
self._connected_window
```

如果激活失败，则连接失败，并且不得覆盖旧连接状态。

---

## 2. 与整体连接流程的关系

完整流程：

```text
P0-04：连接前校验
    ↓
P0-05：构建 ConnectedWindow 候选对象
    ↓
P0-06：激活目标窗口
    ↓
保存 _connected_window
    ↓
返回 ConnectResult.success
```

P0-06 发生在：

```text
ConnectedWindow 候选对象构建成功之后
```

并且发生在：

```text
_connected_window 保存之前
```

---

## 3. 设计目标

P0-06 需要实现：

- 检查目标窗口是否已经是前台窗口；
- 如已是前台窗口，直接认为激活成功；
- 调用 Windows API 尝试置前；
- 使用 `GetForegroundWindow()` 校验激活结果；
- 失败时返回明确错误；
- 异常时返回明确错误；
- 激活失败时不保存新连接；
- 当前版本不支持跳过激活；
- 当前版本不支持后台连接；
- 当前版本不自动恢复最小化窗口；
- 记录基础日志。

## 6. 关键约束

### 6.1 当前版本必须激活窗口

当前版本默认控制模式为：

```python
ControlMode.FOREGROUND
```

连接成功前必须尝试激活目标窗口。

即使调用方传入：

```python
ConnectOptions(activate_on_connect=False)
```

当前版本也不跳过激活。

原因：

1. 当前版本尚未实现后台输入；
2. 后续自动化动作默认依赖前台窗口；
3. 连接成功但未激活会造成用户理解混乱；
4. 前台输入、截图、坐标换算都需要稳定目标窗口上下文；
5. 与“连接成功前激活目标窗口”的需求一致。

### 6.2 `ControlMode.BACKGROUND` 当前不生效

即使调用方传入：

```python
ConnectOptions(control_mode=ControlMode.BACKGROUND)
```

当前版本仍然执行前台激活流程。

日志建议在 P0-04 中记录，P0-06 不再重复判断控制模式。

### 6.3 最小化窗口不允许连接

P0-04 和 P0-05 已经负责拒绝最小化窗口。

P0-06 不负责自动恢复最小化窗口。

如果激活过程中发现窗口变为最小化，可以返回失败：

```text
激活失败：窗口已最小化，请恢复窗口后重试
```

### 6.4 激活成功必须以 `GetForegroundWindow()` 为准

不能只因为 `SetForegroundWindow()` 没有抛异常就认为激活成功。

必须调用：

```python
win32gui.GetForegroundWindow()
```

并确认返回值等于目标 `hwnd`。

## 8. 激活流程

### 8.1 标准流程

推荐顺序：

```text
1. 校验 hwnd > 0
2. 防御性调用 IsWindow(hwnd)
3. 防御性检查窗口是否最小化
4. 调用 GetForegroundWindow()
5. 如果当前前台窗口就是 hwnd，直接成功
6. 循环重试 2 次：
   6.1 调用 BringWindowToTop(hwnd)
   6.2 调用 SetForegroundWindow(hwnd)
   6.3 等待 0.1 秒
   6.4 调用 GetForegroundWindow()
   6.5 如果前台窗口是 hwnd，返回成功
7. 重试结束仍未成为前台窗口，返回失败
```

### 8.2 已在前台窗口处理

如果：

```python
win32gui.GetForegroundWindow() == hwnd
```

则直接返回：

```python
True, ""
```

无需再调用 `SetForegroundWindow()`。

### 8.3 重试策略

当前版本采用轻量重试：

```python
ACTIVATE_RETRY_COUNT = 2
ACTIVATE_RETRY_INTERVAL = 0.1
```

说明：

- 不进行长时间阻塞；
- 不使用复杂线程输入附加；
- 不做权限提升；
- 不做窗口恢复。

## 9. 错误处理设计

### 9.1 失败错误信息

| 场景                       | 错误信息建议                                                 |
| -------------------------- | ------------------------------------------------------------ |
| `hwnd <= 0`                | `激活失败：窗口句柄无效`                                     |
| `IsWindow(hwnd)` 为假      | `激活失败：窗口句柄已失效`                                   |
| 窗口已最小化               | `激活失败：窗口已最小化，请恢复窗口后重试`                   |
| `BringWindowToTop` 异常    | `激活失败：调用 BringWindowToTop 异常：{exception}`          |
| `SetForegroundWindow` 异常 | `激活失败：调用 SetForegroundWindow 异常：{exception}`       |
| 前台窗口不是目标窗口       | `激活失败：目标窗口未成为前台窗口，可能被系统限制或权限不足` |
| 未知异常                   | `激活失败：窗口激活异常：{exception}`                        |

### 9.2 Windows 前台限制说明

Windows 可能因为以下原因拒绝置前：

- 前台锁定规则；
- 权限等级不同；
- 目标进程权限更高；
- 用户当前正在操作其他窗口；
- 安全策略限制；
- 游戏窗口自身行为限制。

当前版本不使用：

- `AttachThreadInput`；
- 权限自动提升；
- 注入；
- Hook；
- 驱动级输入。

因此遇到系统限制时返回失败即可。

## 10. 推荐伪代码

```python
ACTIVATE_RETRY_COUNT = 2
ACTIVATE_RETRY_INTERVAL = 0.1


def _activate_window(self, hwnd: int) -> tuple[bool, str]:
    try:
        if hwnd <= 0:
            return False, "激活失败：窗口句柄无效"

        if not win32gui.IsWindow(hwnd):
            return False, "激活失败：窗口句柄已失效"

        if win32gui.IsIconic(hwnd):
            return False, "激活失败：窗口已最小化，请恢复窗口后重试"

        current_foreground = win32gui.GetForegroundWindow()
        if current_foreground == hwnd:
            logger.info("目标窗口已经是前台窗口: hwnd=%s", hwnd)
            return True, ""

        for index in range(ACTIVATE_RETRY_COUNT):
            logger.info(
                "尝试激活目标窗口: hwnd=%s, attempt=%s",
                hwnd,
                index + 1,
            )

            try:
                win32gui.BringWindowToTop(hwnd)
                win32gui.SetForegroundWindow(hwnd)
            except Exception as exc:
                logger.warning(
                    "调用窗口激活 API 异常: hwnd=%s, attempt=%s, error=%s",
                    hwnd,
                    index + 1,
                    exc,
                )

            time.sleep(ACTIVATE_RETRY_INTERVAL)

            current_foreground = win32gui.GetForegroundWindow()
            if current_foreground == hwnd:
                logger.info("目标窗口激活成功: hwnd=%s", hwnd)
                return True, ""

        logger.warning(
            "目标窗口激活失败，前台窗口不是目标窗口: hwnd=%s, foreground=%s",
            hwnd,
            win32gui.GetForegroundWindow(),
        )

        return (
            False,
            "激活失败：目标窗口未成为前台窗口，可能被系统限制或权限不足",
        )

    except Exception as exc:
        logger.exception("窗口激活异常")
        return False, f"激活失败：窗口激活异常：{exc}"
```

## 11. 与 `connect_window()` 的集成

推荐最终 `connect_window()` 主流程如下：

```python
def connect_window(
    self,
    window: WindowInfo,
    options: Optional[ConnectOptions] = None,
) -> ConnectResult:
    options = options or ConnectOptions()

    ok, error = self._validate_window_for_connect(window, options)
    if not ok:
        self._last_error = error
        return ConnectResult.fail(error)

    ok, error, connected_window = self._build_connected_window(window)
    if not ok:
        self._last_error = error
        return ConnectResult.fail(error)

    ok, error = self._activate_window(window.hwnd)
    if not ok:
        self._last_error = error
        return ConnectResult.fail(error)

    self._connected_window = connected_window
    self._last_window_info = window
    self._last_error = None

    return ConnectResult.success(connected_window)
```

注意：

- `_build_connected_window()` 成功后不得立即保存；
- `_activate_window()` 失败后不得保存；
- 只有所有步骤成功后才保存新连接；
- 连接失败不清空旧连接。

## 12. 状态保存语义

### 12.1 激活成功

激活成功后，`connect_window()` 才允许执行：

```python
self._connected_window = connected_window
self._last_window_info = window
self._last_error = None
```

然后返回：

```python
ConnectResult.success(connected_window)
```

### 12.2 激活失败

激活失败时：

```python
self._last_error = error
return ConnectResult.fail(error)
```

不得执行：

```python
self._connected_window = connected_window
```

不得清空旧连接。

## 15. 三份文档统一后的最终连接语义

最终三份文档应共同表达同一个主流程：

```text
P0-04：验证这个搜索结果现在还能不能连接。
P0-05：读取当前真实窗口状态，构建 ConnectedWindow 候选对象。
P0-06：把目标窗口激活到前台。
全部成功后：保存连接状态并返回连接成功。
任一步失败：返回失败，写入 _last_error，不清空旧连接。
```

最终状态保存只允许发生在：

```text
P0-04 校验成功
+
P0-05 构建成功
+
P0-06 激活成功
```

之后。
