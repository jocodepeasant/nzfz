# 5. 截图按钮启用规则

## 5.1 启用条件

“刷新截图”按钮启用条件：

```text
1. 当前存在已连接上下文
2. 当前不在连接中
3. 当前不在截图中

```

---

## 5.2 推荐状态规则

| 连接 UI 状态 | 是否允许刷新截图 | 说明 |
|---|---:|---|
| `DISCONNECTED` | 否 | 未连接 |
| `CONNECTING` | 否 | 正在连接 |
| `CONNECTED_READY` | 是 | 推荐状态 |
| `CONNECTED_NOT_READY` | 是 | 可用于诊断 |
| `CONNECTED_UNHEALTHY` | 是/可尝试 | Core 会返回具体失败原因 |

---

## 5.3 设计原则

截图是诊断能力，不完全等同于执行能力。

因此：

```text
只要存在 connected_context，手动截图可以尝试。
```

但如果窗口：

```text
最小化
不可见
hwnd 无效
```

则由 Core 返回失败结果。

# 7. ScreenshotCaptureWorker 设计

## 7.1 职责

`ScreenshotCaptureWorker` 负责在线程中执行一次截图任务。

它只做：

```text
调用 ScreenshotManager.capture()
将结果通过 signal 返回
捕获异常并返回 failed
```

---

## 7.2 信号设计

```python
finished = Signal(int, object)
failed = Signal(int, str)
```

含义：

| 信号 | 参数 | 说明 |
|---|---|---|
| `finished` | `request_id, ScreenshotResult` | 截图调用完成 |
| `failed` | `request_id, message` | Worker 异常 |

---

## 7.3 构造参数

```python
def __init__(
    self,
    request_id: int,
    screenshot_manager: ScreenshotManager,
    context: ConnectedWindowContext | None,
    options: CaptureOptions,
) -> None:
    ...
```

---

## 7.4 推荐代码骨架

```python
from __future__ import annotations

from PySide6.QtCore import QObject, Signal, Slot

from nzfz_executor.core.models import CaptureOptions, ConnectedWindowContext
from nzfz_executor.core.screenshot_manager import ScreenshotManager


class ScreenshotCaptureWorker(QObject):
    finished = Signal(int, object)
    failed = Signal(int, str)

    def __init__(
        self,
        request_id: int,
        screenshot_manager: ScreenshotManager,
        context: ConnectedWindowContext | None,
        options: CaptureOptions,
    ) -> None:
        super().__init__()
        self._request_id = request_id
        self._screenshot_manager = screenshot_manager
        self._context = context
        self._options = options

    @Slot()
    def run(self) -> None:
        try:
            result = self._screenshot_manager.capture(
                self._context,
                self._options,
            )
            self.finished.emit(self._request_id, result)
        except Exception as exc:
            self.failed.emit(self._request_id, str(exc))
```

# 8. ScreenshotTaskRunner 设计

## 8.1 职责

`ScreenshotTaskRunner` 负责：

```text
1. 创建 QThread
2. 创建 ScreenshotCaptureWorker
3. 将 Worker moveToThread
4. 连接线程启动信号
5. 转发 finished/failed 信号
6. 清理 worker 和 thread
```

---

## 8.2 信号设计

```python
capture_finished = Signal(int, object)
capture_failed = Signal(int, str)
```

---

## 8.3 对外接口

```python
def start_capture(
    self,
    request_id: int,
    context: ConnectedWindowContext | None,
    options: CaptureOptions,
) -> None:
    ...
```

---

## 8.4 推荐代码骨架

```python
from __future__ import annotations

from PySide6.QtCore import QObject, QThread, Signal

from nzfz_executor.core.models import CaptureOptions, ConnectedWindowContext
from nzfz_executor.core.screenshot_manager import ScreenshotManager
from ui.workers.screenshot_workers import ScreenshotCaptureWorker


class ScreenshotTaskRunner(QObject):
    capture_finished = Signal(int, object)
    capture_failed = Signal(int, str)

    def __init__(
        self,
        screenshot_manager: ScreenshotManager,
        parent: QObject | None = None,
    ) -> None:
        super().__init__(parent)
        self._screenshot_manager = screenshot_manager
        self._threads: list[QThread] = []

    def start_capture(
        self,
        request_id: int,
        context: ConnectedWindowContext | None,
        options: CaptureOptions,
    ) -> None:
        thread = QThread(self)
        worker = ScreenshotCaptureWorker(
            request_id=request_id,
            screenshot_manager=self._screenshot_manager,
            context=context,
            options=options,
        )

        worker.moveToThread(thread)

        thread.started.connect(worker.run)

        worker.finished.connect(self.capture_finished)
        worker.failed.connect(self.capture_failed)

        worker.finished.connect(thread.quit)
        worker.failed.connect(thread.quit)

        worker.finished.connect(worker.deleteLater)
        worker.failed.connect(worker.deleteLater)

        thread.finished.connect(thread.deleteLater)
        thread.finished.connect(lambda: self._remove_thread(thread))

        self._threads.append(thread)
        thread.start()

    def _remove_thread(self, thread: QThread) -> None:
        if thread in self._threads:
            self._threads.remove(thread)
```

# 10. request_id 与 connection_generation

## 10.1 为什么需要 request_id

截图是异步任务。

可能出现：

```text
用户点击刷新截图
截图尚未返回
用户断开连接
旧截图返回
```

此时旧截图不能覆盖当前 UI。

---

## 10.2 为什么需要 connection_generation

如果用户：

```text
连接窗口 A
点击刷新截图
快速切换到窗口 B
A 的截图稍后返回
```

不能将 A 的截图显示为当前连接窗口的截图。

因此每个截图请求需要记录当时的：

```python
connection_generation
```

---

## 10.3 UI 维护映射

推荐由 UI 维护：

```python
self._capture_request_generations: dict[int, int] = {}
```

发起截图时：

```python
request_id = self._next_capture_request_id()
self._active_capture_request_id = request_id
self._capture_request_generations[request_id] = self._connection_generation
```

结果返回时：

```python
request_generation = self._capture_request_generations.get(request_id)

if request_id != self._active_capture_request_id:
    return

if request_generation != self._connection_generation:
    return
```

# 12. 截图请求流程

## 12.1 用户点击刷新截图

```text
点击刷新截图
    ↓
检查是否正在截图
    ↓
获取 connected_context
    ↓
生成 request_id
    ↓
记录 request_id -> connection_generation
    ↓
设置 _is_capturing=True
    ↓
显示“正在刷新截图...”
    ↓
启动超时计时器
    ↓

```

---

## 12.2 推荐代码骨架

```python
def _on_refresh_screenshot_clicked(self) -> None:
    if self._is_capturing:
        return

    context = self._window_manager.get_connected_context()
    if context is None:
        self._show_screenshot_feedback(
            FeedbackCode.SCREENSHOT_UNAVAILABLE,
        )
        return

    request_id = self._next_capture_request_id()
    self._active_capture_request_id = request_id
    self._capture_request_generations[request_id] = self._connection_generation

    self._is_capturing = True
    self._show_screenshot_feedback(
        FeedbackCode.SCREENSHOT_CAPTURING,
    )
    self._update_screenshot_button_state()

    self._capture_timeout_timer.start(SCREENSHOT_TIMEOUT_MS)

    options = CaptureOptions(
        region=CaptureRegion.CLIENT,
        backend=CaptureBackendType.AUTO,
        require_foreground=False,
        allow_occluded=True,
    )

    self._screenshot_task_runner.start_capture(
        request_id=request_id,
        context=context,
        options=options,
    )
```

# 13. 截图成功处理

## 13.1 处理流程

```text
收到 capture_finished
    ↓
检查 request_id 是否仍是 active
    ↓
检查 connection_generation 是否一致
    ↓
停止超时计时器
    ↓
设置 _is_capturing=False
    ↓
如果 result.success=True
        转换 PIL Image 到 QPixmap
        显示预览
        显示成功状态
        显示元信息
    ↓
如果 result.success=False
        保留旧图
        显示失败状态
    ↓
恢复按钮状态
```

---

## 13.2 推荐代码骨架

```python
def _on_capture_finished(
    self,
    request_id: int,
    result: ScreenshotResult,
) -> None:
    if not self._is_active_capture_result(request_id):
        return

    self._capture_timeout_timer.stop()
    self._active_capture_request_id = None
    self._is_capturing = False

    if result.success and result.image is not None:
        self._update_screenshot_preview(result)
        self._show_screenshot_feedback(
            FeedbackCode.SCREENSHOT_SUCCESS,
        )
        self._show_screenshot_meta(result)
    else:
        self._show_screenshot_failed_result(result)

    self._update_screenshot_button_state()
```

# 14. 截图失败处理

## 14.1 Worker failed

Worker 异常不直接展示原始异常。

用户侧显示：

```text
截图失败，请稍后重试
```

日志记录原始异常。

```python
def _on_capture_failed(
    self,
    request_id: int,
    message: str,
) -> None:
    if not self._is_active_capture_result(request_id):
        return

    self._capture_timeout_timer.stop()
    self._active_capture_request_id = None
    self._is_capturing = False

    logger.warning("Screenshot worker failed: %s", message)

    self._show_screenshot_feedback(
        FeedbackCode.SCREENSHOT_FAILED,
    )
    self._update_screenshot_button_state()
```

---

## 14.2 ScreenshotResult 失败

如果 Core 返回：

```python
ScreenshotResult(success=False, message="窗口已最小化，无法截图")
```

则用户侧可以显示：

```python
result.message
```

因为这是 Core 业务失败信息，面向用户友好。

如果 `result.message` 为空，则显示标准文案：

```python
FeedbackCode.SCREENSHOT_FAILED
```

---

## 14.3 失败时是否清空预览

P2-02 规定：

```text
截图失败不清空上一张预览。
```

原因：

```text
1. 方便用户保留最近一次可用截图
2. 避免 UI 闪烁
3. 用户可以根据状态文案知道当前刷新失败
```

但状态必须显示失败，避免误解。

# 15. 断开连接与切换连接处理

断开连接时：

```text

2. 清空截图元信息
3. 显示未连接无法截图
```

# 16. PIL Image 到 QPixmap 转换

## 16.1 转换规则

Core 返回：

```python
PIL.Image.Image
```

UI 显示需要：

```python
QPixmap
```

推荐转换：

```python
def _pil_image_to_pixmap(self, image: Image.Image) -> QPixmap:
    rgba_image = image.convert("RGBA")
    data = rgba_image.tobytes("raw", "RGBA")

    qimage = QImage(
        data,
        rgba_image.width,
        rgba_image.height,
        QImage.Format_RGBA8888,
    )

    return QPixmap.fromImage(qimage.copy())
```

关键点：

```text
必须 qimage.copy()
```

原因：

```text
data 是 Python 临时 bytes，对 QImage 生命周期不安全。
copy() 可以确保 Qt 持有自己的图像数据。
```

# 17. 预览缩放规则

## 17.1 缩放目标

# 22. 结果有效性判断

## 22.1 推荐方法

```python
def _is_active_capture_result(self, request_id: int) -> bool:
    if request_id != self._active_capture_request_id:
        logger.debug(
            "Discard expired screenshot result: request_id=%s, active=%s",
            request_id,
            self._active_capture_request_id,
        )
        return False

    request_generation = self._capture_request_generations.get(request_id)

    if request_generation != self._connection_generation:
        logger.debug(
            "Discard screenshot result due to generation mismatch: "
            "request_id=%s, request_generation=%s, current_generation=%s",
            request_id,
            request_generation,
            self._connection_generation,
        )
        return False

    return True
```
