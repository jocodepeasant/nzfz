## 1. 文档目标

本文档用于定义游戏连接页的 UI 状态机设计。

P1-02 已解决：

```text
搜索、连接、健康检测不阻塞 UI
```

P1-03 已解决：

```text
旧请求、迟到请求、断开前请求不会污染当前 UI
```

但页面仍需要统一管理以下内容：

- 搜索按钮何时启用；
- 连接按钮何时启用；
- 断开按钮何时启用；
- 搜索结果表格何时可操作；
- 已连接状态下是否允许连接新窗口；
- 健康检测结果如何映射到 UI 状态；
- 最小化窗口能否连接；
- 执行入口何时启用；
- 搜索状态和连接状态如何同时存在；
- 各类异步事件回来后 UI 应进入什么状态。

P1-04 的目标是：

```text
为游戏连接页建立统一的 UI 状态机，使用状态驱动控件启用、状态标签和操作入口，避免各个槽函数分散修改 UI 导致状态不一致。
```

---

## 2. 设计结论

根据 P1 设计讨论，P1-04 最终采用以下方案：

| 项目 | 最终选择 |
|---|---|
| 状态模型 | 双状态模型 |
| 搜索状态 | `SearchUiState` |
| 连接状态 | `ConnectionUiState` |
| 是否新增独立 StateController | 否，先在 `game_connect.py` 内实现 |
| UI 刷新方式 | 状态驱动 `_apply_ui_state()` |
| 状态变更入口 | `_set_search_state()` / `_set_connection_state()` |
| 是否允许已连接时连接新窗口 | 允许 |
| 已连接时连接新窗口 | 弹确认框 |
| 用户确认切换后 | 断开旧连接并连接新窗口 |
| 用户取消切换后 | 保持原连接状态 |
| 切换失败后是否恢复旧连接 | 否 |
| 切换失败后最终状态 | `DISCONNECTED` |
| 搜索中是否保留旧结果 | 是 |
| 搜索中是否允许连接 | 否 |
| 搜索失败/超时是否清空结果 | 是 |
| 断开连接后是否保留搜索结果 | 是 |
| 最小化窗口是否展示 | 是 |
| 最小化窗口是否可选中 | 是 |
| 最小化窗口是否可连接 | 否 |
| 执行入口启用条件 | 仅 `CONNECTED_READY` |
| `CONNECTING` 时是否允许断开 | 否 |
| `CONNECTING` 时是否允许搜索 | 否 |
| 健康检测是否驱动连接状态 | 是 |
| 最终文案标准化 | P1-05 实现 |

## 4. 为什么采用双状态模型

游戏连接页中，“搜索状态”和“连接状态”并不是严格互斥。

例如：

```text
已连接窗口 A，同时用户搜索窗口 B
```

此时页面同时具有：

```text
连接状态：已连接且执行就绪
搜索状态：正在搜索
```

如果使用单一状态，例如：

```text
SEARCHING
CONNECTED_READY
```

则无法准确表达：

```text
已连接且正在搜索
```

因此 P1-04 采用双状态模型：

```text
SearchUiState + ConnectionUiState
```

其中：

```text
SearchUiState 描述搜索区域状态
ConnectionUiState 描述连接生命周期状态
```

UI 控件启用规则由两者综合计算。

## 5. 搜索状态定义

### 5.1 枚举

```python
from enum import Enum


class SearchUiState(Enum):
    IDLE = "idle"
    SEARCHING = "searching"
    EMPTY = "empty"
    HAS_RESULT = "has_result"
    ERROR = "error"
```

---

### 5.2 状态含义

| 状态 | 含义 |
|---|---|
| `IDLE` | 尚未搜索、搜索输入为空，或搜索区域处于初始状态 |
| `SEARCHING` | 正在搜索窗口 |
| `EMPTY` | 搜索完成，但没有找到结果 |
| `HAS_RESULT` | 搜索完成，且存在可展示结果 |
| `ERROR` | 搜索失败或搜索超时 |

---

### 5.3 搜索状态流转

```text
输入为空
    → IDLE

开始搜索
    → SEARCHING

搜索成功且有结果
    → HAS_RESULT

搜索成功但无结果
    → EMPTY

搜索失败
    → ERROR

搜索超时
    → ERROR
```

---

## 6. 连接状态定义

### 6.1 枚举

```python
from enum import Enum


class ConnectionUiState(Enum):
    DISCONNECTED = "disconnected"
    CONNECTING = "connecting"
    CONNECTED_READY = "connected_ready"
    CONNECTED_NOT_READY = "connected_not_ready"
    CONNECTED_UNHEALTHY = "connected_unhealthy"
```

---

### 6.2 状态含义

| 状态 | 含义 |
|---|---|
| `DISCONNECTED` | 未连接、已断开，或当前没有有效连接 |
| `CONNECTING` | 正在连接目标窗口 |
| `CONNECTED_READY` | 已连接，窗口健康且执行就绪 |
| `CONNECTED_NOT_READY` | 已连接，窗口健康但执行未就绪 |
| `CONNECTED_UNHEALTHY` | 已连接对象存在，但健康异常 |

---

### 6.3 连接状态流转

```text
页面初始化
    → DISCONNECTED

开始连接
    → CONNECTING

连接成功
    → CONNECTED_READY

连接失败
    → DISCONNECTED

连接超时
    → DISCONNECTED

用户断开
    → DISCONNECTED

健康检测：未连接
    → DISCONNECTED

健康检测：已连接、健康、就绪
    → CONNECTED_READY

健康检测：已连接、健康、未就绪
    → CONNECTED_NOT_READY

健康检测：已连接但不健康
    → CONNECTED_UNHEALTHY
```

## 8. 状态更新入口

### 8.1 设置搜索状态

```python
def _set_search_state(
    self,
    state: SearchUiState,
    message: str = "",
) -> None:
    self._search_state = state
    self._search_message = message
    self._apply_ui_state()
```

---

### 8.2 设置连接状态

```python
def _set_connection_state(
    self,
    state: ConnectionUiState,
    message: str = "",
) -> None:
    self._connection_state = state
    self._connection_message = message
    self._apply_ui_state()
```

---

### 8.3 统一刷新 UI

所有状态相关控件刷新应集中在：

```python
def _apply_ui_state(self) -> None:
    ...
```

事件处理函数原则上不应分散直接修改：

```python
search_button.setEnabled(...)
connect_button.setEnabled(...)
disconnect_button.setEnabled(...)
execute_button.setEnabled(...)
```

而应通过状态变更触发统一刷新。

## 9. UI 控件启用规则

### 9.1 搜索输入框

搜索输入框启用条件：

```python
search_input_enabled = (
    self._connection_state != ConnectionUiState.CONNECTING
)
```

说明：

```text
连接中不允许搜索。
```

其他状态下允许搜索，包括已连接状态。

---

### 9.2 搜索按钮

搜索按钮启用条件：

```python
search_button_enabled = (
    self._connection_state != ConnectionUiState.CONNECTING
    and self._search_state != SearchUiState.SEARCHING
)
```

说明：

```text
连接中不允许搜索。
搜索中不允许重复点击搜索按钮。
```

---

### 9.3 搜索结果表格

搜索结果表格启用条件：

```python
result_table_enabled = (
    self._connection_state != ConnectionUiState.CONNECTING
)
```

说明：

```text
连接中禁用结果表格。
搜索中可以保留旧结果并允许查看，但不能发起连接。
```

---

### 9.4 连接按钮

连接按钮启用条件：

```python
connect_button_enabled = (
    self._connection_state != ConnectionUiState.CONNECTING
    and self._search_state == SearchUiState.HAS_RESULT
    and self._search_state != SearchUiState.SEARCHING
    and selected_window is not None
    and not selected_window.is_minimized
)
```

说明：

```text
未连接时点击连接，直接连接所选窗口。
已连接时点击连接，弹确认框，确认后切换连接目标。
连接中不可点击连接。
搜索中不可点击连接。
没有选中窗口不可点击连接。
选中最小化窗口不可点击连接。
```

---

### 9.5 断开按钮

断开按钮启用条件：

```python
disconnect_button_enabled = self._connection_state in {
    ConnectionUiState.CONNECTED_READY,
    ConnectionUiState.CONNECTED_NOT_READY,
    ConnectionUiState.CONNECTED_UNHEALTHY,
}
```

说明：

```text
只有存在有效连接或异常连接状态时，才允许断开。
CONNECTING 时不允许断开，因为 P1 阶段不支持取消连接。
```

---

### 9.6 执行入口

执行入口启用条件：

```python
execute_button_enabled = (
    self._connection_state == ConnectionUiState.CONNECTED_READY
)
```

说明：

```text
只有已连接、健康且执行就绪时才允许执行后续动作。
```

执行入口包括但不限于：

```text
开始执行按钮
截图按钮
主流程启动按钮
```

如当前 UI 尚未实现执行入口，应预留该规则。

## 10. `_apply_ui_state()` 推荐实现骨架

```python
def _apply_ui_state(self) -> None:
    selected_window = self._get_selected_window_info()

    is_connecting = (
        self._connection_state == ConnectionUiState.CONNECTING
    )
    is_searching = (
        self._search_state == SearchUiState.SEARCHING
    )

    can_search = not is_connecting and not is_searching

    can_connect = (
        not is_connecting
        and not is_searching
        and self._search_state == SearchUiState.HAS_RESULT
        and selected_window is not None
        and not selected_window.is_minimized
    )

    can_disconnect = self._connection_state in {
        ConnectionUiState.CONNECTED_READY,
        ConnectionUiState.CONNECTED_NOT_READY,
        ConnectionUiState.CONNECTED_UNHEALTHY,
    }

    can_execute = (
        self._connection_state == ConnectionUiState.CONNECTED_READY
    )

    self.search_input.setEnabled(not is_connecting)
    self.search_button.setEnabled(can_search)
    self.result_table.setEnabled(not is_connecting)

    self.connect_button.setEnabled(can_connect)
    self.disconnect_button.setEnabled(can_disconnect)

    if hasattr(self, "execute_button"):
        self.execute_button.setEnabled(can_execute)

    self._update_search_status_label()
    self._update_connection_status_label()
    self._update_execute_status_label()
```

## 12. 连接流程状态机

### 12.1 点击连接按钮

连接按钮点击统一入口：

```python
def _on_connect_button_clicked(self) -> None:
    window_info = self._get_selected_window_info()
    if window_info is None:
        return

    if window_info.is_minimized:
        self._show_hint("窗口已最小化，请恢复窗口后重试")
        self._apply_ui_state()
        return

    if self._is_connected_state():
        if not self._confirm_switch_connection(window_info):
            return
        self._disconnect_for_switch()

    self._start_connect(window_info)
```

---

### 12.2 当前未连接时

```text
点击连接
    ↓
_start_connect()
    ↓
CONNECTING
    ↓
连接成功
    ↓
CONNECTED_READY
```

---

### 12.3 当前已连接时

```text
点击连接其他窗口
    ↓
弹出确认框
    ↓
用户取消
    ↓
保持原状态
```

或：

```text
点击连接其他窗口
    ↓
弹出确认框
    ↓
用户确认
    ↓
停止健康检测
废弃旧连接/健康检测请求
断开旧连接

连接新窗口
```

### 12.4 开始连接

```python
def _start_connect(self, window_info: WindowInfo) -> None:
    self._connect_request_id += 1
    request_id = self._connect_request_id
    self._active_connect_request_id = request_id

    self._connection_generation += 1
    generation = self._connection_generation
    self._connect_request_generations[request_id] = generation

    self._connecting = True

    self._stop_health_timer()
    self._health_check_running = False
    self._invalidate_health_checks()

    self._set_connection_state(
        ConnectionUiState.CONNECTING,
        "正在连接窗口...",
    )

    self._start_connect_timeout(request_id)

    options = ConnectOptions()
    self._task_runner.start_connect(
        request_id=request_id,
        window_info=window_info,
        options=options,
    )
```

---

### 12.5 连接成功

连接成功后：

```python
self._connecting = False

self._set_connection_state(
    ConnectionUiState.CONNECTED_READY,
    "已连接，执行就绪",
)

self._start_health_timer()
```

说明：

```text
由于 P0-06 连接成功前会激活目标窗口，因此连接成功后可暂定为 CONNECTED_READY。
后续健康检测会继续校正状态。
```

---

### 12.6 连接业务失败

```python
self._connecting = False

self._set_connection_state(
    ConnectionUiState.DISCONNECTED,
    result.message,
)
```

说明：

```text
P1-04 当前规则下，已连接切换前会先断开旧连接。
因此连接失败后不恢复旧连接，最终状态为 DISCONNECTED。
```

---

### 12.7 连接异常

```python
self._connecting = False

self._set_connection_state(
    ConnectionUiState.DISCONNECTED,
    message,
)
```

---

### 12.8 连接超时

```python
self._connecting = False
```

## 13. 已连接状态下切换连接目标

### 13.1 交互规则

当当前连接状态为：

```text
CONNECTED_READY
CONNECTED_NOT_READY
CONNECTED_UNHEALTHY
```

且用户选中新的可连接窗口并点击连接按钮时，应弹出确认框。

确认框含义：

```text
当前已连接其他窗口，继续操作将断开当前连接并连接所选窗口。
```

---

### 13.2 用户取消

如果用户取消：

```text
不停止健康检测
不废弃健康检测请求
不调用 disconnect_window()
不修改 connection_generation
不改变 connection_state
不改变 search_state
```

---

### 13.3 用户确认

如果用户确认：

```text
停止健康检测
废弃旧连接请求
废弃旧健康检测请求
断开旧连接
不展示 DISCONNECTED 中间态
发起新连接
进入 CONNECTING
```

---

### 13.4 确认弹框推荐代码

```python
def _confirm_switch_connection(self, window_info: WindowInfo) -> bool:
    reply = QMessageBox.question(
        self,
        "切换连接窗口",
        "当前已连接其他窗口，继续操作将断开当前连接并连接所选窗口。是否继续？",
        QMessageBox.Yes | QMessageBox.No,
        QMessageBox.No,
    )
    return reply == QMessageBox.Yes
```

文案由 P1-05 统一标准化。

### 13.5 切换内部断开

```python
def _disconnect_for_switch(self) -> None:
    self._stop_health_timer()

    self._active_connect_request_id = 0
    self._active_health_request_id = 0

    self._connecting = False
    self._health_check_running = False

    self._connect_request_generations.clear()
    self._health_request_generations.clear()

    self._clear_connect_timeouts()
    self._clear_health_timeouts()

    self._window_manager.disconnect_window()
```

注意：

```text
_disconnect_for_switch() 不设置 connection_state = DISCONNECTED。
_disconnect_for_switch() 不展示“已断开”提示。
_disconnect_for_switch() 不单独递增 connection_generation。
```

原因：

```text
后续 _start_connect() 会进入 CONNECTING，并递增 connection_generation。
```

### 13.6 切换失败后的状态

如果新连接失败、异常或超时：

```text
connection_state = DISCONNECTED
```

不尝试恢复旧连接。

原因：

- 用户已经确认断开旧连接；
- 恢复旧连接需要重新校验和激活；
- 旧窗口状态可能已变化；
- 自动恢复会增加状态机复杂度；
- P1 阶段以稳定性和清晰语义为主。

## 16. 健康检测结果到连接状态的映射

### 16.1 映射规则

```python
def _apply_health_result(self, result: HealthCheckResult) -> None:
    if not result.is_connected:
        self._set_connection_state(
            ConnectionUiState.DISCONNECTED,
            "未连接游戏窗口",
        )
        return

    if result.is_healthy and result.is_ready:
        self._set_connection_state(
            ConnectionUiState.CONNECTED_READY,
            "已连接，执行就绪",
        )
        return

    if result.is_healthy and not result.is_ready:
        self._set_connection_state(
            ConnectionUiState.CONNECTED_NOT_READY,
            "已连接，但窗口未在前台",
        )
        return

    self._set_connection_state(
        ConnectionUiState.CONNECTED_UNHEALTHY,
        result.message or "连接异常",
    )
```


连接状态异常，执行入口禁用，但允许用户断开或重新连接其他窗口。

```python
    self._searching = False

    if results:

        self._set_search_state(SearchUiState.HAS_RESULT)
```

## 26. 验收标准

P1-04 完成后，应满足以下验收标准。

### 26.1 状态模型验收

- 使用 `SearchUiState` 表达搜索区域状态；
- 使用 `ConnectionUiState` 表达连接生命周期状态；
- 搜索状态和连接状态可同时存在；
- 状态更新通过统一方法进入；
- UI 控件刷新集中在 `_apply_ui_state()`。

---

### 26.2 搜索状态验收

- 初始状态为 `IDLE`；
- 开始搜索进入 `SEARCHING`；
- 搜索有结果进入 `HAS_RESULT`；
- 搜索无结果进入 `EMPTY`；
- 搜索失败或超时进入 `ERROR`；
- 搜索失败或超时清空结果；
- 搜索中保留旧结果但不能连接。

---

### 26.3 连接状态验收

- 初始连接状态为 `DISCONNECTED`;
- 开始连接进入 `CONNECTING`;
- 连接成功进入 `CONNECTED_READY`;
- 连接失败进入 `DISCONNECTED`;
- 连接超时进入 `DISCONNECTED`;
- 断开连接进入 `DISCONNECTED`;
- 健康检测可将连接状态更新为 ready/not ready/unhealthy。

---

### 26.4 按钮状态验收

- 连接中搜索按钮禁用；
- 连接中连接按钮禁用；
- 连接中断开按钮禁用；
- 连接中执行入口禁用；
- 搜索中连接按钮禁用；
- 选中最小化窗口时连接按钮禁用；
- 没有选中窗口时连接按钮禁用；
- 选中正常窗口且搜索有结果时连接按钮启用；
- `CONNECTED_READY` 时执行入口启用；
- `CONNECTED_NOT_READY` 和 `CONNECTED_UNHEALTHY` 时执行入口禁用。

---

### 26.5 切换连接验收

- 已连接时点击连接新窗口会弹确认框；
- 用户取消后不改变当前连接；
- 用户取消后健康检测继续；
- 用户确认后停止健康检测；
- 用户确认后断开旧连接；
- 用户确认后进入连接中状态；
- 切换成功后连接新窗口；
- 切换失败后不恢复旧连接；
- 切换失败后状态为 `DISCONNECTED`。

---

### 26.6 最小化窗口验收

- 最小化窗口可以显示在搜索结果中；
- 最小化窗口可以被选中；
- 最小化窗口不能连接；
- 选中最小化窗口时连接按钮禁用；
- 选中最小化窗口时提示恢复窗口。

## 28. 最终结论

P1-04 的核心设计结论是：

```text
游戏连接页采用 SearchUiState + ConnectionUiState 双状态模型。
```

其中：

```text
SearchUiState 管理搜索区域状态；
ConnectionUiState 管理连接生命周期状态；
_apply_ui_state() 统一驱动控件启用、状态标签和执行入口。
```

P1-04 支持已连接状态下切换连接目标：

```text
已连接时点击连接新窗口
    ↓
弹出确认框
    ↓
确认后断开旧连接并连接新窗口
    ↓
切换失败后不恢复旧连接
```

同时明确：

```text
搜索中保留旧结果但禁用连接；
搜索失败/超时清空结果；
断开连接后保留搜索结果；
最小化窗口可展示、可选中、不可连接；
执行入口仅在 CONNECTED_READY 时启用。
```

P1-04 完成后，游戏连接页 UI 将由明确状态驱动，避免按钮、标签、表格、连接状态在异步事件中出现不一致。
