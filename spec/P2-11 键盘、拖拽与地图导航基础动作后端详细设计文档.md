# 2. 背景说明

P2-10 已经完成执行器对地图配置器 V1.1.1 Basic Profile 脚本的加载、校验、手动波次驱动和基础动作执行闭环。

P2-10 打通了以下链路：

```text
地图配置器 Basic Export
        ↓
script.json V1.1.1 Basic
        ↓
执行器 UI 选择脚本
        ↓
ScriptLoader
        ↓
Basic Validator
        ↓
manual_wave_driver
        ↓
log / wait / pan_to_region / place_trap
        ↓
dry-run / 半执行日志闭环
```

但 P2-10 中的部分动作仍然是日志级 dry-run：

```text
1. map_open_close
2. reset_view_to_origin
3. drag_ratio
4. trap.select_key
```

因此，P2-10 虽然完成了“脚本协议到动作语义”的闭环，但尚未完成“动作语义到真实输入”的闭环。

P2-11 的目标是在 P2-10 的基础上补齐真实键盘输入与真实鼠标拖拽能力，使 Basic Profile 脚本可以进入初步实机执行阶段。

---

# 3. 总体目标

P2-11 的目标是：

```text
在现有 MouseController.click 基础上，补齐 KeyboardController 和 MouseController.drag，使 P2-10 的 Basic 脚本可以执行真实地图打开/关闭、视野重置、区域拖拽、陷阱快捷键选择和槽位点击。
```

具体目标：

```text
1. 新增 KeyboardController。
2. 新增 KeyboardInputBackend 抽象。
3. 新增 DryRunKeyboardInputBackend。
4. 新增 WindowsKeyboardInputBackend。
5. MouseInputBackend 增加 drag。
6. MouseController 增加 drag。
7. DryRunMouseInputBackend 增加 drag。
8. WindowsMouseInputBackend 增加 drag。
9. 新增 KeyPressAction。
10. 新增 MouseDragAction。
11. ScriptExecutor drag_ratio 改为调用 MouseController.drag。
12. ScriptExecutor trap.select_key 改为调用 KeyboardController.press。
13. reset_view_to_origin 使用可配置 reset sequence。
14. UI 确保 dry-run 开关存在且默认开启。
15. 用户关闭 dry-run 时允许真实键盘/拖拽/点击。
16. 真实输入前检查窗口连接状态和坐标边界。
17. 真实输入前如果目标窗口不是前台：输出 warning，但不阻止。
18. 日志明确区分 dry-run / real-run。
```

# 8. 动作模型设计

## 8.1 KeyPressAction

```python
@dataclass(frozen=True)
class KeyPressAction:
    key: str
    hold_ms: int = 0
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `key` | `str` | 要按下的键，例如 `"1"`、`"2"`、`"o"` |
| `hold_ms` | `int` | 按住时长，默认 0 |

---

## 8.2 MouseDragAction

```python
@dataclass(frozen=True)
class MouseDragAction:
    start: ScreenPoint
    end: ScreenPoint
    duration_ms: int = 300
    button: str = "left"
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `start` | `ScreenPoint` | 拖拽起点屏幕坐标 |
| `end` | `ScreenPoint` | 拖拽终点屏幕坐标 |
| `duration_ms` | `int` | 拖拽持续时间 |
| `button` | `str` | 鼠标按钮，默认 `left` |

---

## 8.3 可选后续模型

P2-11 最小只要求 `KeyPressAction` 和 `MouseDragAction`。

---

## 9.2 职责

`KeyboardController` 负责：

```text
1. 校验 key 是否为空。
2. 归一化 key。
3. 检查窗口连接状态。
4. 检查目标窗口是否前台。
5. 前台不匹配时输出 warning，但不阻止。
6. dry-run 时只输出日志。
7. real-run 时调用 KeyboardInputBackend。
8. 捕获异常并返回 ActionResult。
```

---

## 9.3 press 流程

```text
1. 检查 stop 状态。
2. 检查 key 非空。
3. normalize_key。
4. 检查目标窗口连接状态。
5. 检查目标窗口是否前台。
6. 如果非前台，输出 warning，但继续。
7. 如果 dry_run=true，输出 dry-run 日志并返回 success。
8. 如果 dry_run=false，调用 backend.press。
9. 返回 success。
```

## 11.2 normalize_key

示例：

```python
def normalize_key(key: str) -> str:
    key = key.strip()
    if len(key) == 1:
        return key.lower()
    return key.lower()
```

示例映射：

| 输入 | 归一结果 |
|---|---|
| `"O"` | `"o"` |
| `"o"` | `"o"` |
| `"1"` | `"1"` |

| `"Esc"` | `"esc"` |

```text

2. mouse down。

4. mouse up。
```

```python
steps = max(5, duration_ms // 16)
```


```python
    move_steps()
finally:
    mouse_up()
```

# 14. 安全策略

P2-11 的真实输入安全检查分为三类。

---

## 14.1 必须阻止执行的情况

以下情况必须返回失败，不执行真实输入：

```text
1. 未连接游戏窗口。
2. 窗口句柄无效。
3. 鼠标点击坐标越界。
4. 鼠标拖拽 from/to 坐标越界。
5. key 为空或不支持。
6. 当前执行状态已停止。
```

---

## 14.2 只 warning，不阻止的情况

以下情况只输出 warning，不阻止执行：

```text
1. 目标窗口不是前台。
```

日志必须明确：

```text
[Safety][Warning] 目标窗口当前不是前台，真实输入可能发送到错误窗口，继续执行。
```

如果能获取窗口信息，建议输出：

```text
目标窗口：hwnd=xxx title=xxx
当前前台：hwnd=yyy title=yyy
```

---

## 14.3 dry-run 与 real-run

```text
dry-run=true：
  只输出日志，不真实输入。

dry-run=false：
  执行真实 keyboard / drag / click。
```

# 17. reset_view_to_origin 设计

## 17.1 总体策略

P2-11 中 `reset_view_to_origin` 使用：

```text
可配置 reset sequence
```

但不新增 `MapNavigationController`。

---

## 17.2 默认配置

建议新增本地配置：

```python
DEFAULT_MAP_RESET_SEQUENCE = [
    {
        "type": "press_key",
        "key": "o",
        "wait_ms": 500,
    }
]
```

如果后续实测需要开关两次，可配置为：

```python
DEFAULT_MAP_RESET_SEQUENCE = [
    {
        "type": "press_key",
        "key": "o",
        "wait_ms": 200,
    },
    {
        "type": "press_key",
        "key": "o",
        "wait_ms": 500,
    },
]
```

P2-11 默认推荐：

```text
press O + wait 500ms
```

原因：

```text
双击 O 会依赖当前地图开关状态，容易反向。
```

---

## 17.3 支持的 sequence step

P2-11 最小支持：

```text
press_key
wait
```

示例：

```python
[
    {"type": "press_key", "key": "o", "wait_ms": 500},
    {"type": "wait", "duration_ms": 300},
]
```

---

## 17.4 执行流程

```text
1. 输出：开始 reset_view_to_origin。
2. 读取 DEFAULT_MAP_RESET_SEQUENCE。
3. 逐步执行 press_key / wait。
4. press_key 走 KeyboardController.press。
5. wait 支持 stop。
6. 执行完成后，current_region_id = map.initial_view.origin_region_id。
7. 输出 warning：P2-11 暂无 map_ui_detection，无法确认 reset 是否成功。
```

日志示例：

```text
[Region] reset_view_to_origin sequence start
[Keyboard] press key=o
[Wait] 500ms
[Region] reset_view_to_origin completed -> current_region_id=origin
[Region][Warning] P2-11 暂无 map_ui_detection，无法确认地图视野是否真的回到 origin
```

# 18. drag_ratio 集成设计

## 18.1 P2-10 行为

```text
drag_ratio dry-run from=(x1,y1) to=(x2,y2)
```

---

## 18.2 P2-11 行为

```text
1. 读取 from/to ratio。
2. ratio -> screen point。
3. 构造 MouseDragAction。
4. 调用 MouseController.drag。
```

伪代码：

```python
def execute_drag_ratio(step: dict[str, Any]) -> ActionExecutionResult:
    from_ratio = parse_ratio_point(step["from"])
    to_ratio = parse_ratio_point(step["to"])
    duration_ms = int(step.get("duration_ms", 300))
    repeat = int(step.get("repeat", 1))

    start = coordinate_mapper.ratio_to_screen_point(viewport_rect, from_ratio)
    end = coordinate_mapper.ratio_to_screen_point(viewport_rect, to_ratio)

    for _ in range(max(1, repeat)):
        result = mouse_controller.drag(
            MouseDragAction(
                start=start,
                end=end,
                duration_ms=duration_ms,
                button="left",
            ),
            context,
        )
        if not result.success:
            return result

    return ActionExecutionResult(success=True)
```

# 20. ScriptExecutor 修改点

## 20.1 region.enter_actions

支持：

```text
reset_view_to_origin
drag_ratio
wait
```

其中：

```text
reset_view_to_origin -> reset sequence
drag_ratio -> MouseController.drag
wait -> 可停止等待
```

---

## 20.2 place_trap

修改：

```text
trap.select_key 从日志 dry-run 改为 KeyboardController.press。
```

---

## 20.3 current_region_id

规则不变：

```text
1. reset_view_to_origin 后 current_region_id = origin_region_id。
2. pan_to_region 成功后 current_region_id = target region_id。
3. place_trap ensure_region 成功后 current_region_id = slot.region_id。
```

# 24. 验收标准

## 24.1 KeyboardController 验收

```text
1. dry-run press 输出日志，不真实按键。
2. real press 能按数字键 1/2。
3. real press 能按字母 O。
4. 无效 key 返回失败。
5. key 为空返回失败。
6. 执行时记录 key 和 hold_ms。
7. 目标窗口不是前台时 warning 但继续。
```

---

## 24.2 MouseController.drag 验收

```text
1. dry-run drag 输出 from/to/duration。
2. real drag 能从起点拖到终点。
3. from/to 超出 viewport 时拒绝执行。
4. duration_ms 生效。
5. drag 后释放鼠标按键。
6. 目标窗口不是前台时 warning 但继续。
```

---

## 24.3 reset_view_to_origin 验收

```text
1. 能读取默认 reset sequence。
2. 能执行 press_key/wait step。
3. dry-run 下不真实按键。
4. real-run 下真实按键。
5. 执行后 current_region_id 更新为 origin_region_id。
6. 日志提示 P2-11 暂无法确认 reset 成功。
```

---

## 24.4 ScriptExecutor 集成验收

```text
1. drag_ratio 调用 MouseController.drag。
2. trap.select_key 调用 KeyboardController.press。
3. place_trap 执行顺序为 select_key -> click slot。
4. reset_view_to_origin 执行 reset sequence。
5. dry-run 下所有输入均不真实执行。
6. 非 dry-run 下 keyboard/drag/click 均走真实后端。
```

---

## 24.5 UI 安全验收

```text
1. dry-run 默认开启。
2. 取消 dry-run 时有确认弹窗或明显 warning。
3. 真实执行启动时日志提示风险。
4. 前台窗口不匹配时 warning 但不阻止。
```

# 27. 风险与应对

## 27.1 真实输入发到错误窗口

风险：

```text
P2-11 决策为目标窗口不是前台时只 warning 不阻止。
```

应对：

```text
1. 日志明确提示风险。
2. real-run 启动时弹窗确认。
3. 输出目标窗口和前台窗口信息。
4. dry-run 默认开启。
```

---

## 27.2 reset_view_to_origin 不可靠

风险：

```text
P2-11 没有 map_ui_detection，无法确认地图是否真的回到 origin。
```

应对：

```text
1. reset sequence 可配置。
2. 执行后输出无法确认 warning。
3. P2-12 实现 map_ui_detection 和导航确认。
```

---

## 27.3 拖拽不生效

风险：

```text
游戏可能不接受过快或非连续拖拽。
```

应对：

```text
1. drag 分步移动。
2. duration_ms 生效。
3. 支持调大 duration。
4. 后续根据实测优化 SendInput 实现。
```

---

## 27.4 键盘输入不被游戏识别

风险：

```text
游戏可能不接受某些注入方式。
```

应对：

```text
1. 优先沿用 P2-08 输入后端。
2. 如 pyautogui 不稳定，切换 SendInput。
3. 日志保留后端类型。
```

# 29. 最终结论

P2-11 的核心目标是把 P2-10 中的日志级输入动作升级为可真实执行的输入动作：

```text
KeyboardController.press
MouseController.drag
reset_view_to_origin sequence
drag_ratio real-run
trap.select_key real-run
```

P2-11 完成后，系统将从：

```text
脚本驱动的 dry-run / 半执行闭环
```

升级为：

```text
脚本驱动的基础真实输入闭环
```

但 P2-11 仍不负责确认游戏状态变化。

因此，P2-11 的边界是：

```text
输入动作能发出去。
```

P2-12 的边界才是：

```text
确认输入动作产生了预期 UI 状态变化。
```
