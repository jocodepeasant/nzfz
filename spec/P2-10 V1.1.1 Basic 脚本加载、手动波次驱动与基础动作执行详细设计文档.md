# 2. 背景说明

当前执行器已经完成 P2 阶段的一系列基础能力：

```text
1. 游戏窗口连接
2. 窗口截图
3. 截图预览
4. 执行器状态机
5. 异步执行与停止机制
6. 执行日志反馈
7. 模板匹配最小闭环
8. Windows 输入动作后端
9. 模板匹配与资源管理
```

当前系统已经可以完成：

```text
连接窗口
  ↓
截图
  ↓
模板匹配
  ↓
坐标映射
  ↓
dry-run / 真实点击
  ↓
输出日志
```

但当前执行逻辑仍然偏固定：

```text
识别模板
点击结果
```

尚未真正接入地图配置器导出的塔防脚本。

根据最新协议设计：

```text
《逆战：未来》塔防自动化通关测试系统需求与架构设计 V1.1.1 修订说明
地图配置器 V1.1.1 Basic Export MVP 实现规则
```

地图配置器第一阶段会导出：

```text
V1.1.1 Basic Profile script.json
```

因此，P2-10 的目标是让执行器接入该脚本，实现从“固定识别点击”到“脚本驱动执行”的第一条基础闭环。

---

# 3. 总体目标

P2-10 建立以下端到端链路：

```text
地图配置器 Basic Export
        ↓
script.json V1.1.1 Basic
        ↓
执行器 UI 选择脚本
        ↓
ScriptLoader
        ↓
Basic Validator
        ↓
manual_wave_driver
        ↓
log / wait / pan_to_region / place_trap
        ↓
dry-run / 半执行日志闭环
```

具体目标：

```text
1. UI 支持选择 script.json。
2. Worker 能接收脚本路径和执行模式。
3. 执行器能加载 script.json。
4. 执行器能解析 V1.1.1 Basic Profile。
5. 执行器能进行基础 compatibility 校验。
6. 执行器能进行基础 semantic 校验。
7. 执行器支持 manual_wave_driver。
8. 执行器支持 sequential / single_wave 两种执行模式。
9. 执行器支持 enabled=false action 跳过。
10. 执行器支持 action_id 日志。
11. 执行器支持 current_region_id 缓存。
12. 执行器支持 GameViewportRect ratio 坐标换算。
13. 执行器支持 log action。
14. 执行器支持 wait action。
15. 执行器支持 pan_to_region action。
16. 执行器支持 place_trap action。
17. 执行器支持 retry_basic。
18. 执行器默认 dry-run 安全执行。
19. 执行器能输出清晰 warnings/errors。
20. 执行器能对接地图配置器 Basic Export MVP。
```

# 7. 支持的 action 类型

P2-10 允许 `enabled=true` 的 action 类型：

```python
SUPPORTED_ENABLED_ACTION_TYPES_BASIC = {
    "place_trap",
    "wait",
    "log",
    "pan_to_region",
}
```

如果 action 为：

```json
{
  "enabled": false,
  "type": "upgrade_trap"
}
```

则允许加载，执行时跳过。

如果 action 为：

```json
{
  "enabled": true,
  "type": "upgrade_trap"
}
```

则加载/校验失败。

# 8. 执行模式

P2-10 使用：

```text
manual_wave_driver
```

代替 OCR 自动触发。

支持两种执行模式：

```text
sequential
single_wave
```

---

## 8.1 sequential

UI 文案：

```text
顺序执行全部波次（Manual）
```

内部值：

```text
sequential
```

语义：

```text
不依赖 wave_ocr。
按 waves 数组顺序执行所有 wave.actions。
每个 wave.trigger 仅作为日志展示和语义参考，不用于 OCR 自动触发。
```

日志示例：

```text
manual_wave_driver: sequential，按脚本顺序模拟触发 wave。
```

---

## 8.2 single_wave

UI 文案：

```text
手动执行指定波次
```

内部值：

```text
single_wave
```

语义：

```text
不依赖 wave_ocr。
只执行 wave == 用户输入波次号 的 actions。
相当于手动模拟 wave_eq(value=N) 成立。
```

日志示例：

```text
manual_wave_driver: single_wave，手动模拟 wave=1 trigger 满足。
```

# 15. ScriptValidator 设计

Validator 输出：

```python
@dataclass(frozen=True)
class ScriptValidationResult:
    success: bool
    warnings: list[str]
    errors: list[str]
```

---

## 15.1 必须失败规则

```text
1. script.json 不存在。
2. JSON 解析失败。
3. schema_version != 1.1.0。
4. game_mode != tower_defense。
5. compatibility 缺失。
6. compatibility.profile != tower_defense_v1_basic。
7. required_capabilities 不满足。
8. map.coordinate_mode 不是 viewport_ratio_after_region_enter 或兼容值。
9. map.floors.length > 1 且执行器不支持 multi_floor。
10. trap_id 重复。
11. region_id 重复。
12. slot_id 重复。
13. action_id 重复。
14. enabled=true action type 不支持。
15. slot.position 缺失。
16. slot.position ratio 越界。
17. slot.region_id 不存在。
18. slot.floor_id 与 region.floor_id 不一致。
19. place_trap.trap_id 不存在。
20. place_trap.slot_id 不存在。
21. pan_to_region.region_id 不存在。
22. enabled=true upgrade_trap 不支持。
23. enabled=true remove_trap 不支持。
24. retry.max_attempts < 1。
```

---

## 15.2 Warning 规则

```text
1. templates 存在但 Basic 不使用。
2. color_rules 存在但 Basic 不使用。
3. rois 存在但 Basic 不使用。
4. action.conditions 存在，Basic 将视为通过。
5. action.verify 存在，Basic 将跳过。
6. disabled action 将跳过。
7. retry.reset_view_before_retry=true，但 P2-10 未支持 retry_reset_view，降级忽略。
8. retry.micro_adjust_on_retry=true，但 P2-10 不支持 micro_adjust，降级忽略。
9. metadata.json 缺失，忽略。
```

---

## 15.3 Capability 校验

检查：

```text
script.compatibility.required_capabilities ⊆ SUPPORTED_CAPABILITIES_BASIC
```

P2-10 忽略：

```text
optional_capabilities
```

不作为失败条件。

# 17. ScriptExecutionState

```python

    current_region_id: str | None = None
    executed_waves: set[int] = field(default_factory=set)
```

---

## 17.1 current_region_id 规则

```text
1. 初始为 None。
2. reset_view_to_origin 后更新为 map.initial_view.origin_region_id。
3. 成功执行 pan_to_region 后更新为目标 region_id。
4. 成功执行 place_trap 的 ensure_region 后更新为 slot.region_id。
5. 如果执行器无法确认当前 region，应置为 None。
6. 如果动作失败或人工干预，可置为 None。
```

# 19. ScriptExecutor 执行流程

```text
1. 输出脚本基本信息。
2. 输出 Basic Profile 说明。
3. 根据 options.script_execution_mode 选择 waves。
4. 计算 action 总数。
5. 遍历 waves。
6. 遍历 wave.actions。
7. 检查 stop。
8. 如果 action.enabled=false，跳过并记录日志。
9. 输出 action_id/type/name。
10. 执行 retry wrapper。
11. 分发 action handler。
12. 更新 progress。
13. 全部完成后返回 success。
```

# 21. Retry Basic 设计

## 21.1 retry 解析

从 action.raw 读取：

```json
"retry": {
  "max_attempts": 2,
  "interval_ms": 800,
  "reset_view_before_retry": false,
  "micro_adjust_on_retry": false
}
```

默认：

```text
max_attempts = runtime.default_retry_count 或 1
interval_ms = 800
```

注意：

```text
max_attempts 表示总尝试次数，包含第一次执行。
```

---

## 21.2 retry wrapper

流程：

```text
for attempt in 1..max_attempts:
    执行 action handler
    if success:
        return success
    if stopped:
        return stopped
    if attempt < max_attempts:
        wait interval_ms
return failed
```

---

## 21.3 retry 增强字段处理

P2-10 不真实支持：

```text
reset_view_before_retry
micro_adjust_on_retry
```

如果为 true：

```text
记录 warning。
本次执行忽略该增强行为。
```

# 26. conditions 处理

Basic Export MVP 默认不导出 action.conditions。

P2-10 策略：

```text
1. 如果没有 conditions：正常执行。
2. 如果 conditions 为空：正常执行。
3. 如果 conditions 存在：warning + 视为通过。
```

日志：

```text
Basic Profile 当前不执行 conditions，已视为通过：action_id=xxx
```

如果未来开启 strict：

```text
strict_compatibility=true 时，包含 unsupported conditions 可失败。
```

P2-10 默认：

```text
strict_compatibility=false
```

# 27. verify 处理

Basic Export MVP 推荐：

```text
action.verify 不导出。
slot.verify 保留。
```

P2-10 策略：

```text
1. slot.verify 解析但不使用。
2. action.verify 如果不存在：正常。
3. action.verify.required=false：warning + 跳过。
4. action.verify.required=true：
   - dry-run：warning + 跳过。
   - 非 dry-run：建议失败。
```

如果当前系统无法可靠判断 dry-run，则 P2-10 可先统一：

```text
warning + 跳过。
```

但日志必须清晰说明。

# 37. 最终结论

P2-10 的目标不是完整塔防自动化，而是建立执行器对地图配置器 Basic Export 的第一条稳定链路：

```text
地图配置器 Basic Export
        ↓
script.json V1.1.1 Basic
        ↓
执行器 UI 选择脚本
        ↓
ScriptLoader
        ↓
Basic Validator
        ↓
manual_wave_driver
        ↓
log / wait / pan_to_region / place_trap
        ↓
dry-run / 半执行日志闭环
```

P2-10 完成后，系统将从：

```text
固定识别点击闭环
```

升级为：

```text
脚本驱动的基础动作执行闭环
```

这将为后续真实键盘/拖拽、OCR 条件、视觉校验、升级/拆除、完整通关自动化打下核心基础。
