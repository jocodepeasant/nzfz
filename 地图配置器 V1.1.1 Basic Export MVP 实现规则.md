# 地图配置器 V1.1.1 Basic Export MVP 实现规则

版本：`MVP-1`

目标协议：`tower_defense_script_v1.1.1 basic profile`

适用模块：`map-configurator`

---

# 1. 实现目标

本规则用于指导地图配置器在**不大拆现有数据模型、不重构状态管理、不完整实现 V1.1.1 全能力**的前提下，最小成本导出一个可被 Basic 执行器理解的 `script.json`。

核心原则：

```text
1. 不大拆模型。
2. 不重做 project.json。
3. 不强制内部状态完全等于协议模型。
4. V1.1.1 作为导出格式，而不是内部重构目标。
5. 先实现 basic profile 导出闭环。
6. OCR、视觉校验、升级、拆除、多楼层执行等能力暂缓。
```

---

# 2. MVP 输出物

地图配置器导出脚本包目录：

```text
<export_package_dir>/
├── script.json
├── templates/      可为空
├── previews/       可为空
└── metadata.json   可选
```

第一阶段必须导出：

```text
script.json
```

第一阶段可选导出：

```text
templates/
previews/
metadata.json
```

---

# 3. MVP 非目标

本 MVP 暂不要求实现：

```text
1. 完整 JSON Schema V1.1.1
2. 完整 Protocol Model / Editor Model 分离
3. 完整 capability 自动推导
4. 完整 resource package 管理
5. OCR 识别
6. 模板匹配真实验证
7. slot visual verify
8. trap level verify
9. resource_gte 条件
10. slot_empty / slot_occupied 条件
11. upgrade_trap 真实执行
12. remove_trap 真实执行
13. custom_steps
14. micro_adjust
15. 多楼层真实执行
16. Full Recognition Profile
```

---

# 4. 导出策略总览

现阶段导出流程：

```text
当前项目状态 / project.json / renderer state
        ↓
normalizeBasicExport()
        ↓
buildScriptV111Basic()
        ↓
validateBasicExport()
        ↓
write script.json
```

不要引入大型模型拆分。

推荐实现一个轻量导出入口：

```ts
export function buildScriptV111Basic(project: ProjectState): unknown
```

如果当前项目已有类似导出函数，可直接在原函数中加 V1.1.1 basic 分支。

# 5. 顶层字段导出规则

## 5.1 输出结构

导出的 `script.json` 必须包含：

```json
{
  "schema_version": "1.1.0",
  "script_id": "",
  "script_name": "",
  "game_mode": "tower_defense",
  "compatibility": {},
  "package": {},
  "map": {},
  "runtime": {},
  "recognition": {},
  "traps": [],
  "regions": [],
  "slots": [],
  "waves": [],
  "boss_reserved": {},
  "metadata": {}
}
```

---

## 5.2 字段补齐规则

| 字段 | 来源 | 缺省值 |
|---|---|---|
| `schema_version` | 固定 | `"1.1.0"` |
| `script_id` | 当前项目 / 旧脚本 | `"unnamed_script"` |
| `script_name` | 当前项目 / 旧脚本 | `"未命名脚本"` |
| `game_mode` | 固定 | `"tower_defense"` |
| `compatibility` | 固定模板 | basic profile |
| `package` | 固定模板 | directory package |
| `map` | 当前项目 | 缺字段时补默认 |
| `runtime` | 当前项目 / 默认 | 默认 runtime |
| `recognition` | 当前项目 / 默认 | 空 rois/templates/color_rules |
| `traps` | 当前项目 / 陷阱库快照 | 空数组或当前引用陷阱 |
| `regions` | 当前项目 | 至少 origin |
| `slots` | 当前项目 | 空数组 |
| `waves` | 当前项目 | 空数组 |
| `boss_reserved` | 固定 | disabled |
| `metadata` | 当前项目 / 固定 | 时间、描述 |

# 6. compatibility 固定模板

MVP 阶段不做复杂 capability 推导，直接导出 fixed basic profile。

```json
{
  "executor_min_version": "0.2.0",
  "profile": "tower_defense_v1_basic",
  "required_capabilities": [
    "script_load",
    "script_validate",
    "viewport_ratio_coordinate",
    "manual_wave_driver",
    "map_open_close",
    "reset_view_to_origin",
    "pan_to_region",
    "place_trap",
    "wait_action",
    "log_action",
    "retry_basic",
    "condition_basic",
    "dry_run"
  ],
  "optional_capabilities": [
    "window_capture",
    "map_ui_detection",
    "template_matching",
    "resource_ocr",
    "wave_ocr",
    "slot_visual_verify",
    "trap_level_verify",
    "micro_adjust",
    "upgrade_trap",
    "remove_trap"
  ]
}
```

## 6.1 MVP 限制

由于 required_capabilities 固定为 basic profile，所以导出时必须保证：

```text
enabled=true 的 action 不依赖 optional_capabilities。
```

因此：

```text
1. upgrade_trap 必须 enabled=false。
2. remove_trap 必须 enabled=false。
3. custom_steps 不得 enabled=true。
4. action.verify.required 不得为 true。
5. 不得导出依赖 resource_ocr 的 resource_gte condition。
6. 不得导出依赖 slot_visual_verify 的 slot_empty / slot_occupied condition。
7. 不得导出依赖 trap_level_verify 的 trap_level_lt / trap_level_gte condition。
8. retry.micro_adjust_on_retry 必须为 false。
9. retry.reset_view_before_retry 第一阶段建议为 false，除非执行器已经支持 retry_reset_view。
```

# 8. map 导出规则

## 8.1 输出结构

```json
{
  "map_id": "space_station",
  "map_name": "空间站",
  "difficulty": "normal",
  "strategy_id": "baseline",
  "base_resolution": {
    "width": 1920,
    "height": 1080
  },
  "coordinate_mode": "viewport_ratio_after_region_enter",
  "floors": [
    {
      "floor_id": "f1",
      "name": "一层"
    }
  ],
  "default_floor_id": "f1",
  "initial_view": {
    "type": "fixed_after_open_map",
    "origin_region_id": "origin"
  }
}
```

---

## 8.2 字段补齐规则

| 字段 | 来源 | 缺省值 |
|---|---|---|
| `map_id` | 当前项目 | `"unknown_map"` |
| `map_name` | 当前项目 | `"未命名地图"` |
| `difficulty` | 当前项目 | `"normal"` |
| `strategy_id` | 当前项目 | `"baseline"` |
| `base_resolution.width` | 当前项目 | `1920` |
| `base_resolution.height` | 当前项目 | `1080` |
| `coordinate_mode` | 固定 | `"viewport_ratio_after_region_enter"` |
| `floors` | 当前项目 floors | 默认单楼层 |
| `default_floor_id` | 当前项目 | floors[0].floor_id |
| `initial_view.type` | 固定 | `"fixed_after_open_map"` |
| `initial_view.origin_region_id` | 固定/当前项目 | `"origin"` |

---

## 8.3 floors 规则

如果当前项目已有：

```json
"floors": []
```

则沿用。

如果没有，则补：

```json
[
  {
    "floor_id": "f1",
    "name": "一层"
  }
]
```

`default_floor_id`：

```text
1. 优先使用当前项目 active/default floor。
2. 其次使用 floors[0].floor_id。
3. 如果 floors 为空，则补 f1。
```

---

## 8.4 多楼层 MVP 限制

MVP 可以导出多楼层字段，但 basic 执行器只支持单楼层。

因此：

```text
如果 floors.length > 1：
  方案 A：导出前 warning，不阻止。
  方案 B：阻止 basic 导出，提示需要 multi_floor。
```

建议 MVP 使用 **方案 B**：

```text
Basic Export MVP 暂不允许导出 floors.length > 1 的可执行脚本。
```

除非所有非默认楼层的 region/slot/action 都未启用。

---

## 11.2 字段补齐规则

| 字段 | 缺省值 |
|---|---|
| `trap_id` | 必须存在；缺失报错 |
| `trap_name` | trap_id |
| `select_key` | `"1"` 或空字符串 |

| `upgrade_mode` | `"all_same_type"` |
| `recognition.placed_templates` | `[]` |
| `recognition.icon_template` | `""` |
| `recognition.level_templates` | `{}` |

---

## 11.3 陷阱库处理

当前应用级陷阱库可以继续保留原逻辑。

MVP 导出时只需要：

```text
1. 把当前脚本实际引用的陷阱导出到 traps。
2. 或者导出当前项目选择的所有陷阱。
3. 不要求导出整个应用陷阱库。
```

推荐：

```text
只导出 slots.default_trap 和 wave place_trap.trap_id 实际引用到的陷阱。
```

如果引用的 trap 不存在：

```text
导出前报错。
```

# 13. slots 导出规则

## 13.1 输出结构

```json
{
  "slot_id": "A01",
  "floor_id": "f1",
  "name": "左入口减速位1",
  "region_id": "entrance_left",
  "position": {
    "x_ratio": 0.452,
    "y_ratio": 0.561
  },
  "precision": {
    "click_tolerance_px": 6,
    "allow_micro_adjust": false,
    "micro_adjust_pattern": "none",
    "micro_adjust_step_px": 0
  },
  "slot_type": "ground",
  "default_trap": "slow_trap",
  "verify": {
    "empty_method": "none",
    "occupied_method": "none",
    "level_method": "none",
    "check_area": {
      "x_ratio": 0.435,
      "y_ratio": 0.545,
      "w_ratio": 0.035,
      "h_ratio": 0.035
    }
  }
}
```

---

## 13.2 字段补齐规则

| 字段 | 缺省值 |
|---|---|
| `slot_id` | 必须存在；缺失报错 |
| `floor_id` | `map.default_floor_id` |
| `name` | `slot_id` |
| `region_id` | `"origin"` 或报错，推荐报错 |
| `position` | 必须存在 |
| `precision.click_tolerance_px` | `6` |
| `precision.allow_micro_adjust` | `false` |
| `precision.micro_adjust_pattern` | `"none"` |
| `precision.micro_adjust_step_px` | `0` |
| `slot_type` | `"ground"` |
| `default_trap` | 可空；如果非空必须存在 |
| `verify.empty_method` | `"none"` |
| `verify.occupied_method` | `"none"` |
| `verify.level_method` | `"none"` |
| `verify.check_area` | 根据 position 生成默认小矩形或沿用现有 |

---

## 13.3 坐标规则

MVP 阶段推荐：

```text
1. 如果当前项目 slot.position 已经是 viewport ratio，则原样导出。
2. 如果当前项目明确标记为 content_rect ratio，则转换后导出。
3. 如果无法判断，先按 viewport ratio 导出，并在导出日志中 warning。
```

---

## 13.4 默认 check_area

如果没有 `verify.check_area`，可以基于 position 生成：

```text
half_size = 0.015
x = clamp(position.x_ratio - half_size, 0, 1)
y = clamp(position.y_ratio - half_size, 0, 1)
w = min(half_size × 2, 1 - x)
h = min(half_size × 2, 1 - y)
```

# 15. actions 导出规则

## 15.1 通用字段补齐

所有 action 必须导出：

```json
{
  "action_id": "w1_001_place_trap",
  "type": "place_trap",
  "enabled": true,
  "name": "",
  "description": ""
}
```

字段补齐：

| 字段 | 缺省值 |
|---|---|
| `action_id` | 自动生成 |
| `type` | 必须存在 |
| `enabled` | `true` |
| `name` | action_id |
| `description` | `""` |

---

## 15.2 action_id 自动生成规则

如果 action 没有 `action_id`，生成：

```text
w{wave}_{index3}_{type}
```

例如：

```text
w1_001_place_trap
w1_002_wait
w1_003_log
```

如果冲突，则追加后缀：

```text
w1_001_place_trap_2
```

---

## 15.3 MVP 允许 enabled=true 的 action 类型

MVP basic profile 只允许以下 action 启用：

```text
place_trap
wait
log
pan_to_region
```

其中 `pan_to_region` 可选，因为 `place_trap` 默认 ensure_region。

---

## 15.4 place_trap 导出

```json
{
  "action_id": "w1_001_place_trap",
  "type": "place_trap",
  "enabled": true,
  "name": "放置陷阱",
  "trap_id": "slow_trap",
  "slot_id": "A01",
  "retry": {
    "max_attempts": 2,
    "interval_ms": 800,
    "reset_view_before_retry": false,
    "micro_adjust_on_retry": false
  },
  "on_fail": {
    "policy": "skip"
  }
}
```

MVP 规则：

```text
1. trap_id 必须存在。
2. slot_id 必须存在。
3. 不导出 conditions，或仅导出空 conditions。
4. 不导出 verify.required=true。
5. retry.micro_adjust_on_retry 固定 false。
6. retry.reset_view_before_retry 建议 false。
```

# 16. conditions 处理规则

MVP basic profile 不支持真实状态条件。

因此：

```text
1. enabled=true action 默认不导出 conditions。
2. resource_gte 不导出。
3. slot_empty / slot_occupied 不导出。
4. trap_level_lt / trap_level_gte 不导出。
5. wave_eq 只保留在 wave.trigger，不作为 action condition 导出。
```

如果用户已有条件配置：

```text
导出时 warning：
Basic profile 暂不支持该条件，已忽略。
```

或者如果希望更安全：

```text
阻止导出，需要用户确认。
```

推荐 MVP：

```text
warning + 忽略。
```

---

# 17. verify 处理规则

MVP basic profile 不支持 required visual verify。

因此：

```text
1. action.verify 默认不导出。
2. 如果保留 verify，则 required 必须为 false。
3. slot.verify.check_area 可以继续导出，作为后续识别资源。
4. verify.required=true 必须降级为 false 或阻止导出。
```

推荐 MVP：

```text
action.verify 不导出。
slot.verify 保留。
```

---

# 18. retry 处理规则

## 18.1 输出结构

```json
{
  "max_attempts": 2,
  "interval_ms": 800,
  "reset_view_before_retry": false,
  "micro_adjust_on_retry": false
}
```

---

## 18.2 字段规则

| 字段 | 默认 |
|---|---:|
| `max_attempts` | `runtime.default_retry_count` 或 `2` |
| `interval_ms` | `800` |
| `reset_view_before_retry` | `false` |
| `micro_adjust_on_retry` | `false` |

注意：

```text
max_attempts 表示总尝试次数，包含第一次执行。
```

如果旧字段是：

```text
retry_count 表示失败后重试次数
```

则转换为：

```text
max_attempts = retry_count + 1
```

如果不确定旧语义，MVP 直接使用：

```text
max_attempts = max(1, default_retry_count)
```

# 23. action normalize 伪代码

```ts
function normalizeAction(action: any, ctx: NormalizeActionContext): any {
  const type = action.type
  const enabled = action.enabled ?? true
  const actionId = action.action_id || generateActionId(ctx.wave, ctx.index, type)

  if (type === 'place_trap') {
    return {
      action_id: actionId,
      type: 'place_trap',
      enabled,
      name: action.name || actionId,
      description: action.description || '',
      trap_id: action.trap_id,
      slot_id: action.slot_id,
      retry: normalizeBasicRetry(action.retry, ctx.warnings),
      on_fail: normalizeOnFail(action.on_fail)
    }
  }

  if (type === 'wait') {
    return {
      action_id: actionId,
      type: 'wait',
      enabled,
      name: action.name || actionId,
      duration_ms: Math.max(0, action.duration_ms ?? 1000)
    }
  }

  if (type === 'log') {
    return {
      action_id: actionId,
      type: 'log',
      enabled,
      name: action.name || actionId,
      level: action.level || 'info',
      message: action.message || action.name || actionId
    }
  }

  if (type === 'pan_to_region') {
    return {
      action_id: actionId,
      type: 'pan_to_region',
      enabled,
      name: action.name || actionId,
      region_id: action.region_id,
      retry: normalizeBasicRetry(action.retry, ctx.warnings),
      on_fail: action.on_fail || { policy: 'fail' }
    }
  }

  if (type === 'upgrade_trap') {
    ctx.warnings.push(`${actionId}: basic profile 暂不支持 upgrade_trap，已禁用`)
    return {
      action_id: actionId,
      type: 'upgrade_trap',
      enabled: false,
      name: action.name || actionId,
      trap_id: action.trap_id,
      target_level: action.target_level,
      disabled_reason: 'basic profile 暂不支持 upgrade_trap'
    }
  }

  if (type === 'remove_trap') {
    ctx.warnings.push(`${actionId}: basic profile 暂不支持 remove_trap，已禁用`)
    return {
      action_id: actionId,
      type: 'remove_trap',
      enabled: false,
      name: action.name || actionId,
      slot_id: action.slot_id,
      disabled_reason: 'basic profile 暂不支持 remove_trap'
    }
  }

  if (enabled) {
    ctx.errors.push(`${actionId}: basic profile 不支持 enabled=true 的 action type=${type}`)
  }

  return {
    action_id: actionId,
    type,
    enabled: false,
    name: action.name || actionId,
    disabled_reason: `basic profile 不支持 action type=${type}`
  }
}
```

# 26. 最终 MVP 验收标准

地图配置器导出的 `script.json` 满足：

```text

7. 至少存在 origin region
8. regions 带 floor_id / enter_policy
9. slots 带 floor_id / position / precision / verify
10. waves.actions 都有 action_id / enabled
11. enabled=true 的 action 只包含 place_trap / wait / log / pan_to_region

13. 不导出 required=true action.verify
14. 不导出 resource_gte / slot_empty / trap_level 等依赖识别的 action.conditions
15. retry 使用 max_attempts
16. 导出前有轻量校验
17. errors 非空时禁止导出
18. warnings 非空时允许用户确认后导出
```

# 27. 结论

本 MVP 的目标不是完整实现 V1.1.1，而是以最小改动建立：

```text
地图配置器 → V1.1.1 Basic script.json → 执行器 Basic dry-run
```

的端到端闭环。

后续可在该闭环上逐步补充：

```text
1. JSON Schema
2. 完整 Semantic Validator
3. Compatibility 推导
4. templates 资源包
5. OCR
6. slot visual verify
7. upgrade_trap
8. remove_trap
9. Full Recognition Profile
```

当前阶段不建议大拆模型，不建议重构状态管理，不建议一次性实现 Full Profile。
